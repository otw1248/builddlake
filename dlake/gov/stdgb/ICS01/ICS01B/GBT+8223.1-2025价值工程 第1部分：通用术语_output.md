## 引言

价值工程是使用功能性和经济性的方法、有组织和创造性的活动，旨在最大限度地提高价值工程对象的价值。价值工程通常用于组织的一般管理层面，对提高组织效率和竞争力有重要影响。

为了更好地促进组织对价值工程的理解和技术交流，我国制定了 GB/T 8223《价值工程》，拟由两个部分构成。

——第1部分：通用术语。目的在于界定价值工程的通用术语及其定义。

——第 2 部分: 一般工作程序。目的在于为价值工程的开展确立工作程序, 并对工作程序各阶段活动提供指导和建议。

价值工程建立于价值的概念和基于功能的方法之上。因此，界定价值工程的通用术语及其定义至关重要。GB/T 8223.1 发布实施已十余年，这期间价值工程方法和技术取得了长足发展，一些通用术语及其定义随着实践和认知的发展发生了变化，并产生了一些新的术语。鉴于此，修订 GB/T 8223.1。此外，为了便于理解价值工程通用术语之间的关系，本文件的附录 A 给出了概念关系图。

## 价值工程 第 1 部分: 通用术语

## 1 范围

本文件界定了价值工程的基本术语以及涉及功能、成本、组织管理的术语和定义。

本文件适用于价值工程活动。

## 3 基本术语

### 3.1 

## 价值 value

对象 $  (3.3)  $ 所具有的功能 $  (4.1)  $ 与获得该功能 $  (4.1)  $ 所需成本 $  (5.1)  $ 之比。

### 3.2 

## 价值工程 value engineering

使用功能性和经济性的方法最大限度地提高对象（3.3）的价值（3.1）的、有组织和创造性的活动。

注1：价值工程的各项活动由价值工程工作组（6.4）实施，并在价值工程工作计划（6.1）中进行概述。

注2：产品、过程或服务等对象（3.3）的价值提高主要表现为以最低的寿命周期成本（5.2）达到所要求的功能（4.1）、可靠性、安全性等。

### 3.3 

## 价值工程对象 value engineering subject

对象 subject

应用价值工程3.2的客体。

注：价值工程对象（简称“对象”）主要指产品、过程、服务、系统或它们的组成部分。

## 4 功能术语

### 4.1 

功能 function

对象3.3能满足某种需求的效用或属性。

注：功能通常以抽象形式表达（如以非特定的双词抽象表达，由动词和名词组成），不含解决方案。

### 4.2 

基本功能 basic function

实现对象3.3存在的目的或意图的功能4.1。

### 4.3 

## 辅助功能 secondary function

支持基本功能4.2实现的功能4.1。

### 4.4 

## 用户相关功能 user related function

满足某个特定用户的需求的功能4.1。

### 4.5 

## 产品相关功能 product related function

为执行用户相关功能(4.4)而由产品的某个组成部分或产品的组成部分相互作用所产生的功能(4.1)。

注 $ ^{1} $ ：产品相关功能有时称为内部功能。

注2：产品相关功能可能是该产品的组成部分的用户相关功能(4.4)。

注3：产品相关功能可能与可用的技术相关。

注 $ ^{4} $ ：产品相关功能可能是必要的，也可能是不必要的，可取的或不可取的。

### 4.6 

## 不必要功能 unnecessary function

对满足用户需求没有贡献的功能4.1。

### 4.7 

## 不可取功能 undesirable function

对用户、可持续性或环境有不利影响的功能(4.1)。

注：不可取功能对对象3.3的价值3.1有负面影响，通常是技术选择带来的未曾预料到的结果。

### 4.8 

## 全时功能 all-the-time function

连续发生或重复、持续发生的功能4.1。

### 4.9 

## 一 次性功能 one-time function

仅发生一次的功能4.1。

### 4.10 

## 功能分析 function analysis

对各功能(4.1)及其相互关系进行系统表征、分类和评估的过程。

注：功能分析通常包括两种方法：功能需求分析（或外部功能分析）和技术功能分析（内部功能分析）。

### 4.11 

## 功能需求分析 functional need analysis

以用户相关功能4.4和限制条件的形式描述对象3.3需要满足的需求。

注：功能分析4.10的一部分，有时称为“外部功能分析”或“客户功能分析”。

### 4.12 

## 技术功能分析 technical function analysis

通过识别对象(3.3)的各组成部分或各组成部分的产品相关功能(4.5)，促进研究和形式化对象(3.3)的架构。

注：功能分析4.10的一部分，有时称为“内部功能分析”。

### 4.13 

## 功能结构 function structure

功能分析4.10产生的功能4.1的排列。

注 $ ^{1} $ ：功能结构能以树或图表的形式呈现，以便给出完整、直观的书面演示。

注2：当考虑产品相关功能4.5时，功能结构显示了功能4.1相互作用的方式。

### 4.14 

## 功能载体 function carrier

功能4.1得以实现的部件。

### 4.15 

## 功能的性能规范 functional performance specification

以用户相关功能(4.4)和限制条件的形式表达对象(3.3)的性能水平和评估准则的文件。

注：对于每一个功能 $ （4.1） $ 都考虑给出评估准则及其性能水平是必要的，具有一定程度的灵活性。

### 4.16 

## 功能需求表达 functional need expression

功能需求分析4.11的结果。

注：功能需求表达是编制功能的性能规范（4.15）主要部分的基础。

## 5 成本术语

### 5.1 

## 成本 cost

在特定对象 $  (3.3)  $ 上发生的或可归因于特定对象 $  (3.3)  $ 的支出。

注：成本通常以一个或多个利益相关方花费的资金表示。

### 5.2 

## 寿命周期成本 life cycle cost

在对象 $  (3.3)  $ 的整个寿命周期内的所有经常性和一次性(非经常性)成本 $  (5.1)  $ 的总和。

注：如果对象（3.3）为产品，寿命周期成本指有关这个产品的策划、设计、采购、生产、经营、维护、使用和处置等所发生费用的总和。

### 5.3 

## 功能成本 function cost

为对象 $  (3.3)  $ 具备功能 $  (4.1)  $ 而预计或发生的全部支出。

注：在设计或重新设计之前，功能成本是一个目标或极限：使具有该功能（4.1）所允许的支出。在开发或实施后，功能成本是已经实际发生的成本（5.1）。

### 5.4 

## 目标成本 target cost

在给定条件下最终功能4.1实现的成本5.1目标值。

## 6 组织管理术语

### 6.1 

## 价值工程工作计划 value engineering work plan

确保成功开展价值工程3.2的安排。

注：价值工程工作计划可能包括一个初步阶段，在此阶段建立数据以便了解价值工程项目（6.3）。

### 6.2 

## 价值工程目标 value engineering target

为价值工程工作组(6.4)设定的价值工程项目(6.3)的功能(4.1)、成本(5.1)和资源目标的统称。

注 $ ^{1} $ ：根据价值工程项目（6.3）的工作进度，能够确定一般目标或详细目标。

注2：资源目标指成本以外的目标，如可用性、时间、数量等。

### 6.3 

## 价值工程项目 value engineering project

价值工程(3.2)在某个对象(3.3)上的实施。

注：价值工程项目可能包括许多具体研究。

### 6.4 

## 价值工程工作组 value engineering team

承担价值工程项目6.3的多学科参与者组成的团队。

注：团队参与者的产生根据他们在对象 $ (3.3) $ 的各个方面表现出的能力、专业知识/责任选择。

### 6.5 

## 价值工程项目负责人/主持人 value engineering project leader or facilitator

被管理层任命的、具有专业知识，并具备负责组织、领导和协调价值工程工作组（6.4）相关工作能力的人。

### 6.6 

## 按目标设计 design to objective

从对象 $  (3.3)  $ 研发开始，以优化的方式满足指定的目标的一种管理方法。

注1：指定的目标是一组性能的预定水平，相较于将要设计和实现的对象（3.3）的其他性能和特性，需要优先实现。指定的目标原则上是不可协商的，一旦达成一致，即视为稳定的。

注2：按目标设计扩展了按成本设计（6.7），将功能（4.1）或其性能与其他因素（如质量）、成本（5.1）、资源和交付进行权衡。

注3：按目标设计可能涉及供应链，它通过合作方式将明确需求和指定的目标的实体与那些努力提高竞争力的供应商联系起来。明确共同目标和单个目标是项目早期阶段的一项重要活动。

### 6.7 

## 按成本设计 design to cost

价值工程项目(6.3)自开始阶段即在受控制状态，并使其在计划的成本(5.1)目标和时间目标内达到规定的性能的一种管理方法。

注1：按成本设计是一种管理价值工程项目（6.3）的方法，要求在项目早期阶段，根据过去的经验、预算、市场价格等，将成本（5.1）分配给对象（3.3）的组成部分或组件，然后在达成总体协议之前，对技术性能水平和成本（5.1）进行权衡。

注2：按成本设计要求在研发过程中确保成本5.1可见性。
