## 引言

北斗短报文服务作为北斗卫星导航系统的特色功能，是北斗系统区别于其他全球卫星导航系统独有的特色服务之一。其主要应用价值在于，当用户处于地面通信网络不能覆盖的区域时，能使用带有北斗短报文功能的北斗终端进行应急通信。

GB/T 44086 拟由两个部分构成。

——第1部分: 用户管理模块接口。目的在于确立北斗三号区域短报文通信用户终端与用户管理模块之间的接口规范。

——第 2 部分: 通用数据接口。目的在于确立北斗三号区域短报文通信用户终端与上位机之间通信的接口规范。

## 3 术语、定义和缩略语

### 3.1 术语和定义

GB/T 2312—1980、GB/T 39267 界定的以及下列术语和定义适用于本文件。

#### 3.1.1 

## 地址 address

用于标识北斗三号短报文业务中交互个体身份的唯一编号。

注：地址分为内网地址和外网地址两部分。

#### 3.1.2 

## 内网地址 intranet address

用于标识北斗三号系统内部个体身份的唯一编号。

注：内网地址也称用户ID，包括点播ID、通播ID、组播ID等。

#### 3.1.3 

## 外网地址 extranet address

用于标识与北斗三号系统点播通信进行信息交互的外部用户的唯一编号。

注：外网地址包括手机号码、微信用户号码、微博号码等。

#### 3.1.4 

## 国标码 GB code/GB 2312 code

中国于 1980 年发布的第一个汉字编码标准。

注：全称为《信息交换用汉字编码字符集 基本集》，简称 GB 2312，也称为汉字交换码，共收录 6 763 个常用汉字和字符，每个汉字或字符以两个字节来表示，每个字节的最高位均为 0，用于在计算机之间交换信息。

#### 3.1.5 

## 区位码 zone-location code

区位码是一种用于汉字库寻址的四位十进制数。

注：每个区位码都对应一个唯一的汉字或符号，其中高两位为区码，低两位为位码，取值范围均为01～94，由此组成一个 $ 94\times94 $ 的矩阵。

#### 3.1.6 

## 压缩区位码 compressed zone-location code

北斗短报文出入站空间接口格式中采用 14 个 bit 表示区位码。

注：其中前7bit表示区号，后7bit表示位号。

#### 3.1.7 

## 汉字内码 machine code

计算机内部存储，处理加工和传输汉字时所用的由0和1符号组成的代码。

注：也称“汉字 ASCII 码”“机内码”，简称“内码”。

### 3.2 缩略语

下列缩略语适用于本文件。

BDCS: 北斗坐标系 (BeiDou Coordinate System)

RNSS: 卫星无线电导航业务 (Radio Navigation Satellite Service)

UTC: 协调世界时 (Coordinated Universal Time)

## 4 数据传输协议要求

### 4.1 物理接口

用户终端物理接口采用 RS-232C 标准接口。电信号特性应符合 GB/T 6107—2000 中第 2 章的规定，串行数据传输标准应符合 GB/T 11014—1989 中第 4 章的规定。用户可根据业务需要自行选择物理接口形态：DB9、DB25 或其他形态通信标准接口。

### 4.2 数据传输

数据以串行异步方式传送。第一位为起始位，其后是数据位，最后是停止位。数据遵循最低有效位优先的规则。所用参数如下：

a）波特率：4 800 bit/s、9 600 bit/s、19 200 bit/s 和 115 200 bit/s，可根据需要设定，默认值为 115 200 bit/s；

b) 起始位: 1 bit;

c） 数据位：8 bit；

d）停止位：1 bit；

e） 校验位: 无。

有效字符集包括  $ 0 \times 20 \sim 0 \times 7 $  F 的 ASCII 字符。

#### 5.1.3 非定义字符

没有定义成“预留字符”和“有效字符”的 ASCII 字符，任何时候都不应用于除用户数据外的其他字段。

#### 5.1.4 字符符号

当用个别字符定义测量单位、说明数据字段类型和语句类型等内容时，应依据语句中的注释解释这些字符。

### 5.2 字段

#### 5.2.1 字段定义

字段是指位于两个适当的定界字符之间的一串有效字符。

#### 5.2.2 数据字段

数据字段包括字符型和数字型。语句中的数据字段跟在定界符“，”之后，通过观察字段定界符“，”确定数据字段在一条语句中的位置。对接收端，通过定界符的计数来确定字段位置。专有语句中的数据字段只包含有效字符和定界符“，”

对于可变长度的数据字段，数据字段长度固定、可变、固定+可变均可。可变长度的字段可是字符型也可是数字型。可变长度的数据字段包含一个小数点，开头和结尾可是几个“0”。

对于固定长度的数据字段，若有效数据位长度不够，则在前面补上足够数量的 ASCII 码字符“0”。

#### 5.2.3 校验和字段

在所有的语句中均应传送校验和字段：

a）校验和字段是语句中的最后一个字段，它在校验和定界符“*”之后；

b）校验和是对语句中所有字符的8位（不包括起始和结束位）执行XOR（异或）运算得到的结果，所有字符指在定界符“$”与“*”之间（但不包括这些定界符），包括“，”在内的全部字符；

c） 发送时将 16 进制的高 4 位和低 4 位转换成两个 ASCII 字符（0～9，A～F）。最高有效位首先发送。

示例：

$CCTCQ,13 721 585 961,2,0,2,0,140,一二三四五六七八九十*1A<CR><LF>

表示向13 721 585 961发送短报文，使用Lf2频点，单次发送，需要入站确认，编码类别为汉字，通信长度为140bit，其中十六进制数1A是对“CCTCQ，13 721 585 961，2，0，2，0，140，一二三四五六七八九十”进行异或计算得到的校验和。

### 5.3 语句

#### 5.3.1 语句定义

语句以语句起始定界符“$”开始，以语句终止符<CR><LF>结束。第一个字段是地址字段，其中包含发送端的标识符和语句格式符，该格式符规定语句中数据字段的个数、所含数据的类型，以及数据段的传送顺序。语句的其余部分可以是零个或多个数据段。

#### 5.3.2 有效语句

符合语句格式要求的语句都是有效语句，其他任何形式的语句都不是有效的，不应进行传输。

#### 5.3.3 服务输入语句发送间隔

服务输入语句发送间隔应符合用户注册时的频度限制。完整传送一条语句的时间不大于1 s。设置串行通信速率时宜考虑报文长度与通信速率的匹配。

### 5.4 错误检测和处理

终端应能检测数据传送中的差错，包括：

a）校验和错误；

b）无效字符；
