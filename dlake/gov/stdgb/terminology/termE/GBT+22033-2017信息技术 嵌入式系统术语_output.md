## 前言

本标准按照 GB/T 1.1—2009 给出的规则起草。

本标准代替 GB/T 22033—2008。

本标准与 GB/T 22033—2008 相比主要变化如下：

——增加了31个条目（见2.002、2.003、2.007～2.015、2.022、2.023、2.025～2.028、2.038～2.041、2.070～2.072、2.089、2.090、2.101、2.102、2.106和2.117）。

——修改了69个条目（见2.001、2.004～2.006、2.016、2.017、2.019～2.020、2.029～2.035、2.044、2.045、2.048～2.056、2.059～2.063、2.066、2.068、2.069、2.073、2.075、2.076、2.078、2.086～2.088、2.091～2.098、2.100、2.103、2.104、2.107、2.108、2.110～2.113、2.115和2.118～2.125。——删除了1个条目“个人智能通信器”（见2008年版的2.88）。

——中英文索引也作了调整。

本标准由全国信息技术标准化技术委员会(SAC/TC 28)提出并归口。

本标准起草单位：湖州师范学院、中国电子技术标准化研究院、杭州电子科技大学、北京中电华大电子设计有限责任公司、北京凯思昊鹏软件工程技术有限公司、太极计算机股份有限公司、上海市软件行业协会、中国科学院计算所。

本标准主要起草人：刘利民、张旸旸、陈德传、周建锁、韩冰、于跃、杨根兴、顾玉良、李云翔、朱珍民、闫萍、罗孝、刘杰、肖良军、孙赋瑶、陈晓华、侯向华、王晓霞。

## 信息技术 嵌入式系统术语

## 1 范围

本标准界定了嵌入式系统常用的术语和定义。

本标准适用于嵌入式系统的应用、开发、教学、出版和信息交流。

## 2 术语和定义

2.001

嵌入式系统 embedded system

置入应用对象内部，起信息处理或控制作用的专用计算系统。

2.002

嵌入式系统解决方案 embedded system solution

ESS

基于嵌入式系统或其软硬件的实际应用。

2.003

嵌入式系统的生态系统 ecosystem of embedded system

已经存在的，对嵌入式系统开发具有支撑作用的环境元素。

注：这些元素包括硬件、软件、平台、工具、开发实例等。

2.004

嵌入式软件 embedded software

满足嵌入式系统应用环境特殊要求的软件。

注：这些软件包括嵌入式应用软件和嵌入式系统软件。

2.005

嵌入式操作系统 embedded operating system

满足嵌入式系统应用环境特殊要求的操作系统软件。

2.006

实时操作系统 real-time operating system

RTOS

面向实时运行对象，并为其服务的操作系统。

注：嵌入式操作系统多采用多任务实时操作系统。

2.007

嵌入式传感器 embedded sensor

设置在检测对象内部的传感测试单元。

2.008

智能设备 smart equipment

内置以中央处理器 CPU 为核心的数字处理单元的设备。

2.009

可重置系统 reconfigurable system

性能和状态可被重新定义或配置的硬软件系统。

## GB/T 22033—2017

### 2. 010

## 动态可重置系统 dynamically reconfigurable system

在不断电状态下，性能和状态可被重新定义或配置的硬软件系统。

### 2. 011

## 硬软件协调设计 co-design of hardware and software

运用硬件和软件协同设计技术，实现系统功能的方法。

### 2. 012

## 低功耗系统 low-power system

通过降低运行频率或减少操作功能达到降低能量消耗目的的系统。

### 2. 013

## 多中央处理器 multi-CPU

多 CPU

设置在一个系统中的多个独立，相互之间具有通信联系的中央处理器。

### 2. 014

## 微处理器（单元） microprocessor（unit）

MPU

中央处理器被集成在一块芯片上的运算和控制单元。

注：以软件构成的微处理器模块，通常被称为微处理器核。

### 2. 015

## 多核处理器 multi-core processor

多个处理单元被集成在同一芯片上构成的处理器。

注：通常具有超过30个处理单元的多核处理器被称为超多核处理器。

### 2. 016

## 微控制器（单元） microcontroller（unit）

MCU

在一个芯片上构成的以实现控制目的为主的功能单元。其包括中央处理器、存储器、寄存器、接口等。

注：单片机（single-chip computer）与微控制器同义。

### 2. 017

## 多单片机系统 multi-microcontroller system

由多个单片机构成的系统。

### 2. 018

## 数字信号处理 digital signal processing

DSP

对信号进行数字化分析、运算、加工和使用的技术。

### 2. 019

## 数字信号处理器 digital signal processor

DSP

以微处理器为核心的数字信号处理单元，是嵌入式系统的硬件实现之一。

### 2. 020

## 单片系统 system on a chip

片上系统

SoC

在一个芯片上实现的嵌入式系统。

2.021

可编程单片系统 system on a programmable chip

可编程片上系统

SoPC

在一个可编程器件芯片上实现的嵌入式系统。

复杂指令(集)计算机 complex instruction set computer CISC

指令系统规模较大的计算机。

注：该类计算机指令系统通常不少于100条指令。

（精）简指令（集）计算机 reduced instruction set computer RISC

指令系统规模较小的计算机。

注：该类计算机指令系统通常少于100条指令。

专用集成电路 application specific integrated circuit

ASIC

为特定应用所设计的集成电路。

固定(逻辑)器件 fixable logic device

FLD

半导体芯片内逻辑功能不可以改变的电子器件。

可编程(逻辑)器件 programmable logic device

PLD

半导体芯片内逻辑功能可以通过软件编程来设置的电子器件。

### 2. 027

可编程逻辑阵列 programmable logic array

PLA

逻辑电路的与或阵列均可编程的电子逻辑单元。

### 2. 028

可编程阵列逻辑 programmable array logic

PAL

一种复杂的可编程逻辑电路单元。

注：该逻辑电路的与阵列是可编程的，其或阵列是固定的。

## 现场可编程门阵列 field programmable gate array FPGA

一种可通过软件设计实现硬件应用电路的器件。 $ ^{*} $ 注：通常把基于查找表技术的可编程器件称为FPGA。

### 2. 030

## 复杂可编程逻辑器件 complex programmable logic device

## CPLD

一种可通过软件设计实现硬件应用电路的器件。

注：通常把基于乘积项技术的可编程器件称为 CPLD。

### 2. 031

## 知识产权核 intellectual property (core)

## I P

具有知识产权的功能应用模块。

注：IP 可以是软件模块，也可以是硬件模块。

### 2. 032

## 固件 firmware

被写入非易失程序存储器内的专用软件。

### 2. 033

## 硬件描述语言 hardware description language

## HDL

一种进行硬件设计的软件语言。

### 2. 034

## 哈佛结构 Harvard architecture

一种程序和数据使用不同存储器的计算机体系结构。

### 2. 035

## 多核结构 multi-core

在一个芯片上集成多个 CPU 核。

注：在一个半导体芯片集成有多个微处理器、微控制器、数字信号处理器核，或多个不同用途的 CPU 核。

### 2. 036

## 寻址方式 addressing mode

计算机指令系统查找操作数地址的方式。有立即寻址、直接寻址、间接寻址、变址寻址等多种方式。

### 2. 037

## 寻址范围 addressing range

计算机指令系统所能查找的地址范围。由地址线的位数和寻址方式决定。

### 2. 038

## 数据总线 data bus

用于计算机内部或计算机之间数据交换的总线。

### 2. 039

## 地址总线 address bus

用于计算机内部或计算机之间地址交换的总线。

### 2. 040

## 控制总线 control bus

用于计算机内部或计算机之间控制信号通信的总线。

### 2. 041

## 系统总线 system bus

计算机数据、地址和控制总线的集合。

### 2. 042

## 数据格式 data format

数据处理过程中对数据结构形式的要求。

### 2. 043

## 实时数据交换 real-time data exchange

## RTDX

面向实时系统的数据交换。

### 2. 044

## 百万 条指令每秒 million instructions per second

## MIPS

每秒钟运行百万条指令的次数。

注：常作为嵌入式系统内中央处理器的运行指标之一。

### 2. 045

## 百万 次运算每秒 million operations per second MOPS

每秒钟完成百万次基本运算的数目。

注：常作为嵌入式系统内中央处理器的运行指标之一。

### 2. 046

## 引导方式 boot mode

计算机运行的程序导入方式。

### 2. 047

## 单一引导方式 single boot

启动时，只具备一种程序导入的方式。
