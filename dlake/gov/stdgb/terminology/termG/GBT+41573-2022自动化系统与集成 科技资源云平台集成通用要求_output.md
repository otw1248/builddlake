# 自动化系统与集成 科技资源云平台集成通用要求

## 1 范围

本文件规定了科技资源云平台集成的基本要求，包括云平台的业务对象、业务对象应具备的业务能力、业务描述及流程、资源集成要求、科技资源的集成方式以及资源集成的技术指标等。

本文件适用于各科技资源云与相关平台之间的集成以及第三方部门/机构进行科技资源云平台培训与应用。

## 5 平台核心业务服务

### 5.1 业务对象

科技资源云平台核心的业务对象主要包括服务供应方与服务获取方（即用户），其中服务供应方主要包括平台运营商、资源供应商和服务供应商；用户是服务的需求者和使用者。业务对象应包括以下4类：

a）资源供应商：科技资源云平台资源的第三方接口提供者以及科技资源云平台注册的科技资源发布者。科技资源的发布者可以是个人、企业、高校或研究院所等；

b）平台运营商：科技资源云平台的运营者及科技资源云平台对科技资源服务的聚合者；

c）服务供应商：科技资源云平台运行维持所需应用服务的直接提供者；

d）用户：科技资源云平台的使用者，所有在云平台注册成为注册会员的法人、自然人及其他组织，即科技服务的需求者和使用者。

服务供应方应针对不同领域、不同数据类型的资源特点，建立不同层次、不同类别的科技服务供用户使用。用户从供应方获取服务后，应将服务信息反馈给服务供应方。业务对象关系见图1。

科技资源云平台的业务应包括知识产权服务、技术转移服务、创业孵化服务、科技咨询服务、科技金融服务、科学技术普及服务、研究开发服务、检验检测认证服务。

### 5.2 业务能力

#### 5.2.1 资源供应商业务能力要求

资源供应商应具备但不限于以下业务能力：

a）所在专业领域较全面的技术资源；

b）能构建所在专业领域的知识体系；

c) 资源信息建模和信息加工能力；

d）能紧跟所在专业领域技术发展并及时更新资源；

e）判别所在专业领域资源权威性和专业性的能力。

#### 5.2.2 平台运营商业务能力要求

平台运营商应具备但不限于以下业务能力：

a）云计算基础设施维护能力；

b) 计算机系统运维基本能力；

c) 根据平台运行情况，提出平台升级改进方案的能力；

d）与资源供应商、服务供应商等建立协作的能力；

e）平台推广及服务能力；

f）根据用户需求指定恰当市场策略的能力。

#### 5.2.3 服务供应商业务能力要求

服务供应商应具备但不限于以下业务能力：

a）技术服务相关业务的专业人员；

b）分析、解剖技术需求并制定集成方案的能力；

c） 按平台规定流程执行专业化服务的能力；

d）审查、制定科技服务业务合同，约定知识产权权属，制定科技服务业务管理制度；

e) 组织协调各协作单位或技术供方开展技术集成工作的能力；

f) 利用和管理科技服务业务信息，制定和实施企业科技服务相关战略的能力。

#### 5.2.4 用户业务能力要求

用户应具备但不限于以下业务能力：

a）理解自身业务需求的能力；

b) 根据自身业务需求搜索适当的资源和服务供应商的能力；

c) 在线支付能力；

d）对所获得服务进行评价的能力。

### 5.3 业务描述

#### 5.3.1 知识产权服务

<div style="text-align: center;"><img src="imgs/img_in_image_box_627_1196_668_1235.jpg" alt="Image" width="3%" /></div>


知识产权服务指针对专利、商标、版权、著作权等科技成果进行代理、转让、评估、认证、咨询、检索等活动。

#### 5.3.2 技术转移服务

技术转移服务应符合 GB/T 34670—2017 中第 3 章的规定。

#### 5.3.3 创业孵化服务

创业孵化服务指针对初始创业者的场地、空间、政策等需求，提供共享服务空间、经营场地、政策指导、技术鉴定、咨询策划、项目顾问、人才培训等技术活动。

#### 5.3.4 科技咨询服务

科技咨询服务指针对客户的需求进行调研、分析、研究、预测，由具有社会科学专业知识并熟悉咨询业务的专家，综合利用科学知识、技术、经验、信息，采用现代科学方法和先进手段，为政府部门、企事业单位和各类社会组织及各阶层客户的决策、运作提供专业化的技术咨询活动。

#### 5.3.5 科技金融服务

科技金融服务指针对初创期到成熟期各发展阶段的科技企业的资金以及政策支持需求，通过创新财政科技投入方式，引导金融机构及创业投资提供融资支持和金融服务的系统活动。

#### 5.3.6 科学技术普及服务

科学技术普及服务指针对专业的领域由高层次科普专门人才进行普及科学知识、倡导科学方法、传播科学思想、弘扬科学精神的活动。

#### 5.3.7 研究开发服务

研究开发服务指针对各种组织机构的产品研发需求，为其提供获得科学技术新知识，创造性运用科学技术新知识，或实质性改进技术、产品、服务而持续进行的具有明确目标的系统活动。

#### 5.3.8 检验检测认证服务

检验检测认证服务指针对各种组织机构对产品或者法律法规的特定对象进行测试的需求，依据相关标准或技术规范，利用仪器设备、环境设施等技术条件和专业技能，对特定对象进行测试，并出具认证文件的活动。

### 5.4 云平台业务流程

#### 5.4.1 知识产权服务流程

科技资源云平台应具有知识产权类的服务功能。知识产权服务主要包括知识产权供应者注册、平台审核、用户匹配、服务进行和用户评价等。知识产权服务应符合以下流程：

a）提供知识产权服务的机构或者个人填写申请表（申请表应包含服务内容、服务资质与服务价值的列项），提交平台进行审核；

b）平台审核通过后，确定发布；

c）用户通过检索查看到机构或者个人的服务内容、服务资质等基本信息，筛选到自己需要的信息，在平台上确认，生成交易信息，并进行付款；

d）用户付款成功后，生成有效订单，平台将知识产权服务商的联系方式推送给用户；

e）用户和知识产权供应商完成交易后，用户对知识产权供应商的服务进行评价反馈，平台将评价信息添加到知识产权服务者的服务信息中。

#### 5.4.2 技术转移服务流程

科技资源云平台应具有技术转移类的服务功能。技术转移服务主要包括技术成果材料提交、平台评价、成果发布、用户匹配、订单交易和用户评价等。技术转移服务应符合以下流程：

a）技术转移资源供应商在平台上进行注册，提交技术评价材料；

b）平台根据材料确定评价价值，生成评价订单；

c）技术转移资源供应商付费，签订委托评价协议；

d）平台遴选专家，召开技术成果评价会，做出成果评价结论，出具成果评价报告；

e）技术转移资源供应商确认技术成果价值，发布成果；

f) 用户通过检索查看到技术转移的大纲内容以及价值信息，筛选到自己需要的技术转移，在平台上确认，生成交易信息，并进行付款；

g） 用户付款成功后，平台将技术成果的内容发送给用户；

h）技术转移资源供应商和用户完成交易后，用户对供应商的服务进行评价反馈，平台将评价信息添加到技术转移资源供应商的服务信息中。

#### 5.4.3 创业孵化服务流程

科技资源云平台应具有创业孵化类的服务功能，创业孵化服务主要包括机构注册、平台审核、用户匹配、机构定价、订单交易和用户评价等。创业孵化服务应符合以下流程：

a）具有社会科学专业知识并熟悉咨询业务的机构填写平台注册表（平台注册表应包含机构创业孵化的方向、内容以及可证明其业务水平的列项），提交平台进行审核；

b）平台审核通过后，发布机构的相关信息，供用户检索；

c） 用户通过检索查看机构的创业孵化专长、业务水平等信息，匹配到自己需要的机构；

d） 用户提供需要咨询的内容，机构确定咨询价值；

e） 用户在平台上确认，生成交易信息，并进行付款；

f）用户付款成功后，生成有效订单，平台将机构的联系方式推送给用户；

g）用户和机构完成交易后，用户对机构的服务进行评价反馈，平台将评价信息添加到机构的服务信息中。

#### 5.4.4 科技咨询服务流程

科技资源云平台应具有科技咨询类的服务功能，科技咨询服务主要包括专家注册、平台审核、用户匹配、专家定价、订单交易和用户评价等。科技咨询服务应符合以下流程：

a）具有社会科学专业知识并熟悉咨询业务的专家填写专家表（专家表应包含本人业务专长以及可证明其业务水平的列项），提交平台进行审核；

b）平台审核通过后，确定专家库；

c）用户通过检索查看专家的业务专长、业务水平等信息，筛选到自己需要的专家；

d） 用户提供需要咨询的内容，专家确定咨询价值；

e） 用户在平台上确认，生成交易信息并进行付款；

f）用户付款成功后，生成有效订单，平台将专家的联系方式推送给用户；

g）用户和专家完成交易后，用户对专家的服务进行评价反馈，平台将评价信息添加到专家的服务信息中。

#### 5.4.5 科技金融服务流程

科技资源云平台应具有科技金融类的服务功能，科技金融服务主要包括机构注册、平台审核、创业者匹配、合同签订和双方互评价等。科技金融服务应符合以下流程：

a）具有融资资质的银行、证券、保险机构或创业投资填写基本信息表（基本信息表应包含融资额度以及可证明其业务水平的列项），提交平台进行审核；

b）平台审核通过后，确定融资库；

c) 创业者通过检索查看融资公司的融资范围、融资产品和融资要求等信息，筛选到自己需要的融资机构；

d） 用户提供需要融资的科技内容以及融资额；

e）机构确定进行融资，签订合同；

f) 项目完成后，创业者对机构服务进行评价反馈，机构对创业者的信誉度进行评价，平台将评价信息添加到创业者和机构的项目信息中。

#### 5.4.6 科学技术普及服务流程

科技资源云平台应具有科学技术普及类的服务功能，科学技术普及服务主要包括专家注册、平台审核、用户匹配、专家定价、订单交易和用户评价等。科学技术普及服务应符合以下流程：

a）具有社会科学专业知识的专家或者机构填写基本信息表（基本信息表应包含业务专长以及可证明其业务水平的列项），提交平台进行审核；

b）平台审核通过后，发布专家库或者机构库；

c) 专家或者机构录制自己科学技术普及的多媒体文件，描写概述并上传到平台，进行定价；

d）用户通过检索查看专家或者机构的多媒体文件概述，筛选到自己需要的文件；

e）用户在平台上确认交易价值，生成交易信息，并进行付款；

f) 用户付款成功后，生成有效订单，平台将多媒体文件的播放权下发给用户；

g）用户享受服务后，对专家或者机构的服务内容等进行评价反馈，平台将评价信息添加到专家或者机构的服务信息中。

#### 5.4.7 研究开发服务流程

科技资源云平台应具有研究开发类的服务功能，研究开发服务主要包括机构注册、平台审核、用户匹配、机构团体定价、订单交易和用户评价等。研究开发服务应符合以下流程：

a）具有社会认可的机构或团体填写基本信息表（基本信息表应包含机构业务范围以及可证明其业务水平的列项），提交平台进行审核；

b）平台审核通过后，机构或团体进入研究开发机构库；

c) 用户通过检索查看机构或团体的业务专长、业务水平、服务案例等信息，筛选到自己需要的机构；

d）用户提供自己的需求，机构或团体确定服务价值；

e） 用户在平台上确认，生成交易信息，并进行付款；

f）用户付款成功后，生成有效订单，平台将机构的联系方式推送给用户；

g）用户和机构完成交易后，用户对机构或团体的服务进行评价反馈，平台将评价信息添加到机构或团体的服务信息中。

#### 5.4.8 检验检测认证服务流程

科技资源云平台应具有检验检测认证类的服务功能，检验检测认证服务主要包括机构注册、平台审核、用户匹配、机构定价、订单交易和用户评价等。检验检测认证服务应符合以下流程：

a）具有社会认可的机构填写基本信息表（基本信息表应包含机构业务范围以及可证明其业务水平的列项），提交平台进行审核；

b）平台审核通过后，机构进入检验检测机构库；

c） 用户通过检索查看机构的业务专长、业务水平和服务案例等信息，筛选到自己需要的机构；

d）用户提供需要检验检测的内容，机构确定认证价值；

e） 用户在平台上确认，生成交易信息，并进行付款；

f）用户付款成功后，生成有效订单，平台将机构的联系方式推送给用户；

g）用户和机构完成交易后，用户对机构的服务进行评价反馈，平台将评价信息添加到机构的检验检测信息中。

## 6 科技资源云平台集成

### 6.1 科技资源云平台集成要求

科技资源云平台集成应具备将资源供应商提供的各类科技资源按照标准的数据格式进行交换和集成的功能，统一对这些资源进行管理，并对用户形成统一的服务窗口，提供各类资源服务，而用户也能通过平台对获取的资源及服务进行反馈。

### 6.2 科技资源集成方式

#### 6.2.1 概述

科技资源集成相关方应达成集成协议，并对相关接口使用进行授权。科技资源各单位应开放和发

布自己的 API, 使用方可使用约定的技术(如 SOAP)到特定地址进行访问 API, 接口通过参数用户名和密码进行鉴权。

#### 6.2.2 采用 Web 服务方式实现集成

各资源平台宜将需要集成的资源编制成为 Web 服务，同时说明各服务的参数及数据格式，宜使用标准协议 HTTP、XML、SOAP 和 WSDL。各资源平台宜编制 Web 服务单位记录服务日志，作为子平台单位之间的协作凭证。采用这种方式的目的是兼容各资源云服务已有的一些 Web 服务。

#### 6.2.3 采用 WCF 方式实现集成

<div style="text-align: center;"><img src="imgs/img_in_image_box_763_436_802_471.jpg" alt="Image" width="3%" /></div>


各资源平台宜将需要集成的资源编制成为 WCF 服务契约，同时应说明数据结构的数据契约、消息契约和异常契约，供各子平台进行调用。各资源平台宜编制 WCF 服务单位记录服务日志，作为科技资源云平台和子平台之间的协作凭证。宜使用这种方式进行科技资源之间的集成。

#### 6.2.4 采用 ESB 方式实现集成

建立科技资源云平台 ESB，建成科技资源云平台各单位可采用的、基于标准的、集成各类科技资源的中枢神经系统。该系统应包括 5 个方面：

a）基于标准的面向消息中间件(MOM)架构；

b）基于标准的互连，如 Web 服务、J2EE 和 .NET 适配器；

c）基于标准的数据转换引擎（即 XSLT 和 Xquery）；

d) 应用部署的 SOA 方式；

e）基于标准的安全性（即 LDAP 和 SSL）。

科技资源云平台 ESB 应支持多语言和多平台开发。

#### 6.2.5 采用 OpenAPI 方式实现集成

建立科技云平台企业服务 API 中心，将科技资源进行整合，基于 OpenAPI3.0 标准技术体系，对外发布标准的 REST API，对内连接多种后端业务系统，并支持后端服务的路由和编排，有效屏蔽科技资源的复杂性，对外提供统一的、粗粒度的标准服务，建成科技资源云平台各单位可采用的系统，应包括以下 4 个方面：

a） API 网关架构；

b）统一集成开发环境及框架；

c）标准科技资源后端适配器定义；

d）科技资源数据映射规则设计。

科技资源云平台 API 中心应遵循前后端分离架构设计方法，提供科技资源服务封装、发布及集成能力，将企业的内部数据、业务逻辑，以标准服务方式快速、安全、可靠的开放出来，实现与其他应用微服务、界面展现、异构系统的集成操作。

### 6.3 资源集成的技术指标

#### 6.3.1 概述

资源的数据格式、描述语言、标引语言等都应标准化，而且也应运用一定的技术手段和方法，使资源得到合理组合，取得最好的组织结构和组织功能。资源集成应满足资源完整性、资源准确性、资源可靠性、资源易用性和资源一致性等技术指标。

#### 6.3.2 资源完整性

科技资源云平台应具有科技资源在其传输、存储、处理和交换的过程中始终保持不被修改、不被破坏和不被丢失的特性，且用户能够对平台发送或接收的科技资源的原样性进行验证。

#### 6.3.3 资源准确性

科技资源云平台所提供的科技资源应反映出其所指信息的真实性、客观性、权威性，应避免不符合事实的低价值资源给使用者带来负面影响。

#### 6.3.4 资源可靠性

科技资源在云平台上同用户的交互过程中，应确保科技资源提供者以及科技资源本身信息的真实可靠性，即所有资源的提供者都不应否认在信息原样性上所做的承诺。

#### 6.3.5 资源易用性

在指定条件下使用时，科技资源云平台所展示、提供的资源及资源模型应具有易理解、易融合以及便于用户使用的能力。

#### 6.3.6 资源一致性

科技资源在云平台上同用户以及其他资源副本、资源实例交互的过程中，应保证资源内容信息、资源表示状态、资源含义的最终一致性。
