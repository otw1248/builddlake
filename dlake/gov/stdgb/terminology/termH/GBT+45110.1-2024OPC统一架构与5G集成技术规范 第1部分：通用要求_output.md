## 引言

第五代移动通信技术(简称：5G)具有大带宽、广联接和低时延的特性，基于5G的工业无线通信能满足工业场景中数目众多的进行柔性组网的通信需求。例如，制造商通过5G将制造装备和其他机器、工具、工厂和仓库中的资产连接起来，或将信息系统与各种设备相连，为相关各方提供更丰富的可操作、可追溯、可定位的信息，助力工业向数字化、信息化和智能化的转型。

5G作为新兴技术，缺少应用层语义交互的统一描述语言，OPC统一架构和5G无线通信技术的融合应用，在应用层提供了5G的使用规范，提高了5G的易用性和交互能力，是实现5G工业设备集成互联互通、语义互操作、柔性可重构等关键技术的重要支撑，赋能工业应用场景更多的可能性。

本文件是指导 OPC 统一架构与工业 5G 网络集成应用的通用技术标准，旨在提出 OPC 统一架构与工业 5G 网络集成应用的通用架构及基本要求，为工业用户、系统集成商、系统供应商、运营商等提供应用指导，简化集成复杂度。GB/T 45110《OPC 统一架构与 5G 集成技术规范》拟由五个部分构成。

——第1部分：通用要求。目的在于规定OPC统一架构与工业5G网络集成应用的总体要求、连接管理集成要求、数据传输集成要求、网络管理集成要求、集成应用模型要求和集成安全要求，为工业用户、系统集成商、系统供应商和网络提供商等提供OPC统一架构与工业5G网络集成应用的指导。

——第 2 部分: 应用规范。目的在于规定 OPC 统一架构与工业 5G 网络集成应用中连接管理、数据传输和网络管理等应用模型, 支持 OPC 统一架构与工业 5G 网络的快速集成应用。

——第3部分:管理规范。目的在于规定OPC统一架构与工业5G网络集成应用中运维管理、网络连接管理等要求，支持OPC统一架构和工业5G网络的集成适配管理。

——第4部分：测试规范。目的在于规定OPC统一架构与工业5G网络集成应用中连接管理、数据传输和网络管理等应用模型的一致性，应用规范、管理规范各项要求的符合性。

——第 5 部分: 安全规范。目的在于规定 OPC 统一架构与工业 5G 网络集成应用中业务运行和管理的安全要求。

## 5 集成总体要求

### 5.1 物理架构

OPC UA 和工业 5G 网络集成的物理架构如图 1 所示。其中，UA 有线设备是支持 OPC UA 应用并通过工业有线网络连接的工业设备，UA-5G 设备是支持 OPC UA 应用并通过工业 5G 网络连接的工业设备，可工作于工厂中企业层、车间层、监视控制层、基本控制层和设备层。OPC UA 和工业 5G 网络集成支持以下通信方式：

——UA-5G 设备与 UA 有线设备经过工业 5G 网络和工业有线网络进行通信；

——UA-5G 设备之间经过工业 5G 网络进行通信。

### 5.2 集成架构

OPC UA 和 5G 集成架构如图 2 所示，OPC UA 功能应用应包括连接管理、应用数据传输和网络管理，5G 系统包括控制面、用户面和管理面，工业 5G 网络访问子层（简称“访问子层”）包括连接管理单元、数据传输单元和网络管理单元，实现 OPC UA 功能应用和工业 5G 网络的集成适配。OPC UA 和工业 5G 网络集成包括如下。

——连接管理集成：OPC UA 连接管理通过访问子层的连接管理单元与 5G 控制面共同实现连接的创建和释放等，5G 控制面是工业 5G 网络内部对数据传输通道进行控制的逻辑功能集合。

——数据传输集成：OPC UA 数据传输通过访问子层的数据传输单元与 5G 用户面共同实现 OPC UA 应用数据传输，支持 OPC UA C/S 和 PubSub 两种数据传输模式，5G 用户面是 5G 系统内部数据传输的逻辑功能集合。

——网络管理集成：OPC UA 网络管理通过访问子层的网络管理单元与 5G 管理面共同实现工业 5G 的网络管理，包括建立工业 5G 网络的信息模型等，5G 管理面是管理 5G 系统的逻辑功能集合。

### 5.3 集成流程

OPC UA 和 5G 集成流程如图 3 所示，包括如下。

——UA-5G 设备配置: 完成此设备的输入/输出数据、优先级、通信周期等参数的配置。

——连接建立：将 UA-5G 设备网络相关配置传给 5G 网络控制面，建立工业 5G 网络连接，可根据工业 5G 网络能力对 UA-5G 设备配置参数进行调整，并同步更新至 OPC UA 连接管理器。

——应用数据传输：基于工业5G网络连接，UA-5G设备按照优先级、通信周期等参数要求进行应用数据传输。OPC UA连接管理器可根据应用需要对OPC UA连接进行添加或删除操作。

——系统管理：基于 OPC UA 建立的 5G 系统模型，监视网络拓扑、网络状态、网络能力等，并告知 5G 系统以进行配置。

### 6.2 连接建立

访问子层的连接管理单元应支持基于 OPC UA 的 C/S 和 PubSub 等通信模式获取应用需求，应用需求可包括发送周期、数据包大小、数据包内冗余量（即数据包 padding 的位数）、传输时延、时延抖动、时间同步精度、生存时间、数据包到达时刻和通信可用性等。

访问子层的连接管理单元宜支持获取 OPC UA 应用数据包到达时刻列表，工业 5G 网络应在数据包到达时刻列表中选择期望的数据包到达时刻。

访问子层的连接管理单元宜支持获取 OPC UA 应用连接释放超时时间，并发送给工业 5G 网络。

访问子层的连接管理单元宜支持获取工业 5G 网络期望的数据包到达时刻，并发送给 OPC UA 应用。

访问子层的连接管理单元宜支持将 OPC UA 应用的需求映射为工业 5G 网络的传输参数，工业 5G 网络的传输参数可包括 5QI、ARP、AMBR、GFBR 等。

访问子层的连接管理单元应支持将 OPC UA 应用需求或映射后的工业 5G 网络传输参数发送给工业 5G 网络，该发送的接口宜为 RESTful、NETCONF。

附录 A 给出了 OPC UA 应用连接管理集成流程示例。

### 6.3 连接释放

访问子层的连接管理单元应支持获取 OPC UA 应用的连接释放通知，并将 OPC UA 连接释放通知发送给工业 5G 网络。

访问子层的连接管理单元宜支持将 OPC UA 应用连接释放异常（如 OPC UA 应用发送数据间隔超过连接释放超时时间等）发送给工业 5G 网络。

### 6.4 时间同步

访问子层的连接管理单元宜支持 OPC UA 与工业 5G 网络的时间同步功能，可在工业 5G 网络的覆盖范围内提供统一的高精度时钟源。

### 7.2 数据传输映射

访问子层的数据传输单元应支持在 OPC UA 应用和工业 5G 网络之间交互 IP 报文或 Ethernet 帧，OPC UA 与 5G 数据传输集成模型如图 6 所示，包括：

——在 C/S 和 PubSub 通信模式下，OPC UA 应用数据可封装到 IP 报文中；

——在 PubSub 通信模式下，OPC UA 应用数据可封装到 Ethernet 帧中。

### 7.3 数据传输状态获取

访问子层的数据传输单元宜支持获取 OPC UA 应用数据包的剩余生存时间并发送给工业 5G 网络。

访问子层的数据传输单元宜在 OPC UA 应用数据包的生存时间小于一定门限时将业务数据包的生存时间发送给工业 5G 网络。

### 7.4 数据包冗余识别

访问子层的数据传输单元可支持指示填充比特数大小。

访问子层的数据传输单元可支持指示和前一数据包载荷相同。

### 7.5 数据订阅分组

访问子层的数据传输单元应支持获取 UA-5G 设备或 UA 有线设备增加、移除组播组（Multicast Group）的信息并发送给工业 5G 网络。

工业 5G 网络宜支持 PIM、IGMP 和 IGMP snooping 协议。

### 8.2 工业 5G 网络运维管理

工业 5G 网络运维管理包括以下规定。

——访问子层的网络管理单元宜支持获取工业5G网络的服务能力：

SIM 卡的号码及激活状态信息查询；

● 计费信息查询。

——访问子层的网络管理单元宜支持获取工业5G网络的运维管理能力：

- 网络拓扑展示能力，拓扑信息包括工业5G网络的组网连接信息及网络的基础信息，宜包含UA-5G设备地址、UA-5G设备状态（离线、在线）、UA-5G设备分布、小区数量、小区状态等；

- 网络性能展示能力，宜包含网元 CPU 及内存，小区级或切片级的端口流量流速、上行端口速率、下行端口速率、负荷、用户数、发射功率、上下行吞吐量、接入成功率、掉话率、在线用户数、拥塞情况等；

网络告警展示能力，宜包含网元的宕机、负荷过高、CPU负载过高、瞬时接入业务数突增告警、被攻击告警、小区负载过高、切换异常告警、异常掉话告警、时钟失步等；

UA-5G 设备信息展示能力，宜包含 UA-5G 设备的位置、在网时长、累计流量、群组信息、性能指标（时延、抖动、通信可靠性等）、信号质量（RSRP、SINR 等）、定界/定位的故障信息等。

——宜支持非 UA-5G 设备信息展示。

### 8.3 工业 5G 网络连接管理

访问子层的网络管理单元宜支持获取工业5G网络链路的QoS信息，包括：

——逻辑链路拓扑信息，包括业务传输路径上的各网元的逻辑连接；

——逻辑链路 QoS 信息，包括时延、拥塞、上下行码流、告警信息等。

## 9 集成应用模型要求

OPC UA 与 5G 的集成应用模型, 实现 5G 系统的透明化管理和应用, 应满足以下要求:

——支持连接管理信息模型、数据传输信息模型、网络管理信息模型等；

——支持层级、并列、引用等方式组织模型的应用；

——支持5G参数在线监视、修改等；

——支持分级管理、权限设置等。

## 10 集成安全要求

OPC UA 与工业 5G 网络集成应满足信息安全、接入安全等要求。

#### A.2.3 连接建立过程

连接建立包含以下过程。

——工业5G网络应用需求获取，即AF获取PLC A设备和PLC B设备的应用需求。获取服务包含：

● 通信特征查询: AF 分别查询 PLC A 设备和 PLC B 设备的通信关系;

- 通信特征查询响应：PLC A 设备和 PLC B 设备分别返回通信关系参数，包括发送周期、数据包大小、数据包内冗余量（即数据包 padding 的位数）、传输时延、生存时间等。

——OPC UA 应用需求确定，即根据通信模式的不同，OPC UA 应用向 AF 的访问子层的连接管理单元提供不同应用需求描述信息：

- A 类: 基于 TCP/IP 的 C/S 通信模式, 基于 UDP/IP 的 PubSub 通信模式, 适用于对实时性、确定性不高的应用场景, 应用需求描述信息包含: 数据包大小、传输时延、时延抖动;

- B 类: 基于 Ethernet 的 PubSub 通信模式, 适用于对实时性、确定性要求较高的应用场景, 应用需求描述信息包含: 发送周期、数据包内冗余量(即数据包 padding 的位数)、应用数据包期望发送时刻列表、时间同步精度、生存时间和传输可用性等。

——OPC UA 应用需求配置，即 AF 的连接管理单元在获取 B 类 OPC UA 应用数据包期望发送时刻列表后，对小区内所有 B 类 OPC UA 应用的访问子层发送分配的应用数据包发送时刻表。B 类 OPC UA 应用按照分配的应用数据包发送时刻表，发送应用数据。

——OPC UA 应用传输通道建立，即 AF 将获取的应用需求信息发送给工业 5G 网络，该信息发送接口可为 RESTCONF 或 NETCONF 接口。基于该信息工业 5G 网络为 PLC A 设备和 PLC B 设备的应用数据传输准备传输资源，并基于 3GPP 规定的传输承载建立流程，为 PLC A 设备和 PLC B 设备的应用数据传输建立通道。

#### A.2.4 应用数据传输

基于建立的传输通道，PLC A 设备和 PLC B 设备进行数据传输。在应用数据传输过程中，如果 PLC A 设备和 PLC B 设备的应用需求有变化，及时将变化信息发送到 AF，并传递给工业 5G 网络设备，以便工业 5G 网络设备能根据变化及时调整传输通道的参数，具体的参数包含如下。

——通信特征变化：PLC A 设备或 PLC B 设备将变化的应用需求参数发送给到工业 5G 网络。

数据传输状态获取: PLC A 设备或者 PLC B 设备监测到 OPC UA 应用数据包的生存时间小于设定值时, 将业务数据包的剩余生存时间发送给工业 5G 网络。
