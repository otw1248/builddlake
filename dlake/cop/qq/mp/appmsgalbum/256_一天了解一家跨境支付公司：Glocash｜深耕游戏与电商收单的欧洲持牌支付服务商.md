本文对跨境支付公司的界定：持支付牌照主营跨境收付款 / 换汇，或为该业务提供技术支撑的持牌服务商。

## **一、企业概况**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与主体**

2017 年成立，注册于立陶宛维尔纽斯，持牌主体为 UAB「Glocash Payment」。香港设有关联公司 Glocash Payment HK Limited。

**🌍****全球布局**

上海为运营总部，立陶宛维尔纽斯设有持牌实体办公室（EMI 牌照注册主体），香港设有持牌实体，覆盖欧洲、亚太区域。

**🛡️****合规资质**

  * 支付牌照：立陶宛银行 EMI 牌照（覆盖支付收单、电子货币发行、汇款等业务）；香港海关 MSO 牌照

  * 卡组织资质：Visa、Mastercard、American Express、Discover、银联国际、JCB 收单资质

  * 安全认证：PCI DSS Level 1

  * EEA 跨境服务覆盖：通过欧盟护照机制可覆盖 30 个 EEA 成员国


**📊****业务覆盖**

支持 200+ 国家/地区、100+ 币种，提供 300+ 支付方式，主要面向跨境电商独立站收单、游戏数字商品支付，同时覆盖在线教育、SaaS 订阅等场景。

## **二、计费模式概览**

**📋****计费项目清单**

国际卡收单：支持信用卡、借记卡、预付卡。

本地支付：欧洲本地银行转账、数字钱包等。

**📊****计费规则与费用说明**

采用混合计费模式（百分比 + 固定金额），提供两种定价模式：

  * 标准定价（Standard Pricing）：按需付费，不设月费或年费，适用于一般商户

  * 定制定价（Customized Pricing）：面向大交易量或特殊业务模式的企业，需联系销售团队




具体费率取决于行业类型、交易规模、支付方式、币种等因素，均通过商户与公司单独协议约定。可能包含货币转换费、退款处理费、拒付处理费等项目。

**📌****小结：** 本章聚焦收费项目与适用规则，不含数值，最终以《商户协议》为准。

## **三、支付网络与合规体系**

###  （一）支付网络与区域布局

**💳****国际卡收单**

接入 Visa、Mastercard、American Express、JCB、Discover、银联国际等国际卡组织，支持信用卡、借记卡、预付卡，主要结算货币包含 USD、EUR、GBP、JPY、AUD、CAD、SGD 等。

**🔗****本地支付**

覆盖多种本地支付类型：

  * 欧洲：iDEAL（荷兰）、Klarna Pay Now（原SOFORT）、Bancontact（比利时）、EPS（奥地利）、Paysafecard（约50个国家和地区）

  * 亚太：支付宝、微信支付

  * 国际钱包：Apple Pay、Google Pay


**📍****区域运营中心**

  * 欧洲：维尔纽斯（立陶宛），覆盖 EMI 牌照监管与 EEA 跨境服务

  * 亚太：上海（总部）、香港（MSO 持牌实体），覆盖支付宝、微信支付等亚太本地支付




### （二）合规与安全体系

**🏛️****牌照与认证**

  * 持有立陶宛 EMI 与香港 MSO 牌照

  * PCI DSS Level 1 认证

  * SWIFT BIC：GLPELT21


**📜****数据保护**

遵循 GDPR（《通用数据保护条例》）规范，已任命数据保护官（DPO），建立了 72 小时数据泄露通报流程，发布独立隐私政策与 Cookie 政策。

**💰****客户资金管理**

遵循 EMI 牌照下的客户资金保障要求，客户资金隔离存放。

**☁️****基础设施**

采用 AWS 云服务架构，部署网络安全防护措施。

## **四、技术接入与开发者工具**

###  （一）开发者工具

**⚙️****API 架构**

采用 RESTful API 架构，支持 HTTPS 协议传输，区分沙盒环境与生产环境，使用独立的接口地址与密钥，支持 liveMode 与 debug 模式切换。

**🔑****认证方式**

采用 API Key 认证，区分沙盒与生产密钥。

**📦****SDK 与插件**

  * 服务端 SDK：Java

  * 电商平台插件：WooCommerce、Shoplazza、OpenCart、ZenCart、Magento


**🔔****Webhook 事件通知**

支持 Webhook 回调通知，覆盖支付成功、支付失败、退款处理、争议创建等事件类型，提供签名验证机制。

### （二）支付接入方式

**托管式收银台（Hosted Checkout）** ：客户跳转至 Glocash 页面完成支付，降低商户 PCI 合规要求

**嵌入式收银台（Embedded）** ：将现成的 UI 组件嵌入商户网站，用户无需跳转

**支付 API** ：商户自建支付界面，完全通过 API 对接

**移动端 SDK** ：APP 内浮窗支付，支持指纹与面容认证

## **五、风控体系与游戏行业方案**

###  （一）风控引擎

Glocash Payment 的风控能力覆盖以下几个方面：

**🧬****生物识别验证**

覆盖设备指纹、面部识别等方式

**📊****行为模式分析**

基于用户交易行为（频率、金额分布等）进行风险判断

**🤖****机器学习模型**

实时风险评估与策略动态优化

**🗂️****综合云数据库**

整合白名单、黑名单等多维数据源，支持规则引擎与机器学习模型联动

**🔒****3D Secure 支持**

采用 Smart 3D Secure 2.0，基于机器学习进行风险评估，支持 SCA（强客户认证，Strong Customer Authentication）合规。

**⚠️********注意：** 欧洲区域受 PSD2 法规约束，强制 3DS 验证可能对支付转化率产生一定影响，具体幅度因市场和实施方式而异，可根据业务场景配置豁免策略（如小额订单、白名单客户等）降低影响。

### （二）游戏与数字商品行业方案

**🎮****服务组合**

国际卡收单 + 本地支付 + 数字钱包并行，支持按区域设置价格与结算货币

**🛡️****风控要点**

  * 虚拟商品/游戏场景：覆盖首充欺诈、批量盗刷、礼品卡与代充风险、跨区域异常支付识别

  * 设备指纹与行为分析联动，支持高频小额交易监控

  * 3DS 触发与豁免策略按风险分层配置




## **六、适用场景与选型建议**

###  适合使用 Glocash Payment 的场景

  * **游戏与数字商品出海企业** 如果您运营游戏虚拟物品交易、数字内容分发等业务，Glocash Payment 将游戏支付作为重点服务方向，提供针对性的风控规则与多支付方式并行方案。

  * **中国出海独立站商户** 如果您是中国出海的跨境电商企业，需要欧洲市场本地支付接入（iDEAL、Klarna Pay Now 等），Glocash Payment 在上海设有运营团队，支持中文沟通。

  * **电商平台插件用户** 如果您运营 WooCommerce、Shoplazza（店匠）等平台独立站，技术资源有限，Glocash Payment 提供多个电商平台插件，支持内嵌、收银台等多种接入模式，降低技术对接门槛。

  * **欧洲市场收单需求** 如果您的目标市场以欧洲为主，需要 EEA 区域合规收单能力，Glocash Payment 依托 EMI 牌照与欧盟护照机制覆盖 EEA 区域，支持多种欧洲主流本地支付方式。




### 决策建议

在选择跨境支付服务商时，建议从以下维度进行综合评估：

**📊****业务场景匹配度**

根据您的主要市场、客户支付习惯、交易规模等因素，对比多家服务商的适配情况。如果您的业务主要集中在欧洲市场，可同时了解 Stripe、Adyen 等方案；如果聚焦游戏与数字商品领域，可对比 Xsolla、PayPal 等方案。

**🛡️****合规与牌照覆盖**

确认服务商在您的目标市场持有相应支付牌照。Glocash Payment 的牌照覆盖以欧洲（EEA）与香港为主，如业务涉及北美、东南亚等区域，需确认服务商在相应国家或地区的合规资质。

**💰****费用与合同条款**

在签约前，务必与多家服务商沟通，明确计费规则、结算周期、服务边界、数据安全、争议处理等关键条款。同时建议重点确认到账时效（T+N 天数）、保证金比例等直接影响资金周转的条款，并评估自身团队的技术能力，选择合适的接入方式。
