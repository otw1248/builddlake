## **一、企业概况**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与主体**

2015 年 11 月于上海注册成立（上海寻汇信息科技有限公司），2016 年开始实际运营。

**🌍****全球布局**

全球业务总部（Global Business Headquarters）位于新加坡，财资中心（Treasury Center）位于香港，并在上海、伦敦、雅加达、东京、吉隆坡（计划中）等地设有运营中心与服务团队。

**🛡️****合规资质**

持有新加坡金融管理局（MAS）大型支付机构（MPI）与资本市场服务（CMS）双牌照、英国金融行为监管局（FCA）电子货币机构（EMI）牌照、中国人民银行（PBoC）支付业务许可证（2025 年 8 月收购传化支付获得）、美国 FinCEN MSB、中国香港 MSO 等 11 个司法管辖区的牌照及注册（含马来西亚原则性批准）。持有 Visa/Mastercard/UnionPay 主会员资格及 PCI-DSS Level 1 安全认证。

**📊****业务覆盖**

支付网络覆盖 **190+** 国家和地区，支持 **130+** 种支付货币与 **30+** 种收款货币。

⚠️ 说明： 因场景与合作方的定义、统计范围存在差异，具体覆盖范围以签约时点及合同清单为准。

**🎯****主要应用场景**

聚焦 B2B 跨境贸易、跨境电商（平台收款与供应商付款）、在线旅游（OTA）、海事航运、SaaS 等行业，提供国际支付、全球收款、商业卡发行、财资管理（TreasuryOS 遵循 ISO20022 标准）及外汇（FX）对冲解决方案。

**📍****主要覆盖市场**

亚太（新加坡、香港、中国大陆、印尼、马来西亚、日本）、欧洲（英国）、北美（美国、加拿大）、澳大利亚。

**💰****计费模式**

采用企业定制化报价模式，具体费率取决于客户的交易规模、业务场景、使用币种、服务层级和行业解决方案等因素，需联系销售团队获取。所有费用在系统发起交易前实时计算并展示（"Get Quote"功能）。不收取账户注册费或账户管理费。

作为 Visa、Mastercard 和 UnionPay 的主会员，直连卡组织网络，并集成 Visa Direct（支持向 **39 个国家/地区** 的 Visa 卡付款）。

**🔄****跨境清算路径**

支持 SWIFT、卡组织网络（Visa/Mastercard/UnionPay）以及主要市场的本地清算网络。

**👤****反洗钱能力（AML）**

对接全球制裁名单库，建立 KYC 质量保证（QA）框架，采用专有技术进行实时交易监控和可疑活动（SAR）检测。

**💰****客户资金管理**

严格遵循新加坡 MAS、英国 FCA 等多地监管要求，实行客户资金隔离（Safeguarding），客户资金独立存放于合作银行。

**📤****批量处理**

## **四、汇率管理与资金流转**

###  （一）汇率管理机制（CMS 牌照）

**📊****汇率基准与数据源**

对接多家银行与流动性提供商，提供接近银行间市场（mid-market rate）的实时汇率。

**🔄****换汇与对冲工具**

基于新加坡 MAS 资本市场服务（CMS）牌照，提供全面的企业级外汇风险管理工具：

  * 实时即期交易（Spot Trades）： 以实时市场汇率即时转换货币。

  * 远期合约（Forward Outright）： 锁定未来交易的汇率，规避汇率波动风险。

  * 无本金交割远期（NDFs）： 用于对冲受限制货币（如 CNY、IDR 等）的汇率风险。

  * 窗口远期（Window Forwards）： 在未来一个时间窗口（而非特定日期）内锁定汇率，提供更高灵活性。

  * FX 订单（FX Orders）： 汇率达到预定水平时自动执行转换。

  * 结构性产品（Structured Products）： 定制化对冲策略。




**💡****提示：远期合约可锁定未来汇率，规避波动风险，但无法享受有利汇率变动。**

###  （二）资金账户与调度

**🏦****账户结构（Global Collection）**

核心为 Multi-Currency Wallet（多货币钱包）结构，提供多币种全球收款账户，支持 **30+** 种货币收款，并可在 **10+** 主要货币（如 USD， EUR， GBP， AUD， SGD， HKD 等）下开设本地账户详情，实现本地化收款。

**🔄****资金池调度（TreasuryOS）**

提供财资管理系统（TreasuryOS），支持多币种资金池统一管理、自动化应收应付（AP/AR）、生成多语言发票及管理付款条款。支持资金归集、灵活调配，以及基于 API 的自动化资金调度。通过多货币账户可以实现收付款的双向抵消（offsetting flows），降低不必要的外汇转换成本。

**💡****应用场景：多平台卖家可将不同币种收款归集至多币种账户，统一管理并优化外汇转换时机，或直接用于支付供应商货款。**

## **六、区域合规要点与注意事项**

###  （一）新加坡

**🛡️****准入与合规要点**

持有 MAS 大型支付机构牌照（MPI），需遵循《支付服务法》(Payment Services Act) 下的客户资金保护（Safeguarding）要求，将客户资金独立存放于指定银行。同时，持有 MAS CMS 牌照，需遵循资本市场服务法规，提供外汇衍生品服务。

**📊****运营与风控重点**

严格遵守 MAS 的 AML/KYC 合规要求，持续进行交易监控和风险评估，需定期向 MAS 提交监管报告。

**📋****上线清单**

KYB/KYC 资料、隐私条款与数据处理协议、客户资金隔离账户设置。

### （二）欧洲（英国）

**🛡️****准入与合规要点**

需遵守英国 FCA EMI 监管要求，包括客户资金隔离（Safeguarding）和 UK GDPR（数据隐私保护）。需遵循 UK 版本的客户资金保护要求，并将客户资金独立存放于国际性银行。

**📊****运营与风控重点**

英国脱欧后，FCA 牌照不再被授权在欧洲经济区（EEA）成员国运营，业务主要集中在英国本土。需遵守 UK GDPR 和 FCA 的消费者责任（Consumer Duty）要求。

**📋****上线清单**

KYB/KYC 资料、隐私条款与数据处理协议、客户资金隔离账户设置。

### （三）北美（美国/加拿大）

**🛡️****准入与合规要点**

需符合美国 FinCEN MSB 和加拿大 FINTRAC FMSB 的注册与报告要求。需遵循《银行保密法》（BSA）和《爱国者法案》(USA PATRIOT Act) 下的反洗钱和反恐怖融资要求。必须对所有交易和客户进行 OFAC 制裁名单筛查。

**📊****运营与风控重点**

在各州运营可能需要额外的州级牌照，需确保业务符合联邦和州级法律要求。

**📋****上线清单**

主体与受益人信息（UBO）、KYC/KYB、AML 合规政策、数据安全评估。

### （四）东南亚（新加坡/印尼/马来西亚）

**🛡️****准入与合规要点**

需同时符合新加坡 MAS (MPI+CMS)、印尼央行、马来西亚央行（BNM）等多国监管，规则差异大。

**📊****运营与风控重点**

遵守各国本地数据保护法（如新加坡 PDPA）。

**📋****上线清单**

主体/KYB、各国本地合规承诺、钱包/账户直连转账测试、数据驻留与隐私政策。

### （五）中国大陆

**🛡️****准入与合规要点**

持有中国人民银行（PBoC）支付业务许可证（互联网支付业务），需遵守央行对非银行支付机构的严格监管，包括客户备付金集中存管、AML/CTF 要求、以及数据本地化存储要求。

**📊****运营与风控重点**

严格遵守中国国家外汇管理局（SAFE）的跨境支付与外汇管理规定；遵守《个人信息保护法》(PIPL) 的数据本地化与跨境传输要求。跨境资金流需遵循国家外汇管理局的外汇管理规定，所有境内支付业务必须符合中国法规。

**📋****上线清单**

主体/KYB、PBoC 与 SAFE 合规流程、数据本地化设置、CNH/CNY 支付路径测试。

**⚠️****区域合规关键要点**

**中国大陆**  
\- 数据与资金流转需符合外汇管理和数据本地化要求  
\- 建议：提前规划技术架构，确保数据存储与跨境传输合规

**新加坡**  
\- MAS CMS 牌照适用于资本市场产品，监管标准高于一般支付牌照  
\- 优势：获取后可开展更广泛的金融服务，但需配备相应风控体系

**英国/欧洲**  
\- FCA EMI 牌照服务范围限于英国，若需覆盖欧洲大陆需申请 EEA 通行证或当地牌照  
\- 提示：明确目标市场，避免跨区域合规风险

**北美**  
\- MSB 为联邦注册，部分州的汇款业务需额外州级牌照  
\- 重点：严格执行 OFAC 制裁筛查，确保反洗钱合规

运营| 熟悉帮助中心文档，确保新员工可快速掌握平台操作  
  
## **九、场景适配性分析**

###  （一）核心能力与场景匹配度

**🌐****场景 1：多区域合规要求**

这类场景常见于在新加坡、香港、中国大陆、英国、印尼、美国等多地同时运营，需满足各地客户资金隔离与 AML 监管要求的 B2B 企业。SUNRATE 在这些主要市场均持有本地牌照，满足各地监管标准；Airwallex 在亚太市场具有较强本地化能力，Stripe 在全球市场均有覆盖但欧美市场能力更为成熟。

**💱****场景 2：外汇风险管理**

典型客户为大型贸易商或集团公司，有大额跨境贸易业务，需要管理未来 3-12 个月的汇率波动风险。SUNRATE 基于新加坡 MAS 资本市场服务（CMS）牌照，提供远期合约、无本金交割远期（NDFs）、窗口远期等专业外汇对冲工具。大多数支付机构（如 PayPal、Payoneer）主要提供即期汇兑服务，少数平台（如 Wise）提供汇率预警功能。需要注意的是，远期合约和 NDFs 等衍生品工具需要持有特定的金融监管牌照才能提供，衍生品交易也需要企业具备相应的财务专业能力，不当使用可能带来金融风险。

**🏭****场景 3：行业定制化需求**

典型行业包括在线旅游（OTA）和海事航运，企业面临向全球酒店、航司供应商付款，或需要发放船员工资等复杂场景。SUNRATE 提供定制化虚拟卡和行业解决方案，如船员卡支持 ATM 取现、卡对卡转账等功能；通用支付平台一般专注于通用场景，在这类垂直行业的定制化能力上相对有限。

**🔄****场景 4：B2B 高频跨境支付**

跨境电商品牌或平台卖家通常需要从亚马逊、TikTok、SHEIN 等多平台收款，并高频向中国供应商批量支付人民币（CNH/CNY）货款。SUNRATE 支持向 1000+ 收款方批量支付，付款最快当日到账，最慢 T+1，覆盖收付两端资金流；PingPong 在电商平台回款方面起步较早且能力成熟，供应商付款业务近年也在快速发展。在中国供应商 CNY 付款方面，SUNRATE 和 PingPong 均持有中国人民银行颁发的支付业务许可证，本地清算能力处于同一监管框架下。

**🏢****场景 5：集团化财资管理**

典型特征是企业拥有多个海外分公司，需要进行频繁且大额的资金调拨，并需要 TreasuryOS 系统自动化处理应收应付账款。SUNRATE 提供统一的财资管理平台和多币种全球收款账户，支持资金的归集与调拨。Wise Business、Revolut Business 等平台虽提供企业账户和 API 服务，但其产品设计主要面向中小型企业，在多主体资金归集、自动化应收应付管理、多币种财资池调度等集团化财资管理的深度和广度上，与 SUNRATE TreasuryOS 这类专注于大型企业财资管理的系统存在定位差异。

**⚠️****风险提示：多货币账户可能产生额外的外汇敞口风险。**

###  （二）决策建议

在选择跨境支付服务商时，建议从以下维度进行综合评估：

**🎯****业务场景匹配度**

根据主要市场定位、行业特性、外汇管理需求等因素，对比多家服务商的能力侧重点。不同平台在特定场景的能力各有差异：

**📍****市场定位方面**

  * B2B 贸易/批发场景可对比 SUNRATE、Airwallex、Currenxie 等

  * B2C 电商/零售场景可对比 Stripe、Adyen

  * 混合模式企业可根据交易量占比选择主平台


**🏭****行业特性方面**

  * 跨境电商（多平台收款）场景可对比 SUNRATE、PingPong

  * SaaS 订阅计费场景可对比 Stripe Billing、Chargebee

  * OTA/海事行业可重点考察 SUNRATE 的行业定制方案

  * 传统贸易场景可对比 SUNRATE、Airwallex、Currenxie 等


**💱****外汇管理需求方面**

  * 仅需即期换汇的企业，所有主流平台均支持此功能

  * 需锁定远期汇率以规避汇率风险的企业，SUNRATE 基于 CMS 牌照提供此类对冲工具

  * 单币种交易且无外汇需求的企业，PayPal、Payoneer 等基础方案即可满足，但需注意部分平台可能存在货币转换的隐性费用


**🔧****技术集成能力方面**

  * 有开发团队的企业可优先考虑 API 接口丰富的平台（如 Stripe、SUNRATE）

  * 无开发资源的企业可优先考虑无需开发即可使用功能的平台（如 PayPal、Payoneer）

  * 需要注意的是，Stripe 基础文档完全公开，部分高级功能需注册查看；Adyen 需注册后访问，但文档详尽且示例丰富，便于评估


**📈****业务规模阶段**

不同发展阶段的企业对支付服务商的需求有所不同：

  * **初创期企业** （月交易额 < 10 万美元）通常优先考虑开户门槛低、快速开户、功能简洁的基础方案，如 PayPal、Payoneer

  * **成长期企业** （月交易额 10-100 万美元）开始需要多平台收款与基础合规能力，可对比 PingPong、Payoneer 等方案

  * **扩张期企业** （月交易额 100-1000 万美元）对 B2B 支付、财资管理和定制化报价有明确需求，可对比 SUNRATE、Airwallex、Stripe 等方案

  * **成熟期企业** （月交易额 > 1000 万美元）通常需要财资管理、外汇对冲、专属客户服务等高级功能，可考虑 SUNRATE 或银行直连方案




建议企业在业务规模达到一定体量后，再评估切换到定制化报价平台的成本与收益。外汇对冲工具（如远期合约、NDFs）通常需要企业具备相应的财务专业能力。

**⚙️****技术实施能力**

不同功能类型的技术要求与实施周期有所不同：

  * **平台收款（无代码方式）和商业卡发行** 功能无需开发，通过 Dashboard 操作即可完成，实施周期通常为 1-5 天

  * **API 批量支付** 功能需要 RESTful API 对接能力，实施周期通常为 1-2 周

  * **ERP 系统集成** 需要定制开发，实施周期通常为 2-4 周

  * **财资管理（TreasuryOS）** 支持 ISO20022 报文格式，能够与银行系统实现无缝对接，实施周期通常为 4-8 周




SUNRATE 的技术文档和 SDK 采取私有化模式（需登录后访问），商业信息保密性较高，但潜在客户无法在前期充分评估集成难度。如果企业有 API 集成需求，建议在签约前与 SUNRATE 技术团队确认 API 文档完整度与示例代码丰富度、沙盒环境测试权限获取流程、技术支持响应时效（是否提供专属技术团队）等细节。如果团队技术资源有限，可优先使用 Dashboard 操作的功能（如平台回款、商业卡发行）。

**💰****费用与合同条款**

在签约前，建议与多家服务商沟通并明确以下关键条款：

**📊****费用结构方面**

  * 交易手续费（百分比 + 固定费用）

  * 货币转换点差（与银行间汇率的差值）

  * 账户管理费或最低交易额要求

  * 提现费用与到账时效

  * 隐藏费用（如退款手续费、争议处理费）


**📋****定价模式方面**

不同服务商的透明度与适用场景有所不同：

  * **SUNRATE** 采用定制化报价模式，高交易量企业可通过协商获得更具竞争力的费率，但前期无法直接评估成本，需要商务谈判

  * **Stripe** 标准账户采用公开阶梯定价，费率透明且可即时计算，大额交易客户也可申请定制化报价

  * **PayPal** 采用公开固定费率，费率透明且无需协商，但大型商户可在销售额达到一定门槛后申请阶梯式优惠费率


**🌐****服务边界方面**

  * 支持的国家/地区清单（黑名单与白名单）

  * 单笔/单日交易限额

  * 资金到账时效承诺（SLA）

  * 冻结/审查规则（触发条件与解冻流程）


**🔒****数据安全与合规方面**

  * 数据存储位置（是否符合 GDPR/PIPL 要求）

  * 数据跨境传输机制

  * 客户资金管理方式（Safeguarding 隔离存管 vs. 混同管理）

  * 保险覆盖范围


**⚖️****争议处理机制方面**

  * 争议处理时效

  * 举证责任分配

  * 仲裁管辖地

  * 服务终止条款




建议向至少 3 家服务商索取报价，对比总体成本（TCO）而非单项费率，并要求提供费用计算器或模拟账单。同时需要明确合同中的"可调整条款"（如费率调整通知期），选择最符合业务需求的方案。
