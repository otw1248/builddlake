本文对跨境支付公司的界定：持支付牌照主营跨境收付款 / 换汇，或为该业务提供技术支撑的持牌服务商。

## **一、企业基本档案（2025 版）**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与主体：**

2014 年成立；其全球控股主体 TERRAPAY HOLDINGS LIMITED 于 2021 年在英国伦敦注册成立；主要运营实体 Terra Payment Services (UK) Limited 可追溯至 2008 年。

**🌍****全球布局：**

全球总部位于英国伦敦；在荷兰阿姆斯特丹、新加坡、印度班加罗尔、阿联酋迪拜、美国迈阿密及非洲、拉美等多地设有区域枢纽与运营团队。

**🛡️****合规资质：**

截至2025年8月，TerraPay在**31个全球市场** 获得监管批准；包括英国 FCA 监管、意大利央行 EMI 授权、美国 FinCEN MSB 许可、新加坡 MAS MPI 牌照、马来西亚央行 Class B 牌照等（企业披露）。

**📊****业务覆盖：**

支持 **210+** 发送国家与 **150** 个接收国家；覆盖 **71** 种货币；支付网络可触达全球 **7.5 亿** 个银行账户、**3.7 亿** 个移动钱包与 **12 亿** 张卡片（企业披露）。

**🎯****行业渗透与应用场景：**

核心定位为 B2B 跨境支付基础设施提供商；主要服务金融机构、货币转账运营商（MTOs）与移动钱包运营商；同时为跨境电商、旅游业、社交媒体与创作者经济等行业提供批量出款、虚拟卡支付与微支付解决方案。

## **二、全球化能力图谱**

###  （一）支付网络与区域布局

**💳****国际卡收单：**

通过与 Visa 的战略合作及股权投资关系接入 Visa Direct；同时支持 Mastercard、American Express 的支付处理（企业披露）；主要为 MTOs 提供支付收单解决方案，而非传统的独立站收单业务。

**🔗****本地支付聚合：**

核心优势在于连接碎片化的本地支付网络；主导建立数字钱包互操作理事会（Wallet Interoperability Council）；创始成员包括 M-PESA（非洲）、bKash（孟加拉国）、Airtel（非洲）、Nequi（哥伦比亚）等，聚焦打通新兴市场的移动钱包互联互通。

**🔄****跨境清算路径：**

直连 SWIFT 网络；金融机构可通过现有 SWIFT 连接向其网络内的移动钱包发送资金；同时支持 SEPA（单一欧元支付区）与 ACH（自动清算中心）等主流清算通道。

**📍****区域枢纽：**

  * 🇪🇺欧洲：伦敦 / 阿姆斯特丹 / 米兰覆盖 FCA 与 EMI 合规；承载 SEPA 清算；

  * 🇸🇬亚太：新加坡作为区域枢纽；持有 MAS MPI 牌照；辐射东南亚及南亚市场；

  * 🇺🇸北美：迈阿密承接 MSB 合规与 ACH 清算路径；

  * 🌍新兴市场：在非洲、拉美、中东多地设立办公室；深度运营本地移动支付网络。




### （二）合规与安全体系

**👤****KYC/AML：**

部署 TerraPay 智能平台（TIP）作为实时反洗钱与欺诈检测系统；可对接政府机构与全球监管数据库；采用模糊逻辑算法进行受益人姓名匹配；

**💰****客户资金管理：**

遵循各司法辖区的客户资金保障（Safeguarding）要求；

**⚖️****合规运营：**

通过 TIP 平台推动流程自动化；可将异常交易的识别与处置时间从数天缩短至几分钟。

## **三、技术能力清单**

###  （一）智能技术与架构

**☁️****云原生与多活：**

基于云平台进行多活部署；跨多个可用区设置主站点、灾备站点与地理冗余站点；平台可用性目标为 99.99%；

**🛡️****风控引擎：**

TIP 作为核心风控引擎；结合规则引擎和机器学习模型的混合系统；对全球交易进行实时分析并识别异常模式；

**📈****数据与洞察：**

合作伙伴可通过 Engage Portal 查看实时外汇汇率、分类账余额与交易分析；提供报表 API（Statements API）与分类账余额 API（Ledger Balance API）；支持 JSON 数据拉取；

**🔧****开发者生态：**

提供 RESTful API（HTTPS/JSON）；官方 SDK 覆盖 Java、JavaScript、Python；支持沙盒 / UAT / 生产三阶段环境。

### （二）支付与结算效率

**⏱️****时效与清算：**

企业披露 **97%** 的交易在 1 分钟内完成；支持向移动钱包、银行账户和卡片的即时或近即时资金下发；通过接入 SEPA、ACH 等本地网络实现区域内加速清算；

**💵****资金与对账：**

系统支持多币种账户与 GL（总分类账）/TL（交易分类账）双分类账体系；便于合作伙伴进行资金管理与对账。

## **四、计费结构与服务规则**

**🌐****跨境收款：**

采用透明的定制化报价模式；强调无隐藏费用；费率根据交易类型（P2P/P2B）、支付工具（钱包 / 银行账户）、货币对与目标市场区域等维度定制；

**💳****国际卡收单：**

主要为 B2B 客户提供支付网关与收单解决方案；支持 Visa、Mastercard 和 American Express 卡片处理；按协议提供具备竞争力的费率；

**💵****本地卡 / 本地支付：**

针对不同区域的本地支付方式提供专属计费规则；

**🏦****提现与对公结算：**

支持 71 种货币的全球出款；以实时与近实时为主；

**✨****增值服务：**

  * **虚拟卡** ：提供多币种虚拟卡发行；覆盖 USD、EUR、GBP、CAD、SGD、AUD、ZAR、JPY、AED 等 14 种货币；消费时无外汇加价（企业披露）；

  * **合规支持** ：提供合规框架；支持 AML、CFT 与 KYC 要求；


**📝****费用项目与条款补充：**

最终费用结构与服务条款以双方签署的《商户协议》为准；本章聚焦收费项目与适用规则，不含具体数值。

## **五、汇率管理与资金流转**

**📊****汇率基准与数据源：**

提供实时外汇汇率 API（Create Quotation API）；合作伙伴可获取有时效性的货币兑换报价；

**🔄****换汇与对冲工具：**

即期换汇支持 71 种货币的实时兑换；

**🤖****财资自动化：**

提供财资自动化解决方案；协助企业管理不同目的地的流动性需求；以数据驱动优化全球资金调度；

**📤****资金账户与调度：**

支持批量付款；企业可通过 API 向全球员工、供应商或客户进行大规模资金下发；

**📌****小结：**

通过实时汇率 API 与财资自动化能力，帮助 B2B 客户实现高效、可预测的全球资金调度。

## **六、生态对接与服务能力**

###  （一）电商平台 / 平台经济接入

**📦****平台资金下发：**

面向全球电商、旅游与创作者经济平台提供跨境支付能力；支持向卖家或服务提供方进行批量结算与资金下发；

**🛠️****核心能力：**

支持通过单一 API 向银行账户、移动钱包和卡片进行大规模、多币种的安全付款；提供收款方验证 API；用于校验收款方账户状态与信息。

### （二）企业系统（ERP / 财税）集成

**🔌****API 集成：**

通过 REST API 实现与内部财务系统的数据自动化同步；

**📋****对账与凭证：**

通过与 SWIFT 的集成；可提供从账户到钱包的端到端交易追踪。

### （三）本地化网络

  * 🇪🇺欧洲：持有意大利 EMI 牌照；支持 SEPA 清算网络；

  * 🇺🇸北美：持有美国 MSB 许可；支持 ACH 清算网络；与 Paramount Commerce 合作在加拿大支持 Interac e-Transfer® for Business 实时支付；

  * 🇸🇬东南亚：持有新加坡 MAS MPI 牌照与马来西亚央行牌照；整合区域内主流电子钱包；

  * 🌍非洲：与 M-Pesa、Airtel 等移动货币运营商深度合作；网络覆盖优势明显。


**🚀****上线要求（通用）：**

合作伙伴需完成业务确认、KYC/KYB 审核与 API 技术对接；通过沙盒、UAT 环境的完整测试后方可上线。

### （四）服务交付与支持

**🔄****实施流程：**

采用"沙盒 → UAT → 生产"的三阶段环境体系；提供专业合规团队支持上线流程；

**💬****支持体系：**

提供 24/7 多语言客户支持；提供开发者门户、API 文档与实践指南。

## **七、增值工具与行业方案**

###  （一）风控套件（TerraPay Intelligence Platform，TIP）

**📡****数据信号：**

整合来自 200+ 来源国到 100+ 目的地国的交易数据；对接政府与全球监管数据库；

**🧠****策略模型：**

定制化机器学习结合规则引擎；对 KYC 与交易数据进行实时分析与建模；

**📋****场景模板：**

提供模糊逻辑算法用于受益人姓名匹配；处理拼写变体、昵称与多语言格式；

**⚙️****商户可配置项：**

提供自动化实时报告；识别异常交易与可疑行为；缩短合规处置时间。

### （二）汇率与资金（Treasury Automation）

**💧****流动性管理：**

基于历史数据与分析模型进行流动性优化；

**🛣️****智能路由：**

按成本与速度等因素自动选择最优支付工具（钱包 / 银行 / 卡）。

### （三）数据与对账

**📊****事件模型与 API：**

提供交易状态查询、报表拉取与分类账查询等 API；支持事件驱动自动化；

**📄****报表与导出：**

支持通过 API 以 JSON 格式拉取交易、清算与费用等报表。

### （四）行业方案

**🏦****金融机构与 MTOs：**

组合为全球支付网络基础设施 + TIP；通过单一 API 连接新兴市场移动钱包网络；确保端到端合规；上手步骤含合规尽调、KYB 审核、API 对接与三阶段测试；

**✈️****旅游业：**

虚拟卡 + 批量出款 + 多币种账户；简化供应商支付与佣金结算；支持快速退款；可配置限额、有效期与商户类别限制；与 Thredd 合作提供 B2B 虚拟卡服务；支持 14 种货币；消费端无外汇加价（企业披露）；

**🎨****创作者经济与社交平台（如内容创作平台、直播平台等）：**

跨境微支付 + 批量出款；针对小额高频转账；企业案例披露可将支付门槛从 $250 降至 $50；费用显著下降；处理时间由数天缩短至数分钟；上手步骤含批量支付 API 集成、全球收款方账户验证与结算策略制定。

**📋****附：快速落地清单**

  * **🔧****技术** ：获取 API Key；完成沙盒 / UAT 集成测试；确认交易创建、状态查询与报表拉取用例覆盖；

  * **💰****财务** ：配置 GL/TL 分类账；设置多币种账户；对接报表 API 实现自动化对账；

  * **🛡️****合规** ：提交主体 / KYB 材料；完成 AML/CFT 流程配置；熟悉 TIP 监控与报告功能；

  * **👥****运营** ：明确 24/7 支持流程；设定 SLA/SLO 预期；使用开发者门户进行交易监控。




**📌****小结：**

TIP（风控）+ 财资自动化（资金）+ 行业方案构成服务 B2B 客户的核心组合。

## **八、适配场景与注意事项**

###  （一）适配场景

  * **🏦****金融机构与支付服务商** ：银行、MTOs、移动钱包运营商；需要跨境支付基础设施以服务其客户；

  * **🌍****新兴市场开拓者** ：在非洲、拉美、东南亚等地深度连接本地移动钱包网络的企业；

  * **📦****平台型经济体** ：旅游平台、电商市场与创作者平台；需向全球卖家 / 供应商 / 创作者进行大规模、多币种资金下发；

  * **🛡️****合规驱动型企业** ：对跨境支付的 AML 与合规要求高的金融科技公司；依托成熟全球合规框架与 TIP 开展业务。




### （二）注意事项

  * ⚖️**合规考量** ：确认签约法律主体与目标市场的牌照状态；不同市场由不同法律实体运营；需明确监管框架与义务；

  * 🔧**技术能力要求** ：服务模式以 API 集成为主；需要具备研发能力进行对接与测试；不适合寻求即插即用插件或无代码集成的客户；

  * 💰**商务定价模式** ：采用定制化报价；上线前明确交易费、月费、设置费等费用结构；

  * 📋**服务边界认知** ：定位为 B2B 支付基础设施；负责后端清算、合规与网络连接；前端体验、客服流程与争议处理由合作伙伴自身承担；

  * ⏱️**实施周期预期** ：遵循三阶段集成测试；含业务确认、KYC/KYB 审核、API 对接与合规配置；周期通常为数周至数月。




## **九、区域合规与适配方案**

###  （一）欧洲

**🛡️****准入与合规要点：**

通过意大利央行 EMI 授权；受 FCA 与意大利央行等监管；遵循 GDPR 数据保护要求；

**🏢****本地化运营：**

支持 SEPA 清算；

**📋****上线清单：**

确认签约主体；提交 KYB 资料；完成 SEPA 对接测试。

### （二）北美

**🛡️****准入与合规要点：**

持有美国佛罗里达州 MSB 许可证；遵循 BSA 反洗钱法案；

**🏢****本地化运营：**

支持 ACH 清算；与 Paramount Commerce 合作在加拿大支持 Interac e-Transfer® for Business 实时支付；

**📋****上线清单：**

提供 MSB 所需合规文件；完成 ACH 对接测试。

### （三）亚太地区

**🛡️****准入与合规要点：**

在新加坡（MAS MPI）与马来西亚（央行 Class B）等市场持有本地牌照；在印度、澳大利亚设有运营实体；

**🏢****本地化运营：**

连接区域内主流电子钱包；

**📋****上线清单：**

按目标国家提交监管材料；重点测试跨境钱包支付的稳定性与成功率。

### （四）拉美

**🛡️****准入与合规要点：**

在哥伦比亚设有办公室；

**🏢****本地化运营：**

与本地钱包提供商 Nequi 建立合作；

**📋****上线清单：**

按当地要求完成主体确认与接口对接。

### （五）中东与北非

**🛡️****准入与合规要点：**

在阿联酋迪拜设区域总部；与 Al Ansari Exchange 建立合作关系（2024 年 5 月 14 日宣布）；

**🏢****本地化运营：**

通过合作网络提供本地收单与资金下发服务；

**📋****上线清单：**

确认当地运营主体；完成合规文件提交与通道测试。

### （六）撒哈拉以南非洲

**🛡️****准入与合规要点：**

在肯尼亚、南非、乌干达等多国设有办公室；获得 IFC 融资支持（2019 年 800 万美元 A 轮；2024 年 7 月 9,500 万美元债务融资）；

**🏢****本地化运营：**

深度集成 M-Pesa、Airtel 等移动货币网络；支持向坦桑尼亚、乌干达、肯尼亚等国移动钱包实时转账；与 Safaricom 的 M-PESA 合作覆盖 3,000 万以上用户；

**📋****上线清单：**

确认本地运营实体；测试与移动货币网络的互操作性。

**📌****小结：**

区域差异化策略明显；在亚太与非洲等新兴市场的合规与网络深度突出；在欧美市场更多依赖主流清算网络。
