## 一、费率体系全景解析

### 🔍 全流程费率特征

基础费率模式说明

### 📌 费用构成关键点

**风险准备金机制：** Rapyd 有权根据商户风险评估要求建立准备金账户，金额为固定数额或交易额百分比，用于抵御拒付、退款等风险，释放条件由Rapyd根据业务稳定性（如拒付率）动态调整，具体条款在合同中明确约定；

**增值服务费用：** 超出标准功能的定制化服务需额外付费，包括高级风险管理规则、定制化报表分析、专属技术支持等，费用根据服务复杂度单独议定；

**拒付处理费用：** 持卡人发起拒付时，除退还交易本金外，商户需支付固定处理费，用于覆盖争议处理的管理成本，具体金额在合同中列明；

**其他潜在费用：** 入网费、账户管理费、退款手续费等未设统一标准，均通过商业合同约定，商户需在签约前详细确认所有可能产生的费用项。

## 二、汇率管理与资金流转

### 🔄 汇率机制架构

**基准汇率体系：**

  * 定价模式：采用固定汇率（Fixed-Rate）机制，商户可通过 API 请求有时效性的锁定报价（通常为数分钟），规避交易处理期间的汇率波动风险；

  * 成本构成：汇率报价为 "全包价"，已包含外汇点差（Spread），具体点差计算方式未公开，作为商业合同的核心条款由双方议定。




**货币兑换能力：**

  * 支持全球主要货币对及新兴市场货币兑换，多币种钱包（Rapyd Wallet）可持有数十种主流货币，满足全球化资金管理需求；

  * 提供自动化货币兑换功能，商户可设置规则将非基础货币资金自动转换为偏好币种，同时支持通过 API 实现批量兑换操作。




### ⏱️ 时效分级

**资金流转优化：**

  * 本地化清算网络：通过直接对接全球各地支付系统，绕过传统代理行网络，缩短资金在途时间，提升跨境流转效率；

  * 实时状态追踪：商户可通过客户端门户或 API 查询每笔交易的实时状态，系统通过 Webhooks 机制在交易状态变更时自动推送通知，实现资金流向的全链路可视化。




## 三、合规架构与安全体系

### 🏛️ 资质矩阵

### 🛡️ 安全与资金保障体系

**技术安全认证：** 通过 PCI DSS Level 1 认证，满足支付卡数据安全的最高标准；获得 ISO 27001（信息安全管理）、ISO 27017&27018（云服务安全）认证，建立全面的安全管理框架；

**数据保护措施：** 采用 TLS 1.2 + 加密传输、AES-256 加密存储敏感数据；部署 WAF、IDS/IPS 防护系统；实施严格访问控制与行为监控，定期开展第三方渗透测试；

**反欺诈引擎：** "Rapyd Protect" 利用 AI 和机器学习实时评估交易风险，分析数百个数据点，结合设备指纹、行为分析技术识别异常模式，提前拦截潜在欺诈；

**资金隔离机制：** 客户资金存放在独立的专用账户中，与公司自有资金完全分离，符合英国 FCA、欧盟 BoL 等监管要求，确保客户资金不会用于抵偿公司债务。

**技术集成选项：**

**全球办公布局：** 在迈阿密（美洲）、伦敦（欧洲）、新加坡（亚太）、迪拜（中东）等核心区域设立办公室，形成覆盖主要市场的服务网络，其中迈阿密聚焦拉美市场，新加坡作为亚太总部；

**多语言支持能力：** 依托全球化团队，提供英语、西班牙语、葡萄牙语等多语言销售与技术支持，覆盖北美、欧洲、拉美及亚太市场的沟通需求；

**本地市场洞察：** 定期发布区域支付指南，如《拉丁美洲支付指南》《亚太地区电商与支付指南》，提供市场概览、支付方式分析及合规要点，助力商户制定本地化策略。

## 五、增值服务与行业方案

### 🛠️ 效率工具矩阵

  * 统一管理后台："Rapyd Client Portal" 提供集中化交易监控、多币种资金管理、财务报表生成功能，支持多用户权限配置，简化全球支付运营；

  * 智能路由系统：支付编排层自动选择最优收单通道，基于成本、成功率、地理位置动态调整，保障支付链路稳定高效；

  * 自动化资金工具："Rapyd Wallet" 支持多币种资金归集，"Rapyd Disburse" 实现全球批量付款，通过规则配置自动化结算流程；

  * 精细化报表系统：支持按时间、支付方式、地区、货币等多维度生成交易报表，通过 API 或后台导出，为业务决策提供数据支撑。

## 六、适用场景与企业类型

### ✅ 高契合度客户画像

#### 新兴市场拓展者：

  * 计划进入拉美、东南亚等本地支付复杂市场的跨境电商、游戏企业，可通过 Rapyd 的本地化网络快速接入 870 + 支付方式，提升当地用户转化率；




#### 跨国资金管理者：

  * 在全球多地有业务的企业，借助多币种钱包实现资金集中管理，简化跨国收付款流程，降低财务运营复杂度；




#### 数字原生企业：

  * 游戏、SaaS、平台型企业等需处理高并发、复杂分账或循环计费的业务，可利用定制化方案解决行业特有支付痛点；




### ⚠️ 决策评估要点

  * **成本透明度：** 需在合同中明确所有费用构成（交易费率、点差、准备金等），避免隐性成本影响预算；

  * **技术资源匹配：** 评估自身开发能力，选择插件集成（低门槛）或 API 集成（高定制）方案，预留充足测试周期；

  * **资金流规划：** 不同支付方式与地区的结算周期差异显著（T+0 至 T+5+），需纳入现金流管理考量；

  * **风控责任边界：** "Rapyd Protect" 提供风险监控，但根据服务协议，拒付与欺诈的最终财务责任明确由商户承担，需建立配套内部流程；

  * **服务水平确认：** 明确客户支持响应时间、服务渠道及是否配备专属经理，确保业务持续运行保障。
