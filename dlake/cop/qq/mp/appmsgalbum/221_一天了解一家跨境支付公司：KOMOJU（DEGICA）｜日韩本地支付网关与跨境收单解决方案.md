本文对跨境支付公司的界定：持支付牌照主营跨境收付款 / 换汇，或为该业务提供技术支撑的持牌服务商。

## **一、企业概况**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与主体：**

2005年成立，总部位于日本东京，法定名称为 DEGICA Co., Ltd.。

**🏷️****品牌架构：**

核心品牌 KOMOJU 专注支付网关业务。2021年将游戏发行业务剥离至独立公司 Komodo，转型为纯支付技术公司。

**🌍****全球布局：**

东京为全球运营中心，团队支持全远程办公，分布于日本、欧洲及亚太地区。

**🛡️****合规资质：**

  * 日本牌照：信用卡信息处理机构备案、合规发票开具资质（日本消费税制度）

  * 行业认证：已获 PCI-DSS、ISO 27001、Privacy Mark 等国际认证

  * 全球模式：在美国、欧洲等市场通过与国际卡组织及持牌收单机构合作实现合规覆盖


**📊****业务覆盖：**

支持 **65+** 种国际及本地支付方式，核心市场包括日本、韩国、中国、东南亚、欧洲。结算货币支持 USD、EUR、JPY、SGD、HKD、AUD 共 **6 种** 。

**🎯****主要应用场景：**

主要服务于数字娱乐（游戏、动漫）、跨境电商、订阅服务等，提供托管收银台、支付 API、平台分账、订阅计费等能力。

## **二、计费结构与服务规则**

**📋****服务项目清单****💳****交易手续费：**

按支付方式和交易类型（国内/跨境）采用差异化收费规则，覆盖信用卡、便利店支付、数字钱包、银行转账、手机话费代扣、韩国本地支付等类别。

**🏦****提现与结算：**

  * 日本银行账户：按金额分档收取转账费

  * 海外银行账户：按笔收取固定转账费，建议根据资金周转需求规划提现频次


**✨****增值服务：**

  * 订阅计费：定期扣款管理、试用期设置、智能重试

  * 平台分账：支持平台与子商户的交易金额拆分

  * 退款处理：信用卡退款与非信用卡退款（后者需人工处理）


**📝****费用项目补充说明：**

可能包含货币转换费、退款手续费（非信用卡）、海外提现费等项目。

**📌****小结：** 本章聚焦收费项目与适用规则，最终以《服务协议》为准。

## **三、全球化服务能力**

###  （一）支付网络与区域布局

**💳****国际卡收单：**

直连 Visa、Mastercard、JCB、Amex、Diners Club、Discover 网络，支持 3D Secure 验证。

**🔗****本地支付聚合：**

整合 **65+** 种本地支付方式，核心覆盖日本（便利店、PayPay、LINE Pay、手机话费代扣等）、韩国（本地卡、Kakao Pay、Naver Pay 等），同时支持中国（支付宝、微信支付）、东南亚（GrabPay、OVO 等）、欧洲（iDEAL、Giropay 等）及拉美（Pix）市场。

**🌍****跨境特性：**

商家无需在日本/韩国设立实体公司或银行账户即可收款。

### （二）合规与数据安全

**📜****牌照与认证：**

  * 日本：信用卡信息处理机构备案、合规发票开具资质

  * 认证：PCI-DSS Level 1（v4.0）、ISO 27001、Privacy Mark

  * 数据保护：遵循日本 APPI、欧盟 GDPR 相关要求


**🔒****数据安全：**

  * 传输加密：TLS 1.2+

  * 存储加密：AES-256

  * 密钥管理：HSM 硬件安全模块

  * 令牌化：将真实卡号转换为加密令牌，商户系统不接触原始卡号


**💰****客户资金管理：**

通过代收代付模式或网关模式，借助国际卡组织通道及合作收单网络实现资金清算。

### （三）汇率与资金管理

**📊****汇率机制：**

  * 实时/每日汇率：交易时锁定汇率，降低汇率波动风险

  * 动态货币转换（DCC）：消费者以本币显示价格并支付


**💵****账户结构：**

支持多币种处理，结算货币包括 **6 种** （USD、EUR、JPY、SGD、HKD、AUD），其他货币需通过汇率转换结算。

**💱****外汇优化：**

支持商户以 JPY 定价但以 USD 结算，适合跨境电商的多币种收款需求。

## **四、技术能力与结算效率**

###  （一）技术架构

**⚙️****API 设计：**

采用 RESTful API 架构，HTTPS（TLS 1.2+）协议，JSON 数据格式，基础认证（Basic Auth），通过 URL 区分接口版本。

**🔌****核心接口：**

  * 交易：创建、捕获、取消、查询

  * 退款：创建、查询、列出

  * 客户：创建/更新/删除（Tokenization）

  * 订阅：定期扣款管理


**🔧****开发者生态：**

  * SDK：官方覆盖 Ruby（核心语言）、PHP 及移动端（React Native、Android、iOS），Python、Node.js 等语言可直接调用 REST API

  * 插件：Shopify、WooCommerce、Magento、Wix、BigCommerce

  * 回调通知（Webhook）：支持 HMAC-SHA256 签名验证，覆盖支付各环节的状态通知

  * 测试环境：提供沙盒测试环境及测试卡号




### （二）智能路由

**🔄****智能路由：**

支持智能路由（Smart Routing）技术，在支付失败时自动切换收单行以提高成功率。

## **五、接入方式与平台能力**

###  （一）接入形态

**💻****托管收银台：**

完全托管的支付页面，自动适配语言、货币、支付方式，支持自定义 Logo 和颜色。

**🔌****支付 API：**

RESTful API，商户完全控制前端，支持预授权、捕获、退款等独立操作。

**📱****移动 SDK：**

提供 iOS 和 Android SDK，支持应用内支付体验。

**🔗****支付链接（Payment Links）：**

免开发收款工具，支持邮件、社交媒体分享。

### （二）平台分账能力

**💰****平台分账能力：**

针对多商户平台（如电商平台、外卖平台），提供内置分账功能：

  * 分账逻辑：指定平台佣金金额，剩余金额自动结算给子商户

  * 费用承担：交易手续费仅向平台商户收取

  * 退款分账：按原分账比例自动扣减




### （三）区域运营注意事项

**🇯🇵****日本市场：**

  * 需遵守《分期付款销售法》和《个人信息保护法》（APPI）

  * 网店需提交《特定商业交易法》页面

  * 便利店支付存在到店付款后放弃付款的风险


**🇰🇷****韩国市场：**

  * 通过本地收单能力提升授权成功率

  * 多种本地支付方式的限额差异需提前测试


**🌍****跨境通用：**

结算货币支持 **6 种** （USD、EUR、JPY、SGD、HKD、AUD），其他货币需通过汇率转换结算。

## **六、增值服务**

###  （一）风控套件

**🛡️****风控引擎：**

基于机器学习的实时风控系统，结合规则引擎，覆盖：

  * 3D Secure：支持 1.0 和 2.0 版本

  * 实时监控：检测异常交易模式

  * 行为分析：分析用户设备指纹和历史行为

  * 黑名单：自动拦截已知欺诈卡号和 IP




风控功能默认开启，无需单独配置。

### （二）订阅服务

**🔄****订阅计费引擎：**

  * 计费周期：支持日、周、月、年等灵活设置

  * 自动扣款：支持信用卡和部分数字钱包的自动续费

  * 试用期管理：支持设置免费试用期

  * 智能重试：提供失败交易自动重试机制




## **七、适配场景分析**

###  适合使用 KOMOJU 的场景

**🎮****数字内容/游戏出海日本：**

如果您的业务是游戏、动漫、数字内容等交易场景，需要覆盖日本无卡用户群体，KOMOJU 提供便利店支付、手机话费代扣等本地支付方式。2021年剥离的游戏部门 Komodo 目前是 Steam Deck 亚洲（日韩港台）官方授权经销商。

**🛒****跨境电商进入日韩市场：**

如果您希望在目标市场无实体的情况下收取当地货币，KOMOJU 支持商家无需在日本/韩国设立实体即可收款，并提供 Shopify、WooCommerce 等平台可快速集成的插件。

**🏢****多商户平台运营：**

如果您运营的是电商平台或 SaaS 平台，需要将交易金额在平台与子商户之间拆分，KOMOJU 提供内置的平台分账能力，支持通过 API 为子商户创建账户和管理结算。

**🔄****订阅服务覆盖无卡用户：**

如果您的订阅服务面向日本市场，希望覆盖无信用卡的用户群体，KOMOJU 支持便利店支付作为订阅付款方式。

### 决策建议

**🎯****决策建议：**

在选择跨境支付服务商时，建议从以下维度进行综合评估：

**🌏****市场聚焦度：**

KOMOJU 在日韩市场的本地支付覆盖较为全面。如果您的业务主要集中在日本或韩国，KOMOJU 在这些市场的本地支付接入选择较多；如果业务覆盖全球多个区域，可同时了解 Stripe、Adyen 等全球化方案。

**💵****结算货币需求：**

KOMOJU 支持 **6 种** 结算货币（USD、EUR、JPY、SGD、HKD、AUD）。如果您需要其他结算货币，需评估汇率转换成本或考虑其他支持更多结算货币的服务商。

**🔧****技术实施能力：**

如果您的技术栈是 Ruby/Rails，KOMOJU 提供官方 SDK 和中英文文档；如果使用 Python、Node.js 等语言，可直接调用 REST API。对于非技术型商户，Shopify、Wix 等平台插件可实现免开发接入。

**📊****交易规模与频次：**

KOMOJU 无每笔固定费的定价模式适合小额高频交易场景（如游戏道具、数字内容）；对于高客单价 B2B 业务，按百分比收费的模式需结合交易规模评估成本。
