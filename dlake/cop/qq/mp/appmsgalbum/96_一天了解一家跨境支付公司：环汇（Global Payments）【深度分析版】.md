## 一、费率体系全景解析

### 🔍 全流程费率构成

### 📌 费率机制特点

**定制化定价逻辑：** 官网不公布统一费率表，需通过官方报价工具提交企业信息（行业类型、月均交易量、目标市场等）获取个性化方案，评估维度包含交易风险等级与结算频率，精准适配不同规模商户的成本结构

**费用透明度保障：** 明确区分 "不可控费用"（交换费、评估费）与 "可控服务费"（处理费等），帮助商户清晰识别成本构成（官网帮助中心提供费用组成详解及示例计算）

**灵活计费模式：**

  * 交换加模式：在交换费 + 评估费基础上附加固定百分比 / 单笔费用，透明度高，适合交易量稳定的中大型商户

  * 分层定价：按交易 "合格 / 中等合格 / 不合格" 分类计费，简化核算，适合交易类型单一的商户

  * 统一费率：单一固定百分比，适合交易量较小、追求流程简化的商户




## 二、汇率管理与资金流转

### 🔄 多币种处理体系

**核心汇率工具：**

  * 多币种处理（MCP）：支持商户以多种当地货币接受支付，汇率转换发生在交易授权或结算阶段，系统自动匹配最优转换时点，适配全球化运营的本地化定价需求

  * 动态货币转换（DCC）：允许国际消费者选择以本国货币支付，交易时实时展示转换后金额及汇率，提升支付透明度，转换费率根据服务协议约定，体现在交易明细中

  * 结算货币选择：商户可指定本地货币或主要国际货币（如美元、欧元）作为结算货币，货币转换基于实时批发汇率加协议加点，转换记录在商户后台可追溯查询




### ⏱️ 结算时效规范

**标准到账周期：**

  * 常规卡交易：T+1 或 T+2 工作日（交易完成后 1-2 个工作日到账），官网明确标注 "T+1 为核心目标"，多数地区实现 90% 以上交易 T+1 结算

  * 跨境交易：因涉及多银行系统清算，时效略长于本地交易，通常为 T+2-T+3 工作日




**特殊情况影响：**

  * 周末 / 节假日：结算周期顺延至下一个工作日（以商户所在地银行工作日为准），节前大额交易可申请加急处理

  * 银行处理：资金清算后，最终到账时间受商户开户银行处理速度影响，通常在清算完成后 24 小时内入账




**加速选项：** 提供 "当日到账" 服务（需额外费用），适合对现金流要求高的行业（如生鲜零售），需在工作日特定时段前提交申请

## 三、合规架构与安全体系

### 🏛️ 全球合规资质矩阵

### 🛡️ 技术安全体系

**数据保护技术：**

  * 全链路加密：传输中数据采用 TLS 1.3 协议，静态数据采用 AES-256 加密存储，密钥定期自动轮换

  * 令牌化技术：替换真实卡号为随机令牌，令牌仅在支付流程中有效，降低敏感信息泄露风险，支持跨渠道令牌复用




**欺诈防护系统：**

  * AI 实时监控：分析多维度风险（含交易地点、设备指纹、消费习惯等），识别异常交易模式，具备快速响应能力

  * 多层验证工具：集成 3D Secure、地址验证（AVS）、CVV 验证等，高风险交易触发阶梯式验证机制




**资金安全机制：**

  * 客户资金隔离：商户资金 100% 存管于合作银行的隔离账户，与公司自有资金完全分离，每日对账确保一致性

  * 透明对账工具：通过商户门户提供实时交易记录、结算明细、费用构成，支持 API 对接企业 ERP 系统，实现自动化对账




## 四、生态对接与服务能力

### 🧩 全渠道集成能力

**技术对接方式：**

  * API/SDK 工具：提供开发者门户及详细文档（含沙箱测试环境），支持 RESTful API 与各语言 SDK，适配自定义网站、移动应用嵌入支付功能，联调周期约 2-4 周

  * 预构建插件：适配 Shopify、WooCommerce、Magento、Salesforce Commerce Cloud 等主流电商平台，插件支持自动更新，减少维护成本

  * 线下集成方案：提供 POS 终端 SDK，支持与收银系统、库存管理软件联动，实现交易数据与业务数据实时同步




**全渠道覆盖：**

  * 线下场景：支持传统 POS 终端、移动支付设备、自助结账系统，覆盖实体门店、展会、上门服务等场景，支持非接触式支付与电子收据

  * 线上场景：提供支付网关服务，支持电商网站、移动应用、社交平台的在线支付，集成 3D Secure 验证提升成功率




### 🌍 全球化服务网络

**本地运营支撑：**

  * 区域布局：在全球 100 + 国家设有本地办事处，核心市场（北美、欧洲、亚太）配备专属客户经理，提供快速响应服务（如4小时内），客服团队覆盖多达 15 种语言，提供 7×24 小时电话与在线支持，常见问题解决率高




**本地化适配：**

  * 支付方式集成：支持区域特色支付工具（如欧洲 SEPA 直接借记、亚洲电子钱包、中东 Sadad 等），本地支付覆盖率≥95%

  * 合规指导：提供当地税务规则、数据隐私法规（如 GDPR、CCPA）的适配建议，协助完成跨境交易的申报流程

  * 文化适配：根据区域消费习惯优化支付流程，提升转化率




## 五、增值服务与行业方案

### 🛠️ 效率工具矩阵

  * 数据分析平台：通过商户门户提供交易趋势、销售报表、退单分析等数据洞察，支持自定义报表与自动推送，数据延迟低，辅助库存调整与营销决策

  * 统一管理系统：整合线上线下交易数据、设备管理、结算记录，实现全渠道数据集中管控，支持多门店权限分级管理，总部可实时监控各终端交易状态

  * 合规支持工具：提供 PCI DSS 合规评估、漏洞扫描、员工安全培训模块，自动生成合规报告，降低合规成本与风险

  * 财务集成工具：对接 QuickBooks、SAP、金蝶等主流财务软件，自动同步交易数据与对账结果，缩短财务结账周期

## 六、适用场景与企业类型

### ✅ 高匹配客户画像

#### 中大型跨国企业：

  * **核心需求：** 需统一管理全球支付流程的跨国零售商、酒店集团等，可利用其 100 + 国家的本地化支持能力，实现多区域支付标准统一（如某连锁酒店通过其解决方案，实现了多国支付方式的整合，显著提升了管理效率）




#### 全渠道经营商户：

  * **核心需求：** 同时拥有实体门店与电商平台的企业，通过集成方案实现线上线下数据互通，适合服饰品牌、家居零售商等（如服饰品牌通过其全渠道方案，可以打通门店与官网支付数据，从而实现会员消费体验一致化）




#### 垂直行业企业：

  * **核心需求：** 零售、餐饮、医疗等有特殊合规要求的行业，可适配其定制化功能模块（如医疗机构利用 HIPAA 合规方案，简化患者支付流程）




### ⚠️ 实操建议

#### 费率获取：

  * 通过官网 "在线报价工具" 提交详细信息（行业类型、月均交易量、目标市场、交易类型占比），获取精准费率方案，建议同时咨询 3 种计费模式的适用场景




#### 技术对接：

  * API 集成建议预留 2-4 周开发周期，利用开发者门户的测试工具与示例代码降低联调风险，技术团队需评估自身系统与支付网关的兼容性




#### 合同审查：

  * 重点确认费用构成（尤其是交换费、评估费的转嫁方式）、服务终止条款、设备维护责任，明确费率调整的通知周期（通常为 30 天）
