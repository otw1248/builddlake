## **一、企业概况与核心定位**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

【注】Spreedly为多网关管理平台，不持支付牌照，不参与资金清算。

**🎯****核心定位：**

Spreedly作为全球领先的开放式支付流程编排平台，专注于提供支付数据存储、多网关连接、智能路由和欺诈防护等技术基础设施服务。

**📅****成立时间与主体：**

2007年成立（公司注册），2008年正式推出平台，2013年3月进行了重大的业务转型，从订阅支付服务转型为当前的支付流程编排业务模式。公司法定全称Spreedly， Inc.，注册于美国。

**🌍****全球布局：**

公司总部位于美国北卡罗来纳州达勒姆，作为全球运营中心。

**🛡️****合规资质：**

支付数据安全：PCI-DSS Level 1（最高级别）、SOC 2 Type 2（安全性与机密性）

网络认证：Visa/Mastercard网络令牌服务商认证、EMVCo 3DS2认证

**📊****业务覆盖：**

截至2025年，通过Connect产品可以连接**140+** 支付网关，覆盖**100+** 国家/地区及**100+** 货币。

**🎯****主要应用场景：**

主要服务于高增长的跨境电商、SaaS订阅、在线旅游、票务平台、多商户市场，为企业提供高授权率优化和PCI合规简化服务。

**⚙️****基本服务能力概览：**

核心产品：Vault（令牌化与存储）、Connect（网关连接与交易路由）、Protect（欺诈防护与3DS）、Optimize（智能路由与交易优化）。Vault仅负责令牌化与存储，不负责交易路由；所有交易需经Connect模块完成。

## **二、收费模式与服务规则**

**📋****收费模式：**

Spreedly采用SaaS订阅费与API调用量相结合的收费模式，作为技术平台，Spreedly本身不收取基于交易额的支付手续费。

**📝****服务项目清单：****🏢****Spreedly收取的费用：**

平台订阅费：按订阅计划分级，包含Flex标准计划和Enterprise定制计划，收取月度或年度订阅费用。

API调用费：按API调用次数计费，适用于Vault（存储库）、Connect（网关连接）、Protect（欺诈防护）等产品模块的调用；具体API单价取决于用量级别。

**🏛️****企业级服务：**

SLA保障：企业级定制合同费用。

技术支持：订阅支持计划可享受专属技术客户经理、电话支持等高级服务。

**💳****支付网关或其他第三方收取的费用：**

交易手续费：由客户集成的支付网关（如Stripe、Adyen、Braintree）收取，Spreedly不收取。

附加费用：可能包含网关收取的拒付处理费、国际清算附加费、货币转换费等，Spreedly平台不收取。

**📊****定价说明：**

根据部分用户反馈，测试环境集成可能产生费用；Enterprise计划的定价调整机制建议在商务谈判时明确约定，以确保成本可控。

## **三、全球化能力图谱与连接**

Spreedly的核心价值在于其广泛的连接能力和高度灵活的支付流程编排能力，允许商户在全球市场实现支付网关的灵活切换。

### （一）支付网络与区域布局

**🔗****支付网关连接：**

连接数量：通过Connect产品，单一API集成**140+** 支付网关和支付服务。

核心网关：包括Stripe、Adyen、Braintree、PayPal、Authorize.Net、CyberSource等全球主流支付处理商。

网络支持：支持Visa、Mastercard、American Express、Discover、JCB、UnionPay等主流国际及本地卡品牌。

**💸****本地支付选项整合：**

整合了全球主要本地和国际支付方式，包括卡支付、数字钱包、银行转账与BNPL，通过集成的网关（如Rapyd）可支持数百种本地选项：

数字钱包：Apple Pay、Google Pay、PayPal、微信支付、支付宝、Venmo、GrabPay等。

银行转账/借记：ACH（美国，通过支持ACH的网关实现）、SEPA（欧洲）、BACS/FPS（英国）、PIX（巴西，通过支持PIX的网关实现）。

现金票据：Boleto（巴西）、OXXO（墨西哥）、日本Konbini等。

先买后付：Affirm、Afterpay/Clearpay、Klarna、PayPal Credit等。

**💱****多货币支持：**

实际支持数量取决于所选网关。汇率处理：Spreedly平台自身不提供汇率锁定或动态货币转换功能，依赖于所集成的网关服务。

### （二）合规与安全基石

**🔒****PCI-DSS Level 1认证：**

Spreedly是PCI-DSS Level 1（支付卡行业数据安全标准最高级别）合规平台，已列入Visa Global Registry和Mastercard SDP Compliant Registered Service Provider列表。

客户通过Spreedly提供的安全收集工具，可大幅减少自身的PCI合规范围。

**📜****监管牌照定位：**

Spreedly定位为支付流程编排技术平台，不是支付服务提供商(PSP)，因此无需持有MSB或EMI/PI牌照。

合规分工：Spreedly提供技术基础设施，资金结算和监管合规由客户集成的支付网关完成（需具备相应支付牌照）。商户需根据业务所在地的监管要求（如AML/KYC）选择合适的持牌合作伙伴。

**🔄****跨境清算路径：**

资金流不经过Spreedly，清算路径（如SEPA、ACH、SWIFT等）由客户与底层网关/银行直接完成。

## **四、技术能力与开发者生态**

Spreedly的技术架构围绕RESTful API构建，专注于提供高度稳定和可扩展的支付基础设施服务。

### （一）API设计与通信规则

**🔌****API类型与通信规则：**

API类型：标准RESTful API v1，所有调用必须通过HTTPS（TLS 1.2）进行Basic Authentication（环境密钥+访问秘钥）。

数据格式：支持JSON和XML双格式。

接口功能：API支持核心支付流程（授权、捕获、退款、作废）、支付方式管理、网关管理和路由编排。客户对象管理和订阅计费逻辑由商户系统或集成的支付网关处理。

**🛡️****支付方式安全收集：**

iFrame/Express API：提供嵌入式安全表单，在商户网站收集支付方式数据，数据直接传输至Spreedly Vault进行令牌化处理，确保商户系统不接触敏感卡数据。Express提供overlay方式收集支付信息，iFrame提供嵌入式表单。

**🔔****Webhook与事件系统：**

Spreedly主要采用API轮询机制，但在特定场景（如集成底层网关的Webhook）也支持事件通知。使用API轮询时，需在架构设计中规划调用频率和重试策略。

### （二）SDK与开发者工具

**📱****官方SDK支持：**

移动端SDK：官方提供iOS SDK（Swift），Android SDK主要用于3DS2场景，主要用于移动应用内支付和支付方式安全令牌化。

服务端SDK：Spreedly采用RESTful API标准，支持任何编程语言通过HTTP客户端直接调用，服务端集成可使用社区维护的第三方库或直接调用REST API。

**🧪****测试环境：**

沙盒环境：提供完善的测试环境，包括Test Gateway（Spreedly测试卡）和Sandbox模式（生产网关的沙盒版本），支持复杂集成场景测试（如3DS）。

**🔧****管理后台与调试：**

管理后台：提供交易列表查看、交易详情查看（请求/响应）、网关状态管理等基础调试功能。

优化服务集成：与FlexPay（失败交易恢复）等合作，提升交易成功率和营收恢复。

票务/旅游：SeatGeek、Hopper

## **六、区域合规与数据安全体系**

Spreedly的合规工作主要围绕支付数据安全和隐私保护展开，作为技术平台，其自身无需持有金融牌照。

### （一）金融监管合规

**📜****牌照合规责任：**

Spreedly不持有美国MSB、欧盟EMI/PI等支付牌照，不进行资金结算、清算或托管。

合规分工：Spreedly提供技术基础设施，资金结算和监管合规由客户集成的支付网关完成（需具备相应支付牌照）。商户需根据业务所在地的监管要求（如AML/KYC/资金托管）选择合适的持牌合作伙伴。

**🇪🇺****欧盟PSD2合规：**

Spreedly提供3D Secure 2.0解决方案，帮助客户满足欧盟SCA要求。

### （二）支付安全认证（最高级别）

**🔒****PCI-DSS Level 1：**

持有PCI-DSS最高级别的Level 1认证，并通过QSA（合格安全评估员）年度现场审核，已在Visa/Mastercard官方服务商列表确认。

数据安全：所有存储的支付卡数据使用AES-256加密，传输层强制要求TLS 1.2+（获得SSL Labs A+评级）。

**📋****SOC 2 Type 2审计：**

通过SOC 2 Type 2审计，涵盖安全性和机密性双重信任标准，且通过审计时"无例外"。

### （三）数据隐私合规

**🔐****GDPR/CCPA：**

GDPR：隐私政策包含GDPR要求的11项要素，已任命欧盟数据保护代表，并提供为用户提供行使其数据权利的渠道。

**🌐****数据跨境传输：**

已通过EU-US Data Privacy Framework认证，并遵循UK Extension和Swiss-US DPF，确保欧盟、英国、瑞士到美国的数据传输合规性。

### （四）AML/KYC与争议解决

**👤****AML/KYC：**

Spreedly不直接对最终用户进行KYC/AML，主要责任在于客户（支付服务提供商）。

平台通过集成Kount、Forter等第三方欺诈工具，辅助客户满足AML/KYC的合规要求。

**⚖️****法律管辖：**

服务条款规定适用美国北卡罗来纳州法律并采用强制仲裁机制，客户可在签约时协商仲裁条款。

**⚠️****注意事项：**

Spreedly的商业模式将资金和监管风险隔离在客户侧，客户必须全面理解其在本地的合规责任（如AML/KYC、资金托管、本地牌照要求），Spreedly的技术合规（PCI/SOC 2）不能替代客户的金融监管合规。

## **七、核心产品与支付编排能力**

Spreedly的产品线围绕"支付编排"的核心理念构建，旨在通过模块化服务解决全球支付中的常见难题。2025年9月，Spreedly收购Dodgeball以增强支付优化与欺诈编排能力。

### （一）Vault：支付信息存储库

**🔐****令牌化与安全存储：**

核心功能：提供PCI DSS Level 1合规的支付信息存储库，对信用卡、银行账户、数字钱包进行令牌化存储。

存储功能：令牌可跨网关、跨区域复用，实现支付网关的灵活切换。

高级存储服务包含网络令牌化与卡片信息自动更新功能，自动管理失效或过期卡信息。

**🔄****主动管理：**

借助网络令牌和卡片信息自动更新服务，自动更新或失效令牌，由底层网络或收单行提供卡信息更新。

**💼****应用场景：**

如果您是SaaS订阅公司，使用高级存储服务可以将客户支付方式令牌化后存储，即使客户的卡片信息过期，系统也能自动更新，大幅减少因卡片过期造成的周期性收入损失。

### （二）Connect：支付服务连接

**🔗****多网关连接：**

单一API：通过单一RESTful API连接全球支付网关和支付服务。

交易处理：支持标准的购买、预授权、捕获、退款、作废等交易流程。

### （三）Protect：欺诈防护

**🛡️****多层级欺诈管理：**

Protect模块包含Global 3DS服务，支持3DS 1.0及EMVCo认证的3DS 2.0，并集成底层网关Webhook以增强实时风险检测。

欺诈工具集成：支持将Forter、Kount等欺诈检测工具嵌入支付编排流程，实现实时风险检测。

基于人工智能技术：基于跨行业交易数据的AI模型（通过合作伙伴），帮助商户减少误报率和拒付率。

### （四）Optimize：支付优化

**🚀****智能路由：**

路由策略：基于地区、币种、支付方式、授权率、成本等条件自动选择最优网关。

故障重试：支持动态切换和故障自动重试机制，提升交易成功率。

**📊****交易分析与报告：**

管理后台：提供交易查询与监控、网关性能报告、支付方式分析、自定义报告功能。

## **八、适配场景与决策建议**

Spreedly作为支付编排平台，其价值在于为具备一定规模和技术能力、需要多网关策略来优化全球支付性能的企业提供基础设施。

**💡****Spreedly在跨境支付行业中的特点：**

Spreedly独特的"SaaS订阅+API调用计费"模式，不收取基于GMV的手续费，是中大型企业支付基础设施的可靠选择。

### 适合使用Spreedly的场景

**🚀****高成长/全球化平台**

用户特征：业务快速扩张至多个国家，年GMV达到千万美元级别，需要同时使用多个支付网关（如Stripe+Adyen+本地网关）。

核心需求：确保支付流程的连续性、提高授权率、降低PCI合规成本。

匹配理由：Connect产品可实现单一API集成所有网关，Vault可安全存储令牌，Optimize可实现智能路由优化性能。

客户案例：The New York Times使用Vault减少合规范围，实现全球支付方式统一管理。

**🔄****SaaS订阅服务商**

用户特征：严重依赖周期性付费，因卡片过期导致的客户流失率高。

核心需求：支付方式的持续维护与更新、减少流失。

匹配理由：高级存储服务提供卡片信息自动更新和网络令牌化，主动管理支付方式，大幅减少因卡片过期造成的周期性收入损失。

**🏢****寻求网关灵活切换的大型企业**

用户特征：希望避免被单一支付网关锁定，便于未来迁移或谈判费率。

核心需求：支付方式令牌的网关独立性。

匹配理由：Vault的令牌可移植且独立于特定网关，商户可随时切换支付网关，无需重新收集客户支付信息。

**🔒****欺诈与安全要求极高的企业**

用户特征：对PCI合规有最高要求，需要集成多个反欺诈工具。

核心需求：PCI Level 1合规、多层级欺诈防护。

匹配理由：Spreedly自身的PCI Level 1认证和Protect产品提供3DS2、Forter/Kount等集成，满足严格的安全需求。

客户案例：SeatGeek通过Optimize提升授权率，同时利用Protect实现多层级欺诈防护。

### 决策建议

在选择跨境支付服务商时，Spreedly的独特模式要求企业决策者进行差异化评估：

**✅****业务需求匹配度**

Spreedly定位为支付底层设施，而非提供全方位服务的支付商。如果您需要发卡、贷款、完整的订阅计费系统或无代码收银台，Spreedly不适合作为唯一解决方案；如果您的核心需求是灵活编排支付流程、管理多个网关和简化PCI合规工作，则Spreedly是少数具备强大能力的平台。

**💰****收费模式与成本**

Spreedly采用SaaS订阅+API调用量模式，起价较高，不适合小微企业和个人开发者。

只有当企业的交易量足够大，通过Spreedly的智能路由优化能节省的网关交易费，能够覆盖其订阅和API调用成本时，Spreedly的性价比优势才能体现。

**🔧****技术实施能力**

Spreedly是一款为开发者设计的API驱动产品，集成需要较强的技术能力。如果您的团队技术资源有限，可优先考虑提供托管收银台或一键插件的支付网关（例如：Adyen、Stripe）。

**⚖️****商业风险考虑**

在签约Enterprise计划时，建议在商务谈判中明确API用量的阶梯定价、费用调整机制、合同期限和续签条款，这是企业级服务采购的标准实践。
