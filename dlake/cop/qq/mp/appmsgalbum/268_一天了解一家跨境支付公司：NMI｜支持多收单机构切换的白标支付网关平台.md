本文对跨境支付公司的界定：持支付牌照主营跨境收付款 / 换汇，或为该业务提供技术支撑的持牌服务商。

## **一、企业概况**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与主体**

2001 年成立，法定名称 Network Merchants, LLC，总部位于美国伊利诺伊州 Schaumburg。

**🌍****全球布局**

在美国（Schaumburg 总部、纽约等地）、英国 Bristol、南非 Cape Town 设有办公室，以远程办公为主。

**🛡️****合规资质**

持有 PCI DSS Level 1 认证，列入 Visa 与 Mastercard 合规服务商名录，遵循欧美主要数据保护法规（GDPR、CCPA 等）。

**📊****业务覆盖与核心定位**

NMI 面向 ISO、SaaS 平台、银行、PayFac 等渠道合作伙伴，提供白标（White-label，即合作伙伴以自有品牌对外提供服务）支付网关服务，NMI 在终端商户面前保持隐形。

**🎯****核心产品线**

支付网关（白标全渠道支付受理）；商户收单（NMI Payments，含商户审核与入驻）；商户关系管理（Merchant Central，CRM + 自动化商户审核）。

## **二、计费结构与服务规则**

**📋****定价模式说明**

NMI 采用 B2B 批发定价模式，不在官网公开标准费率。合作伙伴与 NMI 协商确定费率后，自行定价向下游商户销售；具体定价需联系销售团队获取。

**📝****服务项目清单**

**网关服务：** 网关月费；每笔交易处理费（网关层）

**收单服务（NMI Payments）：** 交易处理费（由收单机构收取的 Interchange + 加价费用）

**增值服务（独立定价）**

  * Kount AI 高级风控

  * Account Updater（自动卡号更新）

  * 3D Secure 认证服务

  * TXT2PAY（短信支付）

  * Network Tokenization（网络令牌化，即卡号加密替换）

  * Open Banking 服务




**商户关系管理（Merchant Central）：** CRM 平台 SaaS 订阅费

**内嵌贷款服务（NMI Business Capital）：** 单一前置费用，无复利、无滞纳金

**⚙️****计费规则**

合作伙伴可选择 Interchange Plus、统一费率（Flat Rate）、分层定价（Tiered）、混合费率（Blended）、订阅制（Subscription 与 Membership）等定价模型。NMI 提供 Fee Schedule API，支持为不同商户动态设置费率方案。

**⚠️****说明：** NMI 作为网关不直接处理资金流转。退款费、拒付费等通常由下游收单机构定价。终端商户的实际费用 = 网关费 + 处理费 + 合作伙伴加价。

本章聚焦收费项目与适用规则，不含数值，最终以《商户协议》为准。

## **三、全球化能力与合规体系**

###  （一）支付受理能力与地理覆盖

**💳****国际卡收单**

对接 Visa、Mastercard、American Express、Discover、Diners Club、JCB、中国银联，支持信用卡、借记卡、EMV 芯片卡（Chip & PIN）、非接触式支付（Contactless）、磁条卡。

**🔗****支付方式覆盖**

以卡支付为核心，另支持 Apple Pay、Google Pay、ACH、eCheck（美国银行转账）、Text-to-Pay（短信支付请求）、QR Code 支付、虚拟终端（Virtual Terminal，手动键入卡号）、标准收银页面（Hosted Payment Page）。

**⚠️****说明：** 本地支付方式的实际可支持范围取决于对接的收单机构。

**💱****多币种能力**

支持 100 余种货币（取决于对接的收单机构），提供动态货币转换（DCC，Dynamic Currency Conversion）功能，允许持卡人在交易当地货币与发卡国货币之间选择支付。结算货币取决于商户与收单机构的约定。

**🔄****跨收单机构架构（Processor-Agnostic）**

对接 Worldpay (FIS)、TSYS (Global Payments)、Fiserv (First Data)、J.P. Morgan、Barclaycard、Elavon 等 150 余家收单机构。

**📍****区域覆盖**

###  （二）合规与安全体系

**🛡️****安全认证**

  * PCI DSS Level 1：自 2005 年起连续通过认证，覆盖 NMI ONE、Cardease、OMNI、USAePay、Merchant Central、ScanX、TransactionIntel 等产品线，5 份 AOC（Attestation of Compliance）文件公开可下载

  * PCI P2PE（点对点加密）：多款终端设备认证

  * PCI SSC 参与组织成员：持续参与支付安全标准制定

  * Visa Global Registry + Mastercard Compliant Service Provider List 双重注册


**🔐****数据加密与网络安全**

SSL/TLS 传输加密；RSA 4096 位卡数据存储加密；P2PE。网络层面采用 DDoS 防护服务、网络隔离架构与严格防火墙规则。

**🏭****基础设施**

北美、英国、欧洲多数据中心部署，定期渗透测试与漏洞扫描。

**📄****数据保护**

隐私政策覆盖 EU GDPR、UK GDPR、CCPA/CPRA、PIPEDA，提供数据处理附录等合规文件，数据存储节点分布于美国、英国、荷兰、加拿大。

## **四、技术能力与开发者支持**

**🔌****API 体系**

提供现代 RESTful API（v5）和兼容旧系统的 Classic API 两套接口。API 端点 100 余个，覆盖在线支付（Sale、Auth、Capture、Void、Refund、Credit、Validate）、订阅管理、发票、客户管理、商户入驻（Merchant Sign-Up）、终端设备管理、报表查询等主要操作环节。

**📱****客户端与终端集成**

  * Collect.js：前端令牌化库（JavaScript），支持嵌入式表单与弹窗模式

  * Collect Checkout：标准收银页面

  * Gateway.js：3DS / 高级服务支持

  * 终端设备 SDK：覆盖 iOS、Android、Windows、Linux 四平台

  * Customer-Present Cloud：浏览器 POS，无需安装 SDK




**⚠️****说明：** NMI 的后端集成方式为直接调用 RESTful API，不依赖特定语言的 SDK 包。API 文档主要提供 cURL 和 JavaScript 示例。

**🛠️****开发者工具**

提供沙盒测试环境、网关模拟器、Postman Collection 等开发者工具。

**🔔****Webhook 事件系统**

覆盖交易、订阅、结算、拒付等多种事件类型，提供重试机制。

## **五、合作伙伴管理与集成生态**

###  （一）购物车与软件集成

**🛒****购物车**

支持 125 余个购物车平台，包括 WooCommerce、Magento、Shopify、3dcart、Joomla、WordPress、Drupal 等，提供 Shopify 商务官方扩展

**📊****会计软件**

QuickBooks、Xero（通过 Bill Connect 扩展）

**🤝****CRM**

Salesforce（通过 Chargent 第三方集成）

### （二）合作伙伴管理

**🤖****商户入驻自动化**

  * ScanX 自动商户审核：自动执行 KYC、KYB、AML 检查，生成风险评分与处理建议，低风险商户自动审批通过，高风险商户转入人工审核

  * TurboApp：将商户入驻时间从 30 余分钟缩短至 5 分钟以下

  * MonitorX：持续监控已入驻商户的风险状况


**🏢****Portal 体系**

Partner Portal（合作伙伴管理门户，含商户入驻、费率管理、收单机构配置、服务管理、白标定制）+ Merchant Portal（商户门户，含虚拟终端、交易管理、客户管理、报表、设备管理、API 密钥管理）+ Merchant Central（独立 CRM 平台）。支持完全白标定制（Logo、颜色、URL、域名）。

### （三）服务交付与支持

**📞****支持渠道**

电话、邮件、在线帮助中心、聊天机器人、开发者文档 AI 问答

**📚****教育资源**

Payments 101 支付基础知识专栏、博客、合作伙伴案例研究、电子书、播客（The Signal）、网络研讨会

## **六、增值服务与行业方案**

###  （一）风控套件（Fraud Defense Suite）

**📏****规则引擎（iSpyFraud）**

自定义交易金额、频率、IP 地址、地理位置、黑名单等过滤规则，商户完全自主配置，支持批准/拒绝/触发 3DS 等动作

**🤖****AI 防欺诈（Kount，扩展服务）**

AI 驱动的全球数据分析，实时欺诈检测与行为分析

**🔄****卡测试检测**

算法识别批量测试卡行为，自动限制嫌疑账户并实时告警

**✅****认证与验证**

3D Secure 2.0（支持 Visa Secure、Mastercard Identity Check、Amex SafeKey，符合欧洲 SCA 要求）；AVS 地址验证；CVV 安全码验证

### （二）支付优化工具

  * Multi-MID 管理与路由：支持在单一网关下管理多个 MID（商户标识号），可基于地理位置、货币或产品线将交易路由至不同的商户账户。对于跨区域经营的商户，可通过不同 MID 实现本地货币结算，简化对账流程

  * ATRI（Automatic Transaction Routing Interface）：基于交易量或卡类型将流量自动分配至不同 MID

  * MID Waterfalling：交易被拒绝时自动通过多个 MID 重试

  * Account Updater：自动更新存储的过期信用卡信息，减少交易失败

  * Network Tokenization：网络级令牌化，提升授权成功率




### （三）内嵌贷款服务（NMI Business Capital）

由 Parafin 提供技术支持、Celtic Bank（FDIC 保险）发放贷款，面向 NMI 合作伙伴的美国商户，基于交易历史数据预审批。

### （四）行业方案

行业覆盖：零售、交通运输、停车、无人零售、自助服务（Vending、Kiosks）、酒店、餐饮、医疗、非营利、教育、EV 充电站、洗车、洗衣。NMI 在无人值守、自助支付领域积累了终端设备集成能力和行业经验，支持完整的终端设备 SDK + Cloud POS + P2PE。

## **七、适用场景与选型参考**

###  适合使用 NMI 的场景

**📈****ISO / 独立销售组织**

如果您是 ISO 或支付经销商，需要白标支付网关向下游商户提供支付服务，NMI 提供跨收单机构架构与白标定制能力，从商户入驻到费率管理均可在平台内完成。

**🖥️****SaaS 平台嵌入式支付**

如果您的 SaaS 产品需要内嵌支付能力，NMI 提供从无代码（标准收银页面、支付按钮生成器）到全控 API 的灵活接入方式，合作伙伴可通过交易分成获利。

**🏪****无人值守、自助支付场景**

如果您运营无人零售、自助服务、EV 充电站、停车等场景，NMI 提供完整的终端设备 SDK 和 Customer-Present Cloud（浏览器 POS），支持 EMV 芯片卡、非接触式支付、Tap to Pay（iPhone 与 Android）、P2PE。

**🇺🇸****北美市场卡支付处理**

如果您的业务以北美市场为主，以信用卡、借记卡为主要收款方式，交易路由功能（ATRI + MID Waterfalling）支持收单机构切换与授权成功率的调优。

**🚫****受限品类的支付接入**

部分行业（如电子烟、保健品、订阅制服务等）在部分支付平台上开户审核较为严格。NMI 的跨收单机构架构允许合作伙伴为这类商户对接专门服务受限品类的收单机构，同时 Multi-MID 管理和 iSpyFraud 规则引擎可为拒付率较高的业务场景配置专门的风控策略。

### 决策建议

在评估支付基础设施方案时，建议从以下几个方面做评估：

**🎯****业务模式匹配度**

如果您需要直接开通支付账户进行收款，可了解 Stripe、Adyen、Checkout.com 等直接面向商户的支付服务商。

**🌍****市场覆盖需求**

NMI 在北美和英国市场适配度较高。如果您的业务覆盖欧洲大陆、东南亚、拉美等区域，且需要接入本地支付方式（iDEAL、GrabPay、PIX、Boleto 等），可对比 Adyen、dLocal、2C2P 等本地支付覆盖更广的方案。

**💱****跨境支付需求**

如果您的核心需求为跨境收款与资金管理（如汇率锁定、多币种账户、本地支付方式聚合等），可同时了解 Airwallex、ONERWAY、PingPong 等专注跨境支付的服务商。

**🔧****技术实施能力**

NMI 不提供特定语言的后端 SDK，集成需直接调用 RESTful API。偏好现成 SDK 快速集成的团队可同时了解其他方案；有定制化需求的开发团队则适合使用 NMI 的 API 体系。
