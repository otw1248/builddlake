本文对跨境支付公司的界定：持支付牌照主营跨境收付款 / 换汇，或为该业务提供技术支撑的持牌服务商。

## **一、企业基本档案**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与主体：**

2004 年在荷兰创立，核心运营主体为 Mollie B.V.，母公司为 Mollie Holding B.V.

**🌍****全球布局：**

总部位于荷兰阿姆斯特丹，在伦敦、慕尼黑、巴黎、里斯本等欧洲多地设有办公室，其中里斯本办公室定位为技术中心

**🛡️****合规资质：**

持有荷兰央行（DNB）电子货币机构许可、英国金融行为监管局（FCA）支付机构许可证，是 PCI-DSS 1 级服务提供商，具备 PSD2 框架下的服务授权

**📊****业务覆盖：**

覆盖欧洲经济区（EEA）、瑞士和英国共 **30 个国家** ；支持 **29 种** 货币收款、**35 余种** （含国际卡、本地支付）支付方式，市场重心聚焦于欧洲区域

**🎯****行业渗透与应用场景：**

主要服务中小型及中型企业，覆盖家居园艺、食品饮料、服装、SaaS 和市场平台等垂直行业；具备平台型业务所需的分账、子商户管理等基础能力

## **二、全球化能力图谱**

###  （一）支付网络与区域布局

**💳****国际卡收单：**

支持 Visa、Mastercard、American Express 等国际卡网络，具备 3D Secure（3DS）、地址验证服务（AVS）、卡片留存及一键支付功能，广泛应用于订阅及重复购买场景，具备智能支付路由技术与动态路径选择能力

**🔗****本地支付聚合：**

整合 **35 余种** 欧洲本地支付方式，直连支持荷兰 iDEAL、SEPA Credit Transfer、SEPA Direct Debit，比利时 Bancontact、KBC/CBC Payment Button，以及波兰 Przelewy24、BLIK 等支付网络

**🔄****跨境清算路径：**

直接支持 SEPA 信用转账和 SEPA 直接借记；支持 SEPA Instant 即时信贷转账服务，在双方银行均支持的情况下，可实现 **10 秒内** 清算

**📍****区域枢纽：**

  * 🇳🇱阿姆斯特丹：全球运营指挥中心，负责高管管理、产品开发及工程研发

  * 🇵🇹里斯本：技术中心，支持商业拓展及客户支持服务

  * 🇬🇧伦敦：英国 FCA 监管主体运营中心


**📞****服务与语言：**

提供人工在线聊天及邮件支持服务，自动助手支持全天候（7x24）在线

### （二）合规与安全体系

**🏛️****牌照与认证：**

持有荷兰央行（DNB）颁发的电子货币机构许可、英国金融行为监管局（FCA）支付机构许可证，并通过 PCI-DSS 1 级服务商认证，相关资质由独立审核机构定期评估，且需通过持续合规检查维持有效性

**👤****KYC/AML：**

实施标准化 KYC/AML 流程，商户入驻需向平台提供公司注册证明、实益拥有人信息等资料，并接受持续合规监控；第三方 AML 数据库的对接以监管和合同要求为准

**💰****客户资金管理：**

客户资金均按监管要求独立托管，荷兰业务由 Stichting Mollie Payments 提供第三方资金保护，英国业务遵循 FCA CASS 规则实施客户资金隔离

**⚖️****合规运营：**

定期接受独立审计师的内部及外部审计，严格遵循荷兰 DNB 及英国 FCA 的监管要求，具体争议处理流程及制裁/出口管制条款按商户协议执行

## **三、技术能力清单**

###  （一）智能技术与架构

**☁️****云原生与多活：**

采用微服务架构，基于 Google Cloud Platform 部署，并利用 Google Kubernetes Engine（GKE）实现容器编排与管理；官方状态页面实时公开系统运维状态，多区域架构及其实现细节以企业官方披露为准

**🛡️****风控引擎：**

风控体系采用规则引擎与机器学习双栈架构，集成 AI 欺诈检测功能、灵活配置的黑名单及风险规则，具备实时交易过滤、AVS 和 CVV/CVC2 核查等能力

**📊****数据与洞察：**

系统支持基础交易状态查询、余额管理、交易报表等功能，Webhook 事件推送及时同步关键操作

**🔧****开发者生态：**

全部 API 采用 RESTful 设计规范，官方 SDK 支持 PHP、Node.js、Python 等主流编程语言，提供沙盒测试环境和测试专用的 API 密钥；官方插件支持适配 WooCommerce、Magento、Shopware 等主流电子商务平台

### （二）支付与结算效率

**⏱️****交易处理性能：**

支持实时及准实时支付渠道（如 Apple Pay、Google Pay、SEPA 即时转账），实现授权过程的实时确认

**💵****资金结算时效：**

国际卡收单与本地支付方式大多采用 T+1 工作日标准结算模式，商户可按日、周、月等多种频率结算；仅当可用余额超过€5 或等值币种时触发提现操作

**💱****多币种账户：**

系统支持多币种独立余额管理功能（最多 **12 种** 货币），交易费用按交易原始币种计收；资金归集与自动调度等进阶功能需参考具体服务协议

**📌****小结：**

Mollie 平台架构与技术能力专注于中小型商户的收单与风控刚需

## **四、计费结构与服务规则**

**🌐****跨境收款：**

采用混合费率（Blended Pricing，即综合定价）模型，将交易费、卡组织费等整合为单一费率标准；对于月处理量达到一定规模的大客户，可根据实际业务量获得定制化费率或采用 Interchange++（交换费 ++）定价模式；非 EEA 国家发行的卡片支付通常适用更高费率标准

**💳****国际卡收单：**

标准费率适用于 EEA 消费者卡、EEA 商业卡和非 EEA 卡，不同发卡地区与卡片类型存在差异化费率结构，部分交易会涉及汇率转换费用

**🏦****本地卡/本地支付：**

欧洲各地本地支付方式实行专属计费标准，荷兰 iDEAL、SEPA Direct Debit 等采用固定费用模式，BNPL 服务如 Klarna 及 Riverty/AfterPay 则按地区实行差异化定价

**📤****提现与对公结算：**

在部分情况下，商户账户可享受免费提现额度，超出后按笔收取费用；跨币种提现可能会收取 **1%** 的费用；不同结算通道的具体规则以商户个别合同为准

**✨****增值服务：**

拒付处理通常会产生手续费；VAT 相关服务、虚拟卡、高级风控等进阶服务另行定价

**📝****费用项目与条款补充：**

Mollie 用户协议中可能包含交易处理费、增值服务费、第三方罚款等项目；可能要求商户预留一定金额的备付金以处理退款需求；退款手续费返还政策及费率以《商户协议》实际约定为准

**📌****小结：**

本章所列系规则与费用项目结构，不含具体金额；所有详细费用、权益及约束以《商户协议》和最新官方公告为准

## **五、汇率管理与资金流转**

**📊****汇率基准与数据源：**

Mollie 汇率参考银行间市场报价，并叠加一定价差后实行 "全包"（All-in Rate，即全包含费率）模式；适用于支付时的正向货币兑换，以及拒付、退款产生的反向货币转换费用

**🔄****换汇与对冲工具：**

平台支持实时货币兑换，对于非余额账户币种的支付，系统将自动兑换为主要账户币种；远期锁汇、批量换汇或自动汇率预警等功能，以官方服务列表及高级协议中实际开放的内容为准

**💵****资金账户与调度管理：**

支持 **12 种** 货币的独立余额管理功能；资金归集、资金内部调拨功能以实际协议条款为准

**⏱️****到账与清算时效：**

国际卡收单和多数本地支付方式采用 T+1（次工作日）结算机制；SEPA 即时转账功能实现 **10 秒内** 到账；商户可根据需求选择日结、周结或月结等结算周期

**📌****小结：**

Mollie 当前的资金流转能力聚焦多币种收款和标准结算；本章节涉及的资金操作规则，以官方服务协议及实时政策为准

## **六、生态对接与服务能力**

###  （一）电商平台/独立站接入

**📦****平台资金回款：**

Mollie 提供 Connect 解决方案，支持多方分账及自动化出款管理功能，并为 SaaS 平台和市场型客户提供白标支付服务

**🛒****独立站插件：**

官方已发布可适配 WooCommerce、Magento、Shopware、PrestaShop 等主流电商平台的插件，支持全额或部分退款、支付方式留存（Card-on-File，即卡片留存）、订阅支付、一键支付和多语言环境配置及结账流程定制

**💪****能力细项：**

平台能够实现留存支付方式与一键支付，支持订阅协议管理和退款流程，具备多语言及结算页面的个性化配置能力

### （二）企业系统（ERP/财务）集成

**🔌****ERP 与财务系统：**

Mollie 各类企业对接能力，以预置的适配层或产品连接器实际情况为准，自动生成会计分录、多实体组织管理等高级功能需依据专业服务清单进行评估

**📊****对账与凭证：**

系统支持自动生成 Settlement Reports（结算报告），并兼容 MT940、CAMT.053 等银行标准报文格式，同时支持通过 SFTP 或 REST API 自动下载对账数据的功能

**🔐****权限与合规：**

商户后台具备角色细分与权限管理、日志留存、数据脱敏等合规相关功能，具体方案以系统配置及合同为准

### （三）本地化网络

**🇪🇺****欧洲：**

支持 iDEAL、SOFORT 等主流本地支付方式，具备动态 3D Secure 认证能力，可应对 SCA（强客户认证）场景并结合 VAT 或 GST 相关服务；支持 SEPA 即时转账（覆盖欧洲超 **90%** 主流银行），相关上线材料、Webhook 稳定性、商户主体资质及 KYB（即商户尽调）等要求以官方文档为准

### （四）服务交付与支持

**🔄****实施流程：**

Mollie 为新商户上线提供官方上线流程引导和 Hypercare（即超期关怀）专项支持流程

**📞****支持体系：**

提供多渠道客户服务，包括在线人工聊天、邮件、知识库和帮助中心；本章节对接方案的技术细节，以官方开发者文档最新内容为准

**📌****小结：**

Mollie 的生态及对接能力专注于电商平台和 SaaS/平台型业务的插件和 Connect 方案，企业级 ERP 集成能力需结合产品与服务包进一步核实

****七、增值工具与行业方案**

###  （一）风控套件（Acceptance & Risk）

**🧠****策略模型：**

风控引擎具备规则与机器学习双栈模型，支持风险规则及阈值的灵活配置，支持动态 3D Secure 认证

**📋****场景模板：**

针对特定行业的预设的风控规则模板请参见官方服务方案说明

**⚖️****拒付管理：**

平台提供拒付处理和自动预警服务，具备举证材料包自动生成等能力，细节以产品和服务清单为准

### （二）汇率与资金（FX）

**💵****账户与结构：**

系统支持多币种余额账户，最多支持 **12 种** 货币的独立管理

**🔄****换汇与对冲：**

可依据银行间市场汇率提供实时货币转换服务；远期锁汇、批量换汇、汇率自动化预警等高阶功能以官方服务清单为准

**🛣️****路线选择：**

系统可基于成本、速度等因素动态选择最优出款路径

### （三）数据与对账

提供 Webhook 系统，可实时发送事件通知，支持幂等机制和回放重试等精细化能力

**📊****报表与导出：**

智能生成 Settlement Reports（结算报告）等对账类报表，兼容 MT940、CAMT.053 标准格式，并支持通过 SFTP 或 REST API 自动下载

**🔐****数据治理：**

平台支持数据驻留地的选择、数据保留周期配置和数据脱敏处理等功能，实际约定以数据处理协议为准

### （四）行业方案

**🖥️****SaaS/市场平台：**

Mollie Connect for Platforms 解决方案可满足多方分账、子商户入驻进件与管理等业务需求

**🔄****订阅制：**

平台可支持订阅全生命周期管理以及试用期、自动化扣款重试和催缴等功能

**📌****小结：**

Mollie 的风控、对账与资金管理等增值工具以平台核心业务为基础，所有进阶功能须以官方产品包、服务协议为准

## **八、适配场景与注意事项**

###  （一）适配场景

  * **🇪🇺****欧洲本地化支付需求强烈** ：适用于以欧洲地区为主要市场、迫切需要 iDEAL、SEPA、Bancontact 等多种本地支付方式以提升转化率的业务

  * **🛒****独立站与平台业务并行** ：适合同时运营 Shopify、WooCommerce 等独立站和平台型业务的商户，可借助 Mollie 实现单一接口管理支付与资金

  * **🏢****平台型业务场景复杂** ：适用于电商市场平台、SaaS 类业务场景，支持多方分账、子商户入驻与批量出款等复杂资金操作

  * **🔄****订阅制与 SaaS 模式** ：适用于核心业务为循环计费的企业，提供完整的订阅生命周期管理与账单处理能力




### （二）注意事项

  * ⚖️**合规** ：需确保合同签署主体与监管主体一致；严格遵守 PSD2（欧盟支付服务指令）、SCA（强客户认证）等欧洲当地监管要求，所有重要业务数据符合 GDPR 等欧盟数据保护立法的要求

  * 🔧**技术** ：商户应在正式上线前完成沙盒环境的联合调试及充分测试，重视 Webhook 重试与幂等机制，以及确保全部集成模块符合 PCI-DSS 1 级标准

  * 💰**资金** ：提现账户需为公司法定对公账户，提现频率及金额可能存在一定限制，具体规则以官方协议为准；不同币种兑换可能产生汇率价差，需留意成本

  * 👥**运营** ：严格配置风控规则与阈值；重点关注客户支持渠道与服务时间安排，非紧急需求建议优先查阅知识库




## **九、区域合规与适配方案**

###  （一）欧洲

**🛡️****准入与合规要点：**

商户接入需遵循荷兰央行（DNB）和英国金融行为监管局（FCA）相关监管规定，全面符合欧洲支付服务指令（PSD2）及强客户认证（SCA）规定；所有重要业务数据符合 GDPR 等欧盟数据保护立法的要求，具体数据存储地点以官方披露为准

**🏢****运营与风控重点：**

平台需配置动态 3D Secure 认证机制，以应对 SCA 强制应用场景；涉及拒付举证、争议流程等实际操作内容，均以双方签署的合同条款为准

**📋****上线清单：**

商户上线时应准备好完整的 KYB（即商户尽调）资料及公司注册相关证明文件，配置 Webhook 端点并完成所有成功与失败支付场景用例的集成测试

### （二）其他区域

**🌐****业务范围：**

Mollie 当前的业务重心主要集中在欧洲市场，合规、技术及服务体系均围绕欧洲本土政策框架构建；非欧洲区域适配与进入方案需以实时市场环境和官方通知为准

**📌****小结：**

Mollie 的市场和产品体系聚焦于欧洲本土，非欧洲区域业务需结合最新官方公告及实际市场要求评估其可行性及适配方案
