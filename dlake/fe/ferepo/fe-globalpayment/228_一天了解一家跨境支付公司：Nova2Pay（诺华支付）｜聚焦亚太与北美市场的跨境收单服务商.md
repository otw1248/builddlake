本文对跨境支付公司的界定：持支付牌照主营跨境收付款 / 换汇，或为该业务提供技术支撑的持牌服务商。

## **一、企业概况**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与公司架构**

2013 年 11 月香港公司 Nova2Pay Limited 注册成立，2016 年 2 月英国子公司设立。

**🌍****全球布局**

运营总部设在香港，在上海、洛杉矶、东京、伦敦、新加坡设有办事处。

**🛡️****合规资质**

  * 金融牌照：香港 MSO、美国 FinCEN MSB

  * 安全认证：PCI DSS Level 1、ISO 27001


**📊****业务覆盖**

支持 **190+** 种本地支付方式、**187** 种交易货币、**15** 种结算货币。

**⚠️****说明：** 因统计口径差异，具体覆盖范围以签约时的合同清单为准。

**🎯****主要应用场景**

聚焦跨境电商、航空旅游、数字娱乐等行业，提供国际卡收单、本地支付接入、全球收款账户、虚拟卡发行等服务。

  


## **二、服务项目与计费说明**

**📋****服务项目清单****🌐****跨境收款**

按交易规模、行业类型、风险等级等因素采用定制化计费模式。

**💳****国际卡收单**

覆盖 Visa、Mastercard、AmEx、JCB、Discover、Diners Club。

**🔗****本地支付接入**

整合支付宝、微信支付、Apple Pay、Google Pay、PayPal 等数字钱包，以及 SEPA、ACH 等银行转账方式。

**🏦****提现与结算**

支持 **15** 种结算货币，最快可 T+1 到账。

**✨****增值服务**

  * 虚拟卡/实体卡发行

  * 争议处理：拒付预警工具（Chargeback Alert Tool）

  * 外汇管理：多币种实时兑换、动态货币转换（DCC）


**📝****费用项目补充说明**

可能包含拒付处理费、汇率转换加价、订阅服务费等项目。定价策略遵循行业惯例，具体取决于行业类型、风险等级、交易量等因素。

**📌****小结：** 本章聚焦收费项目与适用规则，不含具体数值，最终以《服务协议》为准。

  


## **三、全球化服务能力**

###  （一）支付网络与区域布局

**💳****国际卡收单**

与六大国际卡组织直接对接，支持银联卡收单。

**🔗****本地支付聚合**

整合主流数字钱包与银行转账方式。

**🔄****跨境清算路径**

支持 SWIFT、SEPA、ACH 等主流通道，支持多币种兑换。

### （二）技术能力与开发者支持

**⚙️****API 架构**

采用 RESTful API 架构，通过 HTTPS 协议传输 JSON 格式数据，使用 API 密钥认证。

**🔧****开发者工具**

提供沙盒测试环境（Sandbox）、测试卡号、异步回调通知（Webhook）。

### （三）合规与安全体系

**👤****反洗钱能力**

接入 OFAC、UN、EU 等国际制裁名单筛查，覆盖全流程反洗钱环节：交易前 KYC/KYB 审核、交易中实时监控、交易后可疑交易报告。

**🔒****数据保护**

隐私政策覆盖 GDPR 核心要素（数据类别、处理目的、法律依据、数据主体权利），表示使用欧盟标准合同条款（SCCs）进行跨境数据传输。

### （四）风控引擎

**📡****数据信号**

支持多维度风险识别：设备指纹、地理位置、交易行为、黑名单筛查（制裁国家、政治敏感人物）。

**🤖****技术能力**

  * AI + 机器学习模型：实时交易监控，识别可疑行为

  * 3D Secure：支持动态 3DS 验证

  * 虚假卡识别：拦截盗刷交易


**⚖️****拒付管理**

提供拒付预警工具（Chargeback Alert Tool），在拒付正式发起前预警商户。

  


## **四、产品功能与服务能力**

###  （一）电商平台与独立站接入

**🖥️****托管收银台**

提供托管支付页面（Hosted Payment Page），兼容电脑、手机等多种设备。采用托管模式，由支付服务商处理敏感卡数据。

**🔌****插件支持**

提供 WooCommerce 官方插件。

**⚙️****API 集成**

提供 Web Services API，支持商户自建支付流程，支持创建支付、查询支付、退款等核心操作。

**🏦****全球收款账户**

为商户开设香港、新加坡等地的本地收款账户，支持主流电商平台（Amazon、Shopee 等）资金归集。

### （二）高级支付功能

**📤****批量付款**

支持批量转账，适用于供应商结算、服务费支付等 B2B 场景。

**🔄****订阅与周期扣款**

支持周期性自动扣款，商户可通过商户后台（Dashboard）管理订阅状态，适用于 SaaS、会员订阅等场景。

### （三）增值工具

**💳****虚拟卡服务**

提供虚拟卡和实体卡发行服务，可通过 API 管理卡片的开通、冻结、注销等操作，并设置消费限额，适用于广告投放、采购等用途。

### （四）服务交付与支持

**📊****商户后台**

提供商户管理后台，支持查看实时交易、生成对账单、发起退款、多用户分权限管理。

**📞****客户支持**

提供 24/7 中英双语客服支持，大型商户可安排专属客户经理。

  


## **五、行业方案与上手指南**

###  （一）跨境电商

**🛒****服务组合**

国际卡收单、本地支付接入、全球收款账户、WooCommerce 插件集成

**📝****注意事项**

  * 高客单价订单建议启用 3D Secure 验证，有助于管理拒付风险

  * 关注拒付预警工具的使用，及时处理争议交易

  * 不同平台的结算周期和币种存在差异，需提前确认

  * 东南亚钱包限额差异大（如印尼 DANA 普通账户月度限额约 2000 万印尼盾，Premium 账户限额更高），需提前在测试环境验证




### （二）数字娱乐

**🎮****服务组合**

国际卡与本地钱包并行收单、订阅扣款、风控工具

**📝****注意事项**

  * 虚拟商品交易需关注新用户首次充值欺诈、批量盗刷等风险

  * 建议启用设备指纹和交易行为分析

  * 游戏道具等小额高频场景需评估支付成功率




### （三）区域注意事项

  * 🌏**亚太** ：香港 MSO 牌照覆盖跨境汇款与货币兑换。东南亚各国钱包存在实名要求和限额差异

  * 🇺🇸**北美** ：需遵守《银行保密法》（BSA）反洗钱要求；高客单价与订阅并存时，建议分层风控策略；ACH 转账到账时效通常为 T+1 至 T+2

  * 🇪🇺**欧洲** ：通过 SEPA 转账接入欧元区；强客户认证（SCA）对转化率的影响取决于实现方式，建议设置豁免规则并优化验证流程；需关注 GDPR 数据保护要求




### （四）上手清单

**📋****开通前准备（运营/财务负责）**

  * 营业执照、法人身份证明（用于 KYC 审核）

  * 业务资料（网站/App 链接、行业说明）

  * 预估交易量与主要市场


**🔌****对接阶段（技术/运营负责）**

  * WooCommerce 插件安装或 API 联调

  * 沙盒环境测试，验证支付流程

  * 小额测试订单，确认对账数据准确


**🚀****上线后运营（财务/运营负责）**

  * 每日核对订单与资金流水

  * 根据资金需求安排提现

  * 定期查看风控数据，调整策略


  


## **六、适配场景分析**

###  适合使用 Nova2Pay 的场景

**🇨🇳****中国出海商户**

如果您的业务需要同时接入国际卡和中国移动支付，Nova2Pay 支持国际卡和中国移动支付的统一接入。

**🛍️****中小型跨境电商**

如果您使用 WooCommerce 搭建独立站，可通过官方插件接入，减少定制开发工作。

**🛡️****关注风控的商户**

如果您所在行业拒付率较高（如数字娱乐、航空旅游），可评估其风控工具是否满足需求。

**🌏****亚太与北美市场为主的企业**

如果您的主要市场在香港、东南亚或北美，Nova2Pay 在这些区域具备牌照或注册资质。

### 决策建议

在选择跨境支付服务商时，建议从以下维度进行综合评估：

**🎯****业务场景匹配度**

根据您的主要市场、客户支付习惯、交易规模等因素，对比多家服务商是否匹配您的业务需求。如果您的业务主要集中在欧洲市场且需要电子货币机构（EMI）牌照支持，可同时了解 Airwallex、Adyen 等方案；如果聚焦东南亚，可对比 2C2P 等本地化方案。

**🔧****技术实施能力**

评估您的团队是否具备 API 对接能力。如技术团队人手有限，可优先考虑插件集成等方式。

**💰****费用与合同条款**

Nova2Pay 采用定制化定价模式，建议在签约前明确计费规则、服务边界、数据安全、争议处理等关键条款，对比多家服务商后做出决策。
