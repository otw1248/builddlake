本文对跨境支付公司的界定：持支付牌照主营跨境收付款 / 换汇，或为该业务提供技术支撑的持牌服务商。

## **一、企业概况**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与主体**

2020 年底成立，法定名称 MoneyHash, Inc.，注册于美国。

**🌍****全球布局**

注册于美国，采用远程办公为主的团队协作方式，团队分布在美国、阿联酋（迪拜）、埃及（开罗）、尼日利亚、南非、沙特阿拉伯等约 9 个国家。

**🎯****业务定位**

MoneyHash 定位为支付编排（Payment Orchestration）平台，通过统一 API 连接多个持牌支付服务提供商（PSP），帮助商户构建、优化和扩展支付基础设施，核心市场为中东和非洲（MEA）地区，该区域支付方式碎片化程度较高，各国本地卡、钱包、现金支付并存。

**⚠️****说明：** 支付编排平台与传统支付处理商（PSP）的核心区别在于——编排平台本身不直接处理终端用户资金，而是作为技术中间层连接多个 PSP，通过智能路由分配交易至不同 PSP 处理。商户仍需通过底层 PSP 完成实际支付处理。这是支付编排行业的常见模式（如 Primer、Spreedly 等）。

**📊****基本服务能力概览**

集成网络：**300+** 预集成 API 连接（指可连接的 PSP 和支付网关数量，实际可用的终端支付方式取决于商户连接的 PSP 组合），覆盖 **100+** 市场

## **二、计费结构与服务规则**

**📋****计费模式**

采用 SaaS 订阅费 + 交易手续费的混合定价模式。定价策略遵循支付编排行业惯例，为非公开的定制化模式，具体费率需通过商务沟通获取。

**📝****计费项目清单**

  * 支付编排服务（Pay-in）

  * 支出服务（Pay-out、Bulk Payouts）

  * 订阅管理

  * 令牌化金库（Tokenization Vault）

  * 虚拟钱包

  * 数字发票

  * Payment Links


**➕****补充说明**

具体费率取决于交易量、业务规模、集成复杂度等因素，大型企业客户可通过交易量换取更灵活的费率条件。此外，可能包含退款处理费、拒付处理费、跨境交易附加费等项目。

**⚠️****说明：** 商户在评估总支付成本时，需同时考虑编排层费用与 PSP 处理费用两层成本结构。

本章聚焦收费项目与适用规则，不含数值，最终以《商户协议》为准。

## **三、全球化能力图谱**

###  （一）支付网络与方式覆盖

**🔌****PSP 集成网络**

通过统一 API 连接全球及区域 PSP，已确认的合作伙伴包括：

  * 全球 PSP：Stripe、Adyen、Checkout.com

  * GCC 区域：noon payments（覆盖 Mada、KNET、Benefit、Meeza、Omannet）、Tap Payments（持牌覆盖多个 MENA 市场）

  * 埃及：Fawry、Paymob

  * 伊拉克：Wayl（接入 ZainCash、FIB 等本地支付，2026 年 2 月）

  * 非洲：Paystack、Flutterwave、Yoco

  * 卡网络直连：Visa（直接合作，含网络代币化）、Mastercard（Merchant Cloud 接入，2025 年 12 月宣布）


**💳****支付方式覆盖**

  * 卡支付：Visa、Mastercard、AmEx、Discover、JCB、Interac、Mada（沙特本地卡）

  * 数字钱包：Apple Pay、Google Pay（文档确认）；更多本地钱包通过 PSP 覆盖

  * BNPL（先买后付）：Tabby（阿联酋、沙特，2025 年 10 月合作）、Amwal Tech（沙特信用卡分期，2025 年 11 月合作）、ValU

  * 本地支付：Mada（沙特）、KNET（科威特）、Benefit（巴林）、Meeza（埃及）、Omannet（阿曼），通过 noon payments 单一 API 激活

  * 银行转账：A2A 即时银行支付，通过 Lean Technologies 合作接入阿联酋市场（2025 年 5 月）；Pay-by-Bank 开放银行支付，通过 Spare 合作接入阿联酋市场（2026 年 1 月）

  * 现金支付：Fawry（埃及线下网点）

  * 加密货币：支持 crypto 支付，具体可用币种和通道取决于 PSP 配置


**💱****多币种能力**

支持多币种支付处理，API 文档示例涉及 USD、EGP，通过 noon payments 覆盖 GCC 货币（SAR、AED、KWD、OMR 等）。支持基于客户位置的动态货币配置。具体货币数量和结算货币取决于连接的 PSP。

### （二）合规覆盖与数据安全

**🔐****安全认证**

PCI-DSS 合规，采用 iframe 隔离与令牌化架构，卡数据不进入商户环境。

**📜****监管合规覆盖**

  * 通过 Tap Payments 覆盖多个 MENA 市场的支付牌照

  * 通过 noon payments 覆盖 GCC 本地支付合规

  * 通过 Fawry 覆盖埃及本地支付合规


**🛡️****数据安全**

  * API 通信：强制 HTTPS

  * Webhook 签名验证：采用加密签名机制验证通知来源真实性，支持时间戳防重放攻击




## **四、集成能力与开发者工具**

###  （一）集成方式与智能路由

**🔌****集成方式**

提供多种集成方式，覆盖从低代码接入到深度定制的不同需求：托管收银台（Hosted Payment Page）、嵌入式收银台（Embedded Experience）、JavaScript SDK、Direct API、移动端 SDK、Payment Links。采用 RESTful API 架构，支持幂等性机制，避免因网络异常导致重复扣款。

**🚦****智能路由引擎**

支持基于货币、金额、地理位置、卡 BIN（发卡行识别码）、风险评分等条件的动态路由规则，提供自动重试与故障转移（首选 PSP 失败时自动切换至备选 PSP）、成本优化路由，路由策略支持版本化管理，便于对比不同策略的实际效果。可视化流程编辑器支持无代码配置，通过 Dashboard 修改即生效。

**🔑****通用令牌化金库**

卡 Token 类型为 UNIVERSAL，可跨 PSP 使用，支持将已存储的卡信息切换至不同 PSP 处理。在需要切换或新增 PSP 时无需重新采集用户卡信息。

**🛒****Dynamic Checkout（动态收银台）**

支持根据客户所在国家、货币、订单金额等条件，动态决定展示哪些支付方式及展示顺序。商户可通过 Dashboard 自定义支付方式的 Logo、按钮文案、多语言提示等，无需修改代码。

### （二）开发者工具

**📘****SDK 与文档**

提供 JavaScript、React Native、Flutter、Android（Kotlin）、iOS（Swift）等前端和移动端 SDK，所有 SDK 近期均保持活跃更新，支持 Apple Pay、Google Pay 原生集成，界面语言支持英语、阿拉伯语、法语切换。服务器端集成通过 RESTful API + API Key 认证完成。

开发者文档托管于 docs.moneyhash.io，完全公开无需注册，包含快速入门、集成指南、Webhook 文档、实操教程（Recipes）等，API 参考页面提供多种语言的代码示例和交互式测试功能。

**🧪****沙盒与测试**

提供完整的 Test Mode 与 Live Mode 双环境，包含独立测试 API 密钥、内置测试场景（成功、失败、3DS 验证等）、Dashboard 内交互式 API 测试工具、订阅模拟测试，Test Mode 无交易费用。

**🔔****Webhook 与电商插件**

Webhook（支付状态变化时自动推送通知）支持多种事件类型，覆盖支付意图、交易状态变更、卡令牌、客户、订阅等类别。电商平台插件方面，提供 Magento 2 电商插件。

**📊****统一 Dashboard**

提供统一商户 Dashboard（dashboard.moneyhash.io），支持跨所有 PSP 和市场的交易监控与收入追踪、统一的交易状态码体系（区分可重试与不可重试的交易拒绝）、路由决策日志、多业务线管理、定时自动导出等。

## **五、核心产品功能**

###  （一）核心支付功能

支付操作：支持单次购买（Purchase）、预授权（Authorization）、延迟扣款（Capture）、部分捕获、撤销（Void）、全额退款、部分退款、多次部分退款。提供完整的金额状态记录（已授权、已捕获、已退款、可退款余额等）。

客户管理：提供 Customer 模块，支持客户档案创建、自定义扩展字段、按字段搜索筛选、关联多个支付方式（已令牌化的卡片）、关联虚拟钱包（多币种余额）。

### （二）订阅与周期性支付

提供 Subscriptions 产品，支持：

  * 订阅计划管理：可自定义计费周期、一次性费用、免费试用期

  * 订阅操作：升降级、切换计划、修改金额、暂停、取消（立即或到期结束）

  * 折扣配置：固定金额、百分比、适用周期数

  * 自动扣款与发票生成：配置自动扣款后，每个计费周期自动生成发票

  * 订阅测试模拟：支持推进下一周期、模拟客户付款

  * 失败支付恢复：同 PSP 重试、路由至替代 PSP、建议替代支付方式




### （三）虚拟钱包与发票

虚拟钱包服务：提供客户虚拟钱包模块，支持充值（Top-Up）、扣款（Debit）、退款至钱包、钱包支付（Pay with Wallet）等操作。

数字发票服务：支持一次性发票创建、添加多个产品明细（名称、描述、数量、价格）、自动税费计算、品牌自定义（Logo、品牌名、品牌色）、通过邮件、短信、社交媒体分享。

### （四）Platform Connect

提供子商户管理模块，支持：

  * 为子商户创建独立 Account

  * 控制子商户可连接的 PSP 数量和类型

  * 子商户 Dashboard 访问权限控制

  * 白标嵌入（White-label）：可将支付集成门户嵌入商户页面，展示商户自有品牌

  * 分账能力：支持平台与子商户之间按规则拆分资金




### （五）风控能力

MoneyHash 在支付编排层内置基础风控功能：

  * 频率检测（Velocity Checks，限制短时间内的重复交易）

  * 地理位置校验（Geolocation）

  * BIN 黑名单（Blacklisted BINs）

  * 3D Secure 控制：支持强制启用、跳过，或沿用 PSP 默认行为

  * Dynamic 3DS：根据交易风险等级动态决定是否触发 3D 验证，低风险交易可跳过验证以减少结账步骤

  * 基于风险评分的路由规则：可在路由流程中嵌入风控规则

  * 高级风控能力（机器学习模型、设备指纹、行为分析等）通过 PSP 覆盖




## **六、适用场景与选型建议**

###  适合使用 MoneyHash 的场景

  * **MEA 区域拓展企业**  
如果您的业务主要面向中东和非洲市场（阿联酋、沙特、埃及等），需要接入 Mada、KNET、Fawry 等本地支付方式，MoneyHash 通过多家本地持牌合作伙伴，提供 MEA 区域本地支付接入能力。

  * **多 PSP 管理需求**  
如果您同时对接多个支付服务提供商（如 Stripe + Adyen + 本地 PSP），需要统一管理交易路由和故障转移，MoneyHash 提供智能路由引擎与可视化流程编辑器，支持基于多维度条件的动态路由与故障转移。

  * **降低 PSP 锁定风险**  
如果您希望避免被单一 PSP 绑定，保持供应商切换灵活性，MoneyHash 提供通用令牌化金库，降低 PSP 锁定风险。

  * **订阅与周期性收费**  
如果您的业务涉及 SaaS 订阅、会员服务等周期性收费场景，MoneyHash 提供订阅管理功能，支持计划创建、自动扣款与失败支付恢复。

  * **移动应用收款**  
如果您的业务以移动应用为主，MoneyHash 提供 iOS、Android、Flutter、React Native 四个移动端 SDK，支持主流钱包原生集成。




### 决策建议

在选择支付编排平台时，建议从以下维度进行综合评估：

  * **业务区域匹配度**  
支付编排平台的价值与连接的本地 PSP 网络直接相关。MoneyHash 在 MEA 区域已与多家卡网络和区域 PSP 建立合作。如果您的业务主要集中在欧美或亚太市场，可同时了解 Primer、Spreedly、Gr4vy 等编排平台；如果聚焦撒哈拉以南非洲市场，可对比 Precium（原 Revio）等区域编排方案。

  * **编排层成本评估**  
使用编排平台会产生额外的编排层费用，商户可结合路由优化带来的成功率提升和成本节省，综合评估投入产出。通常月交易量达到一定规模的中大型企业更适合使用编排平台。

  * **技术实施能力**  
MoneyHash 提供多种集成方式，从无需开发的 Payment Links、托管收银台到全定制的 Direct API。如技术资源有限，可优先使用托管收银台或嵌入式方案实现快速接入。如需服务器端深度集成，可参考 API 参考页面提供的多语言代码示例。

  * **合规、安全与合同条款**  
在签约前，建议核实服务商的安全认证资质和目标市场的持牌覆盖情况，确保实际支付处理的合规性。同时务必与多家服务商沟通，明确编排层费率、PSP 费用的透传方式、退款和拒付处理费、合同期限等关键条款，选择最符合业务需求和规模阶段的方案。
