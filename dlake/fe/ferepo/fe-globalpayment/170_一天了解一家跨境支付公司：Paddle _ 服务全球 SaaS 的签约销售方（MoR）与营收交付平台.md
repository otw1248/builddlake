## **一、企业基本档案**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考

**📅****成立时间及主体公司：**

2012 年 8 月（Paddle.com Market Limited 在英国注册成立）；公司总部位于英国伦敦，是一家专注于服务软件行业的金融科技公司

**🌍****全球布局：**

总部位于英国伦敦；在美国（纽约、奥斯汀）、都柏林、多伦多、里斯本设有办公室，覆盖欧美核心市场的运营网络

**🏢****核心商业模式：**

采用签约销售方（Merchant of Record - MoR）模式；Paddle 作为其客户（SaaS 及数字产品公司）的法律销售实体，全权负责处理全球支付、销售税 / 增值税合规、欺诈防护及客户账单等所有交易相关的后端运营

**📊****业务覆盖：**

服务全球 **200 +** 国家和地区的数字产品公司；支付网络支持所有主流国际卡、PayPal、Apple/Google Pay 及部分区域性本地支付方式，支持 **30 种** 交易货币

**🎯****行业覆盖与应用场景：**

深度聚焦软件即服务（SaaS）、人工智能、移动应用及游戏等数字经济领域；提供集支付处理、订阅管理、税务合规、数据洞察（ProfitWell）于一体的营收交付平台（Revenue Delivery Platform）

## **二、全球化核心能力体系**

### (一) 支付网络与区域布局

**💳****国际卡收单：**

受理能力：支持 Visa、Mastercard、American Express、Discover、Diners Club、JCB、UnionPay、Mada 等主流国际卡组织的支付

收单模式：作为签约销售方（MoR），Paddle 通过与全球多家收单行合作来处理支付，而非以单一主要会员（Principal Member）身份直连

智能路由（Paddle Pilot）：提供多收单行架构的智能路由服务，利用 A/B 测试数据动态选择最优收单行，并在支付失败时自动级联至备用收单行重试，以提升授权成功率；已为部分商户提升高达 **4%** 的支付成功率

安全认证：支持 3D Secure 2 (3DS2) 认证，提供无感知身份验证和挑战流程，符合 PSD2 SCA 要求

**🔗****本地支付聚合：**

欧洲：支持 iDEAL（荷兰）、Bancontact（比利时）、Blik（波兰）、MB Way（葡萄牙）

亚太地区：支持支付宝、微信支付、UPI（印度）

韩国：支持 KakaoPay、Naver Pay、PayCo、Samsung Pay 及 22 种本地银行卡

拉美：支持 Pix（巴西）

数字钱包：支持 Apple Pay 与 Google Pay（支持订阅支付）及 PayPal

**🎯****战略定位：**

为 SaaS 和数字产品提供全球统一的支付体验，优先整合在全球范围内接受度最高的支付方式（卡、主流钱包）

**🔄****跨境清算路径：**

清算模式：作为 MoR，所有资金首先进入 Paddle 的账户体系

**📍****区域枢纽：**

  * 🇬🇧伦敦总部：全球战略、核心管理与运营中心

  * 🇺🇸纽约 / 奥斯汀办公室：北美市场业务拓展与客户支持中心

  * 🇮🇪都柏林 / 里斯本办公室：支持欧洲区域的运营


**📞****服务与语言：**

提供 **18 种** 语言的本地化结账与交易邮件支持

### (二) 合规与安全体系

**🏛️****核心监管模式（MoR）：**

法律责任归属：Paddle 作为签约销售方（MoR），是法律上的销售方，全权负责处理所有交易的合规事宜，包括自动计算、代收并在全球各税务管辖区代缴销售税（Sales Tax）与增值税（VAT），免除其客户自行处理复杂税务的责任

金融牌照：合规策略侧重于在全球范围内获得作为数字商品经销商所需的商业主体资质与税务登记，与传统支付服务商（PSP）所需的电子货币（EMI）或资金传输（MTL）等金融牌照路径不同

**📜****行业认证：**

SOC 2 Type 2：已成功完成 SOC 2 Type 2 审计，证明其在数据安全、可用性、数据处理完整性、保密性与隐私方面的内部控制有效性

PCI-DSS：符合 PCI-DSS SAQ A 合规标准，即不直接处理或存储原始银行卡数据，将卡信息处理外包给符合最高级别（Level 1）合规的服务商

**🛡️****欺诈与拒付管理：**

责任归属：作为 MoR，Paddle 承担所有欺诈性交易所导致的拒付（Chargeback）损失，为其客户提供 **100%** 的欺诈与拒付保障

争议处理：由 Paddle 的专业团队代表其客户处理所有与银行的拒付争议事宜

## **三、技术能力清单**

### (一) 智能技术与架构

**⚙️****平台架构：**

采用实体驱动（Entity-Oriented）的现代化设计，以提升开发者体验

**📈****系统可靠性：**

系统可用性服务等级协议（Uptime SLA）的具体数值，遵循服务企业级客户的惯例，在商业合同中进行约定

**🛡️****风控引擎：**

内建基于规则的支付路由决策引擎，并融合自有风险评分模型与第三方反欺诈软件

核心原理：通过风险评分系统自动拒绝高风险交易，支持实时风险评估和动态规则调整

**🔧****开发者生态：**

API 与 SDK：提供 RESTful API 及支持 Node.js、Python、Go、PHP 的官方 SDK 与 Paddle.js 客户端库；第三方社区还提供 Rust SDK 支持，覆盖更广泛的技术栈

开发工具：提供完整的开发者文档中心、公开的沙盒环境及支持签名验证的 Webhook 功能；沙盒环境完全独立于生产环境，支持完整的测试流程

预设集成方案：提供官方 Laravel Cashier 包，并通过第三方支持 WooCommerce、Shopify 等平台

API 设计：采用实体优先的设计理念，API 结构清晰映射业务概念，特别适合 AI 代理和现代开发工具集成

### (二) 支付与结算效率

**📈****授权成功率提升方案：**

通过 Paddle Pilot 智能路由等技术，平台致力于最大化授权成功率

**💵****资金结算时效：**

技术上支持灵活的月度结算模式

**🔄****资金管理技术：**

多币种支持能力：支持 **30 种** 货币的支付处理与 **5 种** 货币的结算（USD、EUR、GBP、AUD、CAD）

结算灵活性：商户可以选择在支持的结算货币中持有余额，减少外汇转换成本

## **四、计费结构与服务规则**

**📋****整体定价理念：**

采用全包式（All-Inclusive）的透明定价模型，将支付处理、税务合规、欺诈防护、订阅管理等所有服务整合为单一费率，无月费或隐藏费用

**💳****核心服务定价：**

Paddle Checkout：针对一次性支付和简单订阅的结账服务方案，采用基础交易费率

Paddle Billing：完整的订阅账单管理平台，包含高级订阅管理、发票处理、企业级功能，采用相同的透明费率结构

定制化定价方案：为大交易量企业、低价值产品（单价低于 10 美元）、需要发票功能或高级企业服务的客户提供定制化定价方案

**💱****货币兑换（FX）服务：**

汇率基准标准：使用实时市场汇率进行货币转换

保证金规则：在基准汇率基础上应用透明的外汇保证金

收款币种转换功能：支持多币种收款，客户可选择不同于支付货币的结算货币

**🏦****结算服务：**

本地转账：通过本地 ACH、SEPA 等网络转账的结算服务

国际转账：跨境银行转账的固定费用结构

**⚖️****争议处理服务：**

信用卡拒付处理：包含完整的争议处理和客户保护服务

PayPal 争议处理：针对 PayPal 交易的专门争议处理费用

费用返还政策：若 Paddle 代表商户成功赢得争议，将退还相关处理费用

拒付前预警服务（Pre-Chargeback Alert）：提供 48 小时预警窗口，主动退款以避免正式拒付，维持良好的拒付率

**✨****增值服务：**

咨询服务（Advisory）：为高交易量 SaaS 企业提供的专业咨询和成功管理服务

迁移支持：提供无风险的订阅迁移服务，确保现有客户不受影响

客户支持代处理服务：Paddle 团队代表商户处理所有与支付、订阅和取消相关的客户咨询

企业定制：包括定制实施支持、解决方案架构师服务和企业级 SLA

单页结账功能（One Page Checkout）：提供优化的结账体验，减少结账步骤以提升转化率

**📌****小结：**

本计费结构以透明性和简化为核心，将复杂的全球支付基础设施整合为统一服务，所有费用均以官方发布的《服务协议》为准

## **五、汇率管理与资金流转**

**📊****汇率基准与透明性：**

采用实时市场汇率作为汇率定价基础，所有货币兑换操作保持定价透明，平台会在终端明示相关汇率和转换规则

**📈****价差规则：**

在市场基准汇率基础上，平台将收取公开披露的货币兑换保证金，具体比例以服务协议和结算页面显示为准

**🔄****换汇方式：**

即期换汇：交易发生时，系统自动完成货币转换，并在后台统一归集 / 管理多币种收入

对冲工具：Paddle 定位为一体化营收交付平台，不提供远期锁汇、外汇对冲等专业财资管理工具

**💵****多币种账户体系：**

账户管理模式：作为 MoR，所有资金由 Paddle 统一管理，商户无需自行开设多币种账户；平台支持 **30 种** 支付货币与 **5 种** 结算货币（美元、欧元、英镑、澳元、加元）

高级功能：系统为简化全球销售未提供虚拟国际银行账户号码（vIBAN）及资金池归集等复杂财资功能

**⏱️****到账与清算时效：**

结算周期：所有交易汇总后按月结算，平台将于次月 15 日前完成付款

结算门槛：商户可自设月度结算门槛（最低 100 货币单位，支持 USD/EUR/GBP 等），余额未达门槛将顺延至下月

**📌****小结：**

以实时市场汇率和统一月结模式，Paddle 将多币种结算及全球资金流简化为一站式的透明服务

## **七、增值工具与行业解决方案**

### (一) 税务与合规解决方案

**🏛️****核心能力（MoR 内置）：**

Paddle 自动处理全球 **100+** 税务管辖区的销售税与增值税（VAT）的计算、代收与代缴，并承担全部相关审计及合规风险，实现端到端的全球税务合规，为 SaaS 客户提供 "无需自行处理合规、免注册、免申报" 的极简体验

### (二) 欺诈与拒付解决方案

**🛡️****核心能力（MoR 内置）：**

Paddle 全额承担所有欺诈交易及拒付损失，由专业团队负责所有争议处理和买家协商；同时支持拒付前预警（Pre-Chargeback Alert）机制，通过风险监控提前介入争议处理，若成功赢得争议，将为商户退还相关款项，极大降低商户风险与运营压力

### (三) 订阅指标与留存解决方案（ProfitWell）

**📦****产品化能力：**

通过收购并整合 ProfitWell，Paddle 提供业内领先的订阅财务指标分析与客户留存优化平台

**📊****核心能力：**

ProfitWell Metrics：免费提供 MRR（月度经常性收入）、Churn（客户流失率）、LTV（客户生命周期价值）等关键订阅指标实时分析

ProfitWell Retain：自动化客户留存工具，通过算法优化支付失败重试策略与客户沟通，降低非自愿流失

Price Intelligently：数据驱动的定价咨询服务，帮助 SaaS 企业制定最优价格及本地化定价策略

## **八、适配场景与注意事项**

### (一) 适配场景

  * **🌐****全球化 SaaS 与订阅制企业** ：适合以订阅和周期性计费为主、需合规出海且期望一站式合规与结算托管服务的公司，能够将全球支付、税务、合规运营等后端事宜整体外包

  * **💻****数字产品 / 软件开发者** ：适合需快速且低门槛地将数字产品销售至全球 200 多个市场、无需在当地设立实体公司或处理本地税务注册与申报的独立开发者和团队

  * **📈****注重收入优化的企业** ：对提升收入、持续分析订阅用户留存、降低客户流失，或有客户 LTV（客户生命周期价值）数据管理需求的增长型企业，依赖 ProfitWell Metrics/Retain 等分析工具

  * **🛡️****风险规避型企业** ：对欺诈、拒付和财务合规风险高度敏感，需将所有责任与损失外包给服务商、专注业务本身的数字企业




### (二) 注意事项

  * 📦**业务范围限制** ：Paddle 支持 SaaS 与数字产品，不支持物理商品及需实物配送的产品

  * 💰**成本结构** ：全包费率通常高于传统 PSP（如 Stripe、Adyen、PayPal）单纯的收单费率，但内含全球税务与合规、风险管理、争议处理等 "隐性服务成本"；建议对比自身 TCO（总拥有成本）进行综合决策

  * ⏱️**资金到账时效与对账方式** ：结算采用月度周期汇总出款，最快可在次月 15 日前到账；对现金流有高要求的企业需提前规划，且 Paddle 每月会提供清晰的结算明细与对账文件

  * 🔧**技术控制权** ：采用 MoR 模式后，商户对底层支付路由、收单行、部分风控逻辑等控制权有限，需接受平台托管管理，让渡部分控制权以换取全球合规和运营简易性

  * ⚖️**合规政策变更风险** ：出海企业需持续关注国际法规（如欧盟、美国等对数字服务企业的税务和合规新政），Paddle 负责及时更新合规方案，但仍建议企业内部保留合规审查能力




## **九、合规方案**

**🛡️****准入与合规要点：**

Paddle 作为全球统一的签约销售方（Merchant of Record, MoR），自身负责满足在全球 200 多个市场的商业主体合法性、税务申报及合规要求；客户无需单独在各国注册公司、申请支付牌照或进行增值税登记，极大简化出海准入门槛

**🏢****本地化运营及风控重点：**

Paddle 平台自动处理各市场的支付路由、3DS 认证（支持符合 PSD2 及欧盟等市场的 SCA（强客户认证）标准）、本地化结账语言和税务合规计算，为客户提供标准化、符合监管要求的全球化销售界面

**📋****上线清单：**

合规：客户只需通过 Paddle 的 KYB（企业身份验证，Know Your Business）审核，完成全球主服务协议（Master Services Agreement, MSA）的签署流程，一致适用于所有支持市场

技术：一次性完成平台对接与 API 集成后，即可实现所有市场的数字商品或 SaaS 服务无缝销售，无需针对不同国家和地区进行独立技术开发或合规方案维护
