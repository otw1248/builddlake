## **一、企业基本档案**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与主体：**

1999 年（前身 Direct Payment Solutions 在新西兰成立）；全球总部法律实体为 Windcave Limited（新西兰）。

**🌍****全球布局：**

全球总部及研发与硬件生产中心位于新西兰奥克兰；在澳大利亚、英国、美国等地设有主要办公室，服务网络覆盖亚太与欧洲。

**🛡️****合规资质：**

卡组织：作为 Visa、Mastercard、American Express 及 Discover Global Network 在其核心市场（包括大洋洲、北美、欧洲等地）的收单行。

行业认证：获得 PCI-DSS 4.0 Level 1、P2PE、EMVCo、ISO 27001 及 SOC II Type II 等多项国际安全与质量认证。

**📊****业务覆盖：**

服务全球多个国家和地区、超过 **15,000 家** 商户；支付网络支持所有主流国际卡、**20 余种** 本地支付方式（含 Alipay+ 生态）及 **25 余种** 交易货币。

**🎯****行业覆盖与应用场景：**

重点服务于零售、酒店、旅游、自动售货等垂直行业；提供覆盖线上（API）、线下（POS）、移动端及无人值守设备的全渠道（Omnichannel）支付解决方案，并支持订阅管理、多币种结算与动态货币转换（DCC）等能力。

## **二、全球化能力图谱**

###  （一）支付网络与区域布局

**💳****国际卡收单（核心能力）：**

收单行资质：作为 Visa、Mastercard、American Express 及 Discover Global Network 的全球收单行，并支持 JCB 与 UnionPay（银联），具备与主流卡组织网络的直接连接与清算能力。

安全认证：支持 3D Secure v1/v2，提供完整的浏览器与移动端认证方案，并支持动态触发机制。

网络令牌技术（Network Tokenization）：支持网络令牌技术（Network Tokenization），助力提升支付授权成功率并降低欺诈风险。

验证工具：支持地址验证服务（AVS）与卡片验证码（CVV/CVC2）校验，并提供多种检查等级配置。

高级功能：支持 Apple Pay、Google Pay，并清晰区分商户发起交易（MIT）与持卡人发起交易（CIT）。

**🔗****本地支付聚合：**

亚太地区：通过与 Alipay+ 合作，支付网络拓展至亚洲多个主流移动支付平台，包含中国的支付宝、菲律宾的 GCash、韩国的 Kakao Pay 等；在新西兰提供自有的账户对账户转账（A2A）方案。

北美地区：支持美国的 ACH 转账和加拿大的 Interac Online。

先买后付：支持先买后付（BNPL）服务中的 Zip（澳新）和 Klarna。

欧洲及其他区域：重点在于构建卡收单网络和与 Alipay+ 的生态合作。

**🔄****跨境清算路径：**

清算能力：作为卡组织收单行，直接参与卡组织网络的清算流程。

出款服务：对 Visa Direct/Mastercard Send 或区域性实时支付网络（RTP/FPS）的支持，基于其以传统卡清算为核心的业务模式。

**📍****区域枢纽：**

  * 🇳🇿奥克兰总部：全球总部、研发与硬件生产中心。

  * 🇬🇧伦敦办公室：欧洲区域中心。

  * 🇸🇬新加坡办公室：亚太区域中心。

  * 🇺🇸凤凰城办公室：北美运营中心。


**📞****服务与语言：**

提供 7×24 小时全球支持，并为各市场配备支付专家团队。

### （二）合规与安全体系

**🏛️****核心监管资质：**

金融牌照：合规策略侧重于获得卡组织（如 Visa、Mastercard）的收单行资质以及 PCI-DSS 等全球技术安全认证，并未在每个司法管辖区申请本地的电子货币（EMI）或资金传输（MTL）等金融牌照，业务模式以收单处理为核心。

**📜****行业认证：**

PCI-DSS：获得 PCI-DSS 4.0 Level 1 服务提供商认证，覆盖澳新、北美、欧洲等核心区域。

P2PE：获得点对点加密（P2PE）认证。

EMVCo：其终端设备符合 EMVCo 接触式与非接触式标准。

其他：获得 ISO 27001（信息安全）、SOC II Type II 及 ISO 9001（质量管理）认证（企业披露）。

**👤****反洗钱（AML）与客户身份验证（KYC）：**

流程与能力：作为支付服务提供商，遵循各司法管辖区的反洗钱（AML）与客户身份验证（KYC）要求。

**💰****客户资金管理：**

资金安全：客户资金隔离（Safeguarding）的具体条款，已明确与商户签署的服务协议中，且符合卡组织及 PCI-DSS 的安全规定。

## **三、技术能力清单**

###  （一）智能技术与架构

**⚙️****平台架构：**

提供基于 RESTful 和 SOAP 协议的 API，支持多测试环境（UAT/Demo），并运维公开的系统状态页面以监控核心服务组件。

**🛡️****风控引擎（Risk Management）：**

技术架构：采用规则引擎与基于数百万笔交易数据构建的机器学习风险评分算法相结合的双架构模式。

数据信号：整合交易速度分析数据、历史交易数据，并通过 3D Secure 流程采集设备指纹与浏览器信息。

商户操作支持：提供 Payline 在线门户，支持商户自定义风险规则、配置风险评分阈值及管理白名单、黑名单。

**🔧****开发者生态：**

API 与 SDK：提供 RESTful API 文档与 Postman 集成；支持 PHP、Java、.NET 等多种服务器端开发语言。

开发工具：提供独立的开发者文档中心、公开的测试环境及完整的测试卡号清单。

Webhook 回调机制：支持 notificationUrl 回调机制，并支持最多 6 次的自动重试机制。

预置集成方案：提供官方 WooCommerce、BigCommerce、Magento 插件，并支持与 Xero、NetSuite 等企业后台系统的集成。

### （二）支付与结算效率

**📈****授权成功率：**

通过网络令牌技术，助力提升支付授权成功率并降低欺诈风险。

**💵****资金结算时效：**

结算周期：API 返回 DateSettlement 字段，包含预计的资金到账日期信息，通常为 T+1。

结算截止时间（Cut-off Time）：支持商户自定义结算截止时间，以匹配业务运营节奏。

**🔄****资金管理技术：**

多币种账户：支持 **25 余种** 货币的交易与结算，并允许商户设置多个银行账户，按需为交易指定结算账户。

批量处理：支持批量处理服务，支持通过上传 CSV 文件处理批量交易。

## **四、计费结构与服务规则**

**📋****整体定价策略：**

提供 Blended（混合费率）、Interchange+（交换费加率）及 Interchange++（交换费加服务费加率）三种透明度递增的定价模型，满足不同规模商户的需求。

**💳****国际卡收单：**

基础费率：针对不同卡组织（Visa、Mastercard、Amex、UnionPay 等）制定差异化费率标准。

附加费用：对国际卡交易收取国际服务评估费（ISA fees）。

**🏦****本地支付：**

区域定价：对账户对账户转账（A2A，新西兰）、Alipay、WeChat Pay 等本地支付方式制定专属费率结构。

**⚖️****争议处理：**

处理费：每笔拒付（Chargeback）收取拒付处理费，费用标准依据商户协议约定，且在行业标准范围内。

费用转嫁：商户需承担所有由卡组织征收的罚金、费用与评估费用。

**📝****费用条款补充：**

涵盖账户开户费、月度管理费；协议通常包含自动续约与提前解约条款；所有费用按当地税率额外收取相应税费（如新西兰 GST、欧盟 VAT）。

**📌****小结：**

本章聚焦收费模式及费用构成，公开费率仅为市场参考，不构成最终报价，具体以商务谈判及《商户协议》为准。

## **五、汇率管理与资金流转**

**📊****汇率机制：**

DCC 服务：提供动态货币转换（Dynamic Currency Conversion）服务，允许持卡人选择以本国货币支付，汇率实时生成。

透明度：DCC 服务旨在为客户提供交易时的最终价格，避免汇率波动带来的价格不确定性。

**🔄****换汇与对冲工具：**

即期换汇：DCC 服务内置即期换汇功能。

对冲工具：该公司的产品定位为向商户提供支付处理与收单服务，而非提供专业财资管理服务，因此并未提供远期锁汇等外汇对冲工具。

**💵****多币种账户体系：**

账户结构：支持商户设置多个收单账户，可按交易的特定需求（如币种）结算至不同银行账户，实现资金按来源分离管理。

高级功能：公司的账户体系设计以简化多币种结算为目标，暂未支持虚拟国际银行账户号码（vIBAN）或资金池归集等复杂财资管理操作。

**⏱️****到账与清算时效：**

结算周期：交易结算时效通常为 T+1，具体日期通过 API 的 DateSettlement 字段返回。

结算截止时间（Cut-off Time）：支持商户自定义结算截止时间，为财务对账提供灵活性。

**📌****小结：**

通过动态货币转换（DCC）与多账户结算功能，为商户提供基础的多币种资金处理方案。

###  （一）大洋洲（核心优势区域）

**🛡️****准入与合规要点：**

在新西兰与澳大利亚拥有完善的法律实体与运营团队，高度符合本地商业法规及隐私保护法规。

**🏢****运营重点：**

提供账户对账户转账（A2A，新西兰）等本地化支付方案，并深度整合澳新地区主流的 POS 系统。

**📋****上线清单：**

准备澳大利亚商业编号（ABN）、新西兰商业编号（NZBN）等商业注册信息，配置本地银行账户用于结算。

**🛡️****准入与合规要点：**

通过英国、爱尔兰、立陶宛等地的法律实体运营，其 3D Secure v2 方案满足《支付服务指令 2》（PSD2）及强客户认证（SCA）要求，并符合《通用数据保护条例》（GDPR）要求。

支持强客户认证（SCA）豁免策略配置，以优化欧洲用户的支付体验。

**📋****上线清单：**

签署数据处理协议（DPA），配置 3D Secure（3DS）及强客户认证（SCA）策略，支持欧盟增值税（VAT）相关字段配置。

### （三）北美（成熟市场）

**🛡️****准入与合规要点：**

在美国设有运营实体，PCI-DSS Level 1 认证覆盖美国，符合本地数据安全标准要求。
