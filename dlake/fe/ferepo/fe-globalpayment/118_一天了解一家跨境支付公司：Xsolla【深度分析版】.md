## 一、费率体系全景解析

### 🔍 定制化费率机制

### 📌 费率机制关键点

**非标准化定价逻辑：** 区别于公开费率模式，其核心商业模式基于整体解决方案定制，费率受业务规模、支付方式组合、服务深度等多维度因素影响（如仅支付处理与全套商户即服务方案费率差异显著）

**准备金制度：** 有权根据商户风险状况（如财务状况、历史拒付率）要求保持一定金额准备金，从应付商户款项中扣留，用于覆盖潜在退款、拒付等义务

**增值服务收费：** 即时支付加速结算、高级分账功能、定制化技术支持等增值服务不设公开价，均纳入整体商业方案协商范围

## 二、汇率管理与资金流转

### 🔄 动态汇率体系

**定价机制：**

  * 汇率基准：基于合作银行或金融机构在支付当日提供的实时汇率，属于浮动汇率模式（非固定汇率）

  * 成本构成：汇率中包含汇兑利差（具体数值未公开），作为货币兑换环节的核心成本，在商业合同中单独约定




**货币处理能力：**

  * 多币种支持：实现 130 + 本地货币向结算货币（如美元、欧元）的自动化兑换，资金统一计入商户发行商账户

  * 汇率工具限制：官方未提及锁汇、远期结售汇等主动避险工具，主要按交易发生当日汇率进行兑换处理




### ⏱️ 资金流转时效

**资金流转效率优化：** 通过商户即服务模式，利用本地化实体和支付伙伴减少跨境链路，同时提供实时资金状态追踪功能，支持交易明细、退款、拒付等全流程可视化查询

## 三、合规架构与安全体系

### 🏛️ 资质与安全框架

### 🛡️ 风险与资金管理

**反欺诈技术体系：**

  * 专为游戏行业设计的机器学习模型，整合 20 + 过滤技术，包括实时交易监控、行为分析、设备指纹识别、IP 代理检测等，针对性拦截虚拟物品交易欺诈、自动化脚本攻击等行业特有风险




**资金安全机制：**

  * 基于商户即服务（MoR）模式，Xsolla 作为交易主体接收款项，通过合规运营及与受监管金融机构合作保障资金流转；标准条款中明确准备金制度，根据商户风险状况扣留部分资金以覆盖潜在义务




**合规流程：**

  * 执行商户 KYC 尽职调查，对终端用户交易进行持续 AML 监控；作为美国 MSB 需履行可疑活动报告义务；通过本地化合作确保跨境资金流转符合各国外汇及税务法规




## 四、生态对接与服务能力

### 🧩 行业专属集成能力

**游戏领域深度适配：**

  * 不对接通用电商平台，专注为游戏行业提供定制化集成方案：提供 RESTful API 及 Unity、Unreal Engine 等主流游戏引擎 SDK，支持支付、订阅、库存管理等功能深度嵌入；预构建游戏商城解决方案，帮助开发者快速搭建虚拟物品直销渠道




**支付体验优化：**

  * 其 Pay Station 支付界面可内嵌于游戏内或启动器，减少跳转环节；智能路由逻辑根据玩家地理位置、设备类型优先展示高转化率的本地支付方式，提升支付完成率




### 🌍 全球化服务网络

**区域运营中心：**

  * 全球布局关键市场办公室：洛杉矶总部（北美业务）、柏林办公室（欧洲 / 中东 / 非洲）、首尔办公室（韩国及周边市场）、吉隆坡办公室（东南亚区域），直接对接本地支付伙伴与监管机构




**多语言支持体系：**

  * 玩家帮助中心提供 10 + 语言服务（含英语、德语、日语、韩语、简体中文等）；通过工单系统及专属客户成功经理为商户提供支持，覆盖全球主要游戏市场语言需求




**本地化生态构建：**

  * 除支付网络外，通过合作伙伴网络解决方案连接意见领袖、营销机构等，提供自动化收入分账功能，形成支付 + 营销的本地化服务闭环




## 五、增值服务与行业方案

### 🛠️ 效率工具矩阵

  * 发行商账户：集中管理项目、交易数据、财务报表的控制中心，支持多角色权限分配，简化团队协作流程

  * 自动化报表系统：生成按时间、支付方式、区域等维度的销售报告，自动统计税费、退款、拒付等数据，减少人工对账成本

  * 反欺诈管理工具：提供 IP / 银行卡黑白名单配置、风险规则自定义功能，结合机器学习模型实时拦截异常交易

## 六、适用场景与企业类型

### ✅ 高匹配客户画像

**游戏行业全链条企业：**

  * 中小型 / 独立开发者：借助商户即服务模式快速实现全球化，无需自建合规团队；大型发行商：通过深度定制 API 满足高并发交易、复杂分账等需求，支持全球同步发行

  * 重点拓展新兴市场的企业：需覆盖拉美、东南亚、中东等支付方式复杂市场的游戏公司，可利用其预集成的本地支付网络降低接入成本

  * 追求支付体验优化的企业：希望通过内嵌支付界面、智能路由提升玩家支付体验的游戏团队，减少支付环节流失




### ⚠️ 实施建议

  * **费率确认：** 合作前需与业务团队详细沟通交易佣金、货币兑换点差、拒付处理费等所有费用项，确保合同明确约定各项数值

  * **技术评估：** 组织技术团队审核 API/SDK 文档与现有技术栈的兼容性，预估集成周期（无通用电商插件，需定制开发）

  * **结算规划：** 根据现金流需求选择结算模式（标准月度 / 即时支付），考虑银行处理时间与节假日影响，合理规划资金到账周期

  * **风险厘清：** 明确欺诈损失、拒付争议的责任划分机制，了解反欺诈系统运作流程及所需配合的证据提供要求

  * **服务约定：** 在合同中明确服务级别（如紧急问题响应时间、专属支持人员配置），匹配自身时区与语言需求
