本文对跨境支付公司的界定：持支付牌照主营跨境收付款 / 换汇，或为该业务提供技术支撑的持牌服务商。

## **一、企业基本档案**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考

**📅****成立时间与主体：**

2009 年（乌拉圭成立）；全球运营与合规体系主要依托于多个关键司法管辖区的持牌实体，控股公司位于开曼群岛

**🌍****全球布局：**

全球总部位于英国伦敦；在丹麦、巴西、马恩岛等地设有核心持牌运营实体与区域枢纽，团队分布于 **16 个国家**

**🛡️****合规资质：**

欧洲：持有丹麦金融监管局颁发的电子货币机构（EMI）牌照，具备欧盟护照资质

拉美：持有巴西央行颁发的支付机构（PI）许可证

其他：持有马恩岛 FSA 颁发的货币传输服务许可证

**📊****业务覆盖：**

支持 **10 余种** 货币的多币种账户；支付网络深度覆盖拉丁美洲、非洲及部分亚洲新兴市场，并连接欧洲与北美

**🎯****行业渗透与应用场景：**

深度服务跨境电商、在线旅游、游戏及全球薪资发放平台等行业；提供数字钱包（AstroPay Wallet）、商户收单解决方案（AstroPay Checkout）及嵌入式金融平台（AstroPay Platform）三大核心产品，支持本地支付聚合、多币种账户、全球出款及品牌卡发行

## **二、全球化能力图谱**

### (一) 支付网络与区域布局

**💳****国际卡收单与发行：**

战略定位：战略定位聚焦数字钱包生态与新兴市场本地支付，而非传统国际卡收单

发卡能力：通过卡 BIN 合作方（BIN Sponsor）模式提供 Visa 虚拟和实体卡，并可集成至 Apple Pay 与 Google Pay

收单能力：通过 BR-DGE 等支付调度平台或 Ecommpay 等合作伙伴提供多卡组织收单服务

**🔗****本地支付聚合：**

拉丁美洲：深度覆盖当地市场，可直接支持巴西 PIX、墨西哥 OXXO、哥伦比亚 PSE、阿根廷 RapiPago/Pago Fácil 等核心本地支付方式

非洲：支持肯尼亚的 M-Pesa，并在尼日利亚、加纳、乌干达等多个市场以本地合作伙伴方式运营

亚太地区：印度、中国、日本、泰国等地通过第三方合作伙伴或本地服务商实现覆盖，支持支付宝与微信支付

欧洲：作为 EMI 许可持有者，具备提供 SEPA、iDEAL 等服务能力，目前通过 SEPA Instant 支持欧洲实时转账

**🔄****跨境清算路径：**

支付网络对接：直接对接巴西 PIX、欧洲 SEPA Instant 等区域性实时支付网络

路由优化：通过与 BR-DGE 等支付调度平台的合作，为商户提供智能路由服务

**📍****区域枢纽：**

  * 🇬🇧伦敦总部：全球战略与欧洲运营中心

  * 🇩🇰哥本哈根分支机构：作为欧盟 EMI 持牌实体，是进入欧洲市场的合规入口

  * 🇧🇷圣保罗分支机构：作为巴西持牌支付机构，是拉丁美洲的核心运营与合规中心


**📞****服务与语言：**

提供 7×24 小时客户支持，团队分布于 16 个国家提供区域化服务

### (二) 合规与安全体系

**🏛️****核心监管牌照：**

丹麦 EMI 牌照：Larstal Denmark ApS 持有丹麦金融监管局颁发的 EMI 许可证，可通过欧盟护照资质服务整个欧洲市场

巴西 PI 牌照：获得巴西央行颁发的支付机构（PI）许可证，可在巴西合规运营

马恩岛 FSA 牌照：AstroPay Global (IOM) Limited 持有 8 类货币传输服务许可证（FSA）

**👤****KYC/AML：**

技术集成：集成 AI 驱动风控工具用于交易监控与警报，并对政治公众人物（PEP）及受制裁实体进行筛查

合规流程体系：实施完整的身份验证、交易监控与可疑活动报告（SAR）提交流程

**💰****客户资金管理：**

资金隔离：客户资金实行隔离存放，并每日进行对账，严格遵循各持牌地的监管要求

## **三、技术能力清单**

### (一) 智能技术与架构

**⚙️****平台架构：**

采用开发者友好型的 RESTful API 架构，后端技术栈为 Java，前端为 React；基础设施部署于 AWS 云平台及 Amazon Aurora 关系数据库

**📈****系统可靠性：**

该公司通过专属支持渠道与客户同步系统状态及服务等级，以此作为服务模式的核心

**🛡️****风控引擎：**

技术构成：搭建 AI 与机器学习风控系统，并整合 Telesign 的电话号码风险情报 API

风控效果提升路径：通过整合第三方数据与自有模型提升风控能力

**🔧****开发者生态：**

API 与 SDK：提供 RESTful API 及官方 JS/Node.js、React Native 版本 SDK

开发者支持资源：提供完整的开发者文档中心（developers.astropay.com）与沙盒环境

Webhook 机制：提供 Webhook 系统用于钱包绑定、交易状态变更等实时事件通知

官方插件集成：提供官方 WooCommerce 及 Magento 插件

### (二) 支付与结算效率

**⏱️****结算时效：**

实时结算：实现实时或近实时结算

即时到账：AstroPay 钱包间的转账免费且即时到账

**💵****资金管理技术：**

多币种账户：支持在单一平台持有、管理和转换 **10 余种** 货币

虚拟 IBAN（VIBAN）服务：通过与 OpenPayd 合作，提供虚拟 IBAN（VIBAN）服务，简化欧洲市场的收款流程

程序化支付能力：提供 Payouts API，支持程序化的资金提取与实时国际转账

## **四、计费结构与服务规则**

**📋****整体定价策略：**

主要采用针对企业客户的定制化报价模式，强调无隐藏费用，透明定价；具体费率与规则取决于交易类型、目标区域、风险评估等因素，通过商务谈判确定

**💸****核心服务定价：**

商户侧：需承担拒付、欺诈及退款相关的成本

用户侧：AstroPay 钱包间转账免费；银行提现费用视地域及卡发行方条款而定，通常在 0.3%–1% 区间；通过 Deel 等平台提现通常收取约 1% 的手续费，具体以 Deel 平台费率为准

**⏳****不活跃账户费：**

条款规定：在部分司法管辖区（如丹麦）的服务条款中，明确该公司有权收取不活跃账户费；个人用户条款中提及可能收取最高 20 美元 / 月的管理费

**📌****小结：**

本章聚焦收费模型与项目构成，未包含具体费率标准，所有费用均以双方签署的《商户服务协议》为准

## **五、汇率管理与资金流转**

**📊****汇率机制：**

汇率透明度承诺：承诺提供透明、具有竞争力的实时外汇汇率，并在兑换前清晰展示

**🔄****换汇与对冲工具：**

即期换汇：支持在多币种账户内进行即时的、按需的货币转换

汇率对冲支持：外汇服务除提供多币种账户内的即时按需货币转换外，还可辅助用户管理汇率风险

**💵****多币种账户体系：**

账户体系特点：提供支持 **10 余种** 货币的综合多币种账户，用户无需在当地设立实体机构即可进行全球收付款

虚拟账户（VIBAN）：与 OpenPayd 合作为用户提供链接至真实 IBAN 的虚拟参考账号，简化在欧洲市场的交易对账流程

**⏱️****到账与清算时效：**

即时支付网络对接：通过对接 PIX（巴西）和 SEPA Instant（欧洲），在关键市场实现实时资金流转

钱包内部转账：AstroPay 钱包间的内部转账免费且即时到账

**📌****小结：**

以多币种账户和 VIBAN 服务为基础，通过连接区域性实时支付网络，为新兴市场用户构建高效的资金流转能力

## **六、生态对接与服务能力**

### (一) 电商与平台

**🛒****独立电商网站插件：**

提供官方 WooCommerce 及 Adobe Commerce (Magento) 插件，支持订阅支付等场景需求

**🏢****平台能力（AstroPay Platform）：**

多方分账能力：支持支付流程中的多方分账（例如向上下游合作方拆分款项）及组合支付（如余额 + 银行卡）

大规模出款能力：提供 Payouts API 并支持通过 CSV 文件进行批量工资发放

用户账户管理能力：提供 API 支持为平台用户创建和管理国际账户

品牌化卡片发行服务：支持为创作者和流媒体平台发行品牌化的虚拟与实体卡片

### (二) 企业系统（ERP / 财税）集成

**🔌****集成策略：**

基于嵌入式金融基础设施的战略定位，提供开放 API 以支持与企业的大型 ERP 或财税软件的连接；具体集成方案可由商户或开发者根据自身需求灵活实现

### (三) 服务交付与支持

**📞****支持体系：**

全球支持：提供 7×24 小时客户支持，渠道包含应用内聊天、邮件及社交媒体

开发者专项支持：提供独立的开发者文档中心（developers.astropay.com）与 API、SDK 集成指南

**📌****小结：**

通过 AstroPay Platform 提供嵌入式金融能力，深度服务于平台经济，支持从分账、批量出款到品牌卡发行的全链路需求

## **七、增值工具与行业解决方案**

### (一) 风控套件

**🛡️****风控能力集成方式：**

风控能力作为平台内置服务提供

**🧠****技术核心：**

整合 Telesign 的电话号码风险情报 API，结合自有的 AI 与机器学习模型，在用户注册阶段即可进行风险评估

### (二) 财资与资金管理套件

**💰****资金管理能力集成方式：**

资金管理功能整合在 AstroPay Platform 与多币种账户中

**💪****核心能力：**

提供多币种账户、VIBAN、实时货币兑换及程序化的 Payouts API

### (三) 数据与对账自动化套件

**📊****数据与对账能力交付方式：**

数据与对账能力通过 Business Portal、API 及 Webhook 提供

**🔍****核心能力：**

商户可通过后台查看完整的结算历史与支付状态；Webhook 系统可实时通知交易状态变更

### (四) 行业方案

**🌐****全球薪资 / 自由职业者平台：**

方案核心组件：薪资钱包到账服务（Payroll Payments to Wallet）+ 多币种账户 + 实时跨境转账

能力亮点：支持企业通过 CSV 文件批量向全球员工的 AstroPay 钱包发放薪资，实现即时到账并降低成本

**🎬****创作者 / 流媒体平台：**

方案核心组件：品牌卡片发行 + 本地现金提现渠道 + 收益即时到账

能力亮点：为平台创作者提供品牌化的 Visa 卡，支持即时获取收益，解决传统银行服务覆盖不足的地区的提现难题

**🛒****跨境电商：**

方案核心组件：AstroPay Checkout + 拉丁美洲等新兴市场的本地支付方式

能力亮点：通过覆盖 PIX、OXXO 等关键本地支付方式，显著提升在新兴市场的支付转化率

**📋****附：快速落地清单**

  * **🔧****技术** ：访问开发者门户获取 API 密钥（API Key），利用沙盒环境及 JS/React Native SDK 完成集成开发

  * **💰****财务** ：在 Business Portal（商户后台）配置多币种账户，熟悉结算历史与报表功能

  * **🛡️****合规** ：准备商户尽职调查（KYB）材料，通过合规审核，签署商户协议

  * **👥****运营** ：熟悉 7×24 小时支持渠道，了解内置风控功能，向用户推广 AstroPay 支付选项




**📌****小结：**

将风控、财资与资金管理、数据能力整合为 AstroPay Platform 的核心基础设施，并针对全球薪资、创作者经济等特定场景提供深度嵌入式金融解决方案

## **八、适配场景与注意事项**

### (一) 适配场景

  * **🌎****深耕拉丁美洲的出海企业** ：核心目标市场是巴西、墨西哥、哥伦比亚等，对本地支付（PIX、OXXO、PSE）覆盖率有极高需求

  * **🏢****平台型 / 创作者经济企业** ：需要向全球自由职业者或创作者进行大规模、多币种薪酬结算发放，并希望提供品牌化卡片等增值服务

  * **🛍️****多渠道跨境零售商** ：在 WooCommerce 或 Magento 等平台运营，希望为新兴市场客户提供本地化支付选项

  * **🔌****有嵌入式金融需求的企业** ：希望通过 API 将支付、多币种钱包、出款等金融功能深度集成至自身产品体系中的科技公司




### (二) 注意事项

  * ⚖️**合规** ：商户在选择欧洲服务时应重点关注基于丹麦 EMI 牌照的合规路径与服务范围

  * 🔧**技术** ：开放 API 为连接 Shopify、BigCommerce 等主流电商平台提供标准路径；官方为部分特定平台提供预置插件

  * 💰**资金** ：商户协议中明确了在交易风险（如拒付、欺诈）管理中的责任归属；用户服务条款中包含与账户生命周期管理相关的费用说明，如不活跃账户费

  * 👥**运营** ：服务与支持体系以集中式在线渠道为核心，以确保响应的一致性与效率；对于有特定服务等级需求（SLA）的企业客户，可在商业协议中与客户商定定制化的服务条款




## **九、区域合规与适配方案**

### (一) 拉丁美洲

**🛡️****准入与合规要点：**

持有巴西央行颁发的支付机构（PI）牌照，确保在巴西的合规运营；API 及结账页面支持采集巴西 CPF/CNPJ 等本地身份信息及税号字段

**🏢****本地化运营：**

深度集成 PIX、OXXO、PSE 等核心本地支付方式，提供从票据生成、状态跟踪到核销对账的全流程服务方案

**📋****上线清单：**

准备当地商户尽职调查（KYB）及税务登记材料，配置并测试 PIX 等本地支付方式的全流程

### (二) 欧洲

**🛡️****准入与合规要点：**

通过丹麦 EMI 牌照及欧盟护照资质服务欧洲市场，业务遵循 PSD2/SCA 与 GDPR 法规

**🏢****本地化运营：**

支持 SEPA Instant（欧洲即时转账系统），并通过与 OpenPayd 的合作提供 VIBAN 服务

**📋****上线清单：**

签署数据处理协议（DPA），根据业务风险配置 3DS（3D Secure）/SCA（强客户认证）合规策略

### (三) 北美

**🛡️****准入与合规要点：**

主要通过合作伙伴模式进入市场

**🏢****本地化运营：**

适配北美市场以银行卡支付为主的用户习惯，并通过合作伙伴网络提供服务

### (四) 亚太及非洲

**🛡️****准入与合规要点：**

在马恩岛持有货币传输牌照，在亚太及非洲多个国家开展业务

**🏢****本地化运营：**

在非洲支持 M-Pesa，在亚太部分国家支持当地主流本地支付方式

**📌****小结：**

区域合规与适配能力以该公司在拉丁美洲和欧洲的深度持牌运营为核心，通过本地化支付解决方案在新兴市场构建竞争优势
