本文对跨境支付公司的界定：持支付牌照主营跨境收付款 / 换汇，或为该业务提供技术支撑的持牌服务商。

## **一、企业概况**

以下内容基于企业公开材料、监管公示与行业媒体整理，仅供参考。

**📅****成立时间与主体**

2019 年成立，总部位于澳大利亚悉尼，注册实体为 PAYFURL ANZ PTY LTD。

**🛡️****合规资质**

  * 行业认证：PCI-DSS Level 1 Service Provider（支付行业安全认证）

  * 行业组织：2022年加入 AusPayNet（Australian Payments Network）

  * 合规模式：作为技术调度层，自身不直接处理资金结算，资金结算合规责任由合作收单行承担。商户需与合作收单行（如 Windcave、Nuvei）单独签署合约并完成 KYB（企业身份核验）审核。


**🎯****业务定位**

支付调度平台（Payment Orchestration Platform），通过统一接口连接全球多个支付服务商，提供智能路由、独立令牌化存储、备用通道自动切换等技术服务。

**🔄****主要应用场景**

主要服务于电商订阅、慈善募捐、SaaS 平台等场景，提供多网关聚合、智能路由优化、跨渠道统一对账等能力。

## **二、计费结构与定价模式**

###  服务项目清单

**💳****交易处理**

按交易金额收取百分比费用与固定费用，采用 Interchange++（发卡行成本加成定价）或混合费率模式。

**⚙️****平台服务**

调度平台技术服务费，包含智能路由、令牌库存储、统一报表等功能。

**🤝****合作网关费用**

商户需单独向合作收单行（如 Windcave、Nuvei）支付交易处理费用。

### 增值服务

  * 3D Secure 验证

  * 网络令牌化

  * Webhook（事件回调）通知

  * 跨渠道对账报表




### 计费规则说明

**🏪****XPANSE 产品线（面向澳洲中小企业）**

采用透明定价模式，无开通费、不设最低承诺期限，具体费率可根据业务量定制。

**🏢****Enterprise 产品线（面向中大型商户）**

采用定制化报价模式，根据交易量、连接网关数量、路由规则复杂度等因素协商定价。

**📌****小结：** 本章聚焦收费项目与适用规则，不含具体数值，最终以《服务协议》为准。

## **三、支付能力与技术架构**

###  （一）支付网络与集成能力

**🌐****集成网关一览**

payFURL 通过聚合第三方支付网关实现全球支付能力：

**💱****多货币能力**

采用不限币种的设计，支持对接网关可处理的所有国际标准币种。支持多币种路由，可根据交易币种将资金分配至特定收单账户，实现原币种结算。

**🔄****智能路由（Smart Routing）**

  * 基于规则路由：根据货币、发卡行国家、交易金额等条件自动切换支付渠道

  * 备用通道自动切换：主通道失败时自动重试备用通道

  * 成本优化：将交易路由至费率最低的收单行（即最低成本路由）

  * 本地账户对接：支持跨国商家根据客户所在地接入当地收单账户，实现本地收单




### （二）合规与安全体系

**📜****认证与标准**

  * 3D Secure 2.0：借助集成网关，满足欧盟支付服务指令（PSD2）/强客户认证（SCA）合规要求

  * 《通用数据保护条例》（GDPR）：作为处理全球客户数据的平台，声明符合 GDPR 要求


**🔐****数据安全**

  * 传输加密：采用 HTTPS 和 TLS 1.2+ 协议

  * 存储加密：采用 AES-256 算法加密存储数据

  * 网络令牌化：将明文卡号替换为令牌（Token），降低数据泄露风险


**💰****客户资金保护**

采用非托管模式，资金直接从收单行结算至商户银行账户，payFURL 仅负责技术对接。

### （三）技术架构与开发支持

**⚙️****API 设计**

  * 架构风格：RESTful API，返回 JSON 格式响应

  * 认证方式：服务端密钥（Secret Key）+ 客户端密钥（Publishable Key）

  * 版本控制：支持多版本 API，兼容历史版本，便于系统升级过渡


**🔧****API 功能范围****📚****SDK 支持**

提供 .NET、PHP、Java、Python 四种语言的官方 SDK，均发布在 GitHub 并持续维护。

**🔔****Webhook 与事件**

支持支付状态变更、退款结果等关键事件的主动通知，采用签名验证机制确保通知来源可信。

**🧪****沙盒环境**

提供独立的沙盒环境接口地址和测试密钥，与生产环境隔离，包含测试专用卡号以模拟成功、失败、拒付等场景。

### （四）区域市场操作要点

**🇪🇺****欧洲**

PSD2/SCA 强客户认证为强制要求，强制 3DS 验证可能影响转化率，可设置强认证豁免规则来降低影响。GDPR 跨境数据传输需确保数据处理协议完备。订阅与分期业务需设置失败重试机制。

**🇧🇷****拉美**

Boleto（巴西线下支付凭证）通常需 2～3 个业务日完成确认，部分服务商提供加速版本可缩短至数小时内。建议在订单确认邮件中注明支付截止时间，并设置到期提醒与催款流程。税费与手续费拆分规则需与财务部门提前确认。

**🇸🇬****东南亚**

各国电子钱包的交易限额因账户等级、用户类型而异，且受当地监管政策影响。以印尼 DANA、泰国 TrueMoney 为例，高级认证用户与普通用户的单笔及月度限额存在明显差异，部分国家还针对特定群体设有额外的监管限制。建议在接入前向当地服务商确认最新限额政策。

**🇺🇸****北美**

高客单价与订阅业务并存时，建议配置分层风控策略。平台回款通道与独立站收单通道建议分离管理。拒付率较高的行业需提前准备拒付申诉材料自动生成流程。

**🌍****各区域接入建议：**

在测试环境提前验证当地支付规则，并开启多通道切换与自动重试功能。

## **四、服务生态与附加功能**

###  （一）第三方平台集成

**🔄****订阅管理平台**

Chargebee 官方支持通过 payFURL 网关集成 Windcave、Nuvei、dLocal、Ecentric、Metrics Global、NMI 等支付服务，支持复杂的订阅计费、支付失败自动重试及催款功能。

**📊****CRM 集成**

通过 Payments2Us 插件集成 Salesforce，支持在 CRM 内直接发起收款。

**🛒****电商平台**

提供 Magento 2、WooCommerce 官方插件。

### （二）风控能力

**🛡️****风控引擎**

集成合作网关（如 PayPal、Braintree）的风控能力，并提供内置风控规则系统。

**⚙️****风控功能**

  * 规则系统：支持黑白名单、阈值设置、组合条件

  * 验证机制：AVS（地址验证）、CVV（卡片安全码）验证

  * 强认证：3D Secure 2.0 规则设置

  * 令牌认证：与 Mastercard 合作的 Token Authentication Service


**🔧****风控配置**

商户可根据业务需求自定义风控策略，包括风险阈值、黑名单规则、3DS 触发条件等。

### （三）商户后台

**💻****统一管理后台**

提供跨多个收单行的汇总交易数据界面，支持实时查看交易流水、处理退款，并可在界面上设置路由规则。

**📊****报告功能**

支持导出交易明细、对账单，提供成功率分析图表，支持多种格式导出及通过接口获取。

### （四）Mastercard 数字支付方案

已集成 Mastercard 数字支付方案套件，包括一键支付（Click to Pay）、卡片安全存储（Secure Card on File）、支付通行密钥服务（Payment Passkeys），可减少对传统密码和一次性验证码的依赖。

### （五）独立令牌存储服务（payFURL Vault）

**🔒****跨网关令牌化**

将持卡人数据存储在 payFURL 自有的 PCI 一级认证环境中，生成独立令牌。

**🔄****供应商独立性**

商家更换后端收单行时无需迁移持卡人数据，仅需更改路由规则。

## **六、选择参考与适用场景**

###  典型应用场景

**🌐****多网关管理需求**

已接入多个支付网关、需要统一管理和智能路由的商户。payFURL 通过统一接口聚合多个网关。

**🔄****供应商锁定顾虑**

不希望被单一服务商绑定的商户。payFURL Vault 支持独立的令牌化存储，切换网关时仅需调整路由规则。

**🇦🇺****澳洲本地电商/中小企业**

澳大利亚中小企业可关注 XPANSE 产品线，该产品专为本地中小企业设计，并配有本地客服支持。

**💳****结账流程优化**

希望简化结账流程的商户，可评估 Mastercard 一键支付和支付通行密钥等方案。

**🔄****订阅业务**

运营 SaaS 或订阅业务的商户。payFURL 与 Chargebee 有官方集成，支持订阅管理与支付失败自动重试。

### 选择评估维度

**📊****业务复杂度**

如果仅需单一支付网关，直接对接该网关可能更简单；如果需要管理多个网关、优化成本或避免被单一服务商绑定，调度平台的价值更明显。

**🔧****技术实施能力**

评估团队是否具备 API 对接能力。如技术资源有限，可优先考虑 XPANSE 等简化产品或插件集成（Shopify、WooCommerce、Salesforce）。

**🌏****地理覆盖需求**

payFURL 核心团队位于澳大利亚，如业务主要在澳新市场，本地支持相对便利。

**💰****费用结构**

调度平台涉及平台费与合作网关费两部分，需评估智能路由带来的成本节省是否能平衡整体支出。在签约前，务必与服务商沟通，明确总体成本结构。
